# 表决票数基数计算规则

本文件说明 `ecm-draft-meeting-docs` 生成签到表、决议、表决统计表时所使用的**表决票数基数**计算规则。实务中"占比基数"错用是会议文件的高频问题。

---

## 一、关键基数的区别

| 基数名称 | 定义 | 用于计算什么比例 | 哪些文件用 |
|---------|------|---------------|----------|
| **总股本** | 公司发行的全部股份数 | 签到表的"占总股本比例" | 签到表 |
| **总表决权股份** | 有表决权的股份总数（扣除库存股、回购用于注销的股份） | 签到表的"占总表决权股份比例"（如区分） | 签到表（可选） |
| **出席股东所持有表决权股份总数** | 实际出席（现场 + 网络 + 委托）股东持有的表决权股份数 | 决议的"占出席股东所持有表决权股份总数的 XX.XX%" | 决议、表决统计表 |
| **扣除关联回避后的表决权股份总数** | 出席股东总数 - 本议案关联回避股东所持股份数 | 关联议案的表决比例基数 | 关联议案的决议 |
| **中小投资者表决权股份总数** | 实际出席的中小投资者（非 5% 以上股东、非董监高）所持表决权股份数 | 单独计票议案的"中小投资者表决比例" | 单独计票议案的决议（上市公司） |

---

## 二、基数使用规则（逐场景）

### 2.1 签到表

**分母 = 总股本**

示例：
```
出席股东合计：15 人；代表股份合计：800,000,000 股；占总股本：80.00%。
```

> 这里 80% 是分子为 8 亿股、分母为 10 亿总股本的结果；**不**使用"占表决权股份比例"为主要指标。

### 2.2 决议的普通议案

**分母 = 出席股东所持有表决权股份总数**

示例（出席股东持有 8 亿股的会议）：
```
议案 1：关于 YYYY 年度财务决算报告的议案
表决结果：
  同意 790,000,000 股，占出席股东所持有表决权股份总数的 98.75%；
  反对 5,000,000 股，占 0.625%；
  弃权 5,000,000 股，占 0.625%。
表决结果：通过。
```

### 2.3 决议的特别决议议案

**分母 = 出席股东所持有表决权股份总数**
**判断"通过"的阈值：同意票 ≥ 2/3**

示例：
```
议案 X：关于修改《公司章程》的议案（特别决议）
表决结果：
  同意 540,000,000 股，占出席股东所持有表决权股份总数的 67.50%（高于 2/3 通过线）；
  反对 200,000,000 股，占 25.00%；
  弃权 60,000,000 股，占 7.50%。
表决结果：通过（已获得 2/3 以上表决权股东通过）。
```

### 2.4 决议的关联议案

**分母 = 出席股东所持有表决权股份总数 - 关联股东所持股份数**

示例（假设关联股东 A 持股 2 亿、B 持股 1.5 亿，均需回避）：
```
议案 N：关于与 XX 公司签订《XX 协议》暨关联交易的议案
关联股东 A、B 已回避表决。
扣除关联股东后，本议案的表决权股份总数为 800,000,000 - 200,000,000 - 150,000,000 
  = 450,000,000 股。
表决结果：
  同意 440,000,000 股，占扣除关联股东后的表决权股份总数的 97.78%；
  反对 5,000,000 股，占 1.11%；
  弃权 5,000,000 股，占 1.11%。
表决结果：通过。
```

**关键点**：比例基数**不是**总出席股东的 8 亿股，而是**扣除 A 和 B 的 4.5 亿股**。错用基数会导致大幅失真（错误计算会得出 55% 的同意率，实际是 97.78%）。

### 2.5 上市公司中小投资者单独计票议案

**同时列出两个比例**（两个分母不同）：

```
议案 N：关于发行股份购买资产的议案（特别决议 + 中小投资者单独计票）

整体表决结果（扣除关联股东 A、B 后）：
  基数 = 450,000,000 股
  同意 440,000,000 股，占 97.78%；反对 5,000,000 股，占 1.11%；弃权 5,000,000 股，占 1.11%。
  整体表决结果：通过（已获得 2/3 以上通过）。

中小投资者表决结果（扣除关联股东 A、B 及董监高持股后）：
  基数 = 200,000,000 股（即 4.5 亿股扣除非中小投资者的部分）
  同意 195,000,000 股，占 97.50%；反对 3,000,000 股，占 1.50%；弃权 2,000,000 股，占 1.00%。
  中小投资者表决结果：通过。
```

## 三、常见错误识别

### 错误 1：签到表用"占出席股东表决权股份比例"

**错误**：
```
出席股东合计：15 人；代表股份合计：800,000,000 股；占出席股东所持有表决权股份总数：100.00%。
```

**问题**：签到表统计的是"出席情况"，应与"总股本"比较以体现出席率；分母用"出席股份"得出的必然是 100%（自我对比），无统计意义。

**正确**：
```
出席股东合计：15 人；代表股份合计：800,000,000 股；占总股本：80.00%。
```

### 错误 2：关联议案用总出席股份做分母

**错误**：
```
关联议案的表决结果：同意 440,000,000 股，占出席股东所持有表决权股份总数的 55.00%。
```

**问题**：关联股东 A、B 持股 3.5 亿已回避，他们的股份不应计入比例分母，否则定性错误（可能导致"未获通过"的错误结论）。

**正确**：
```
关联议案的表决结果：同意 440,000,000 股，占扣除关联股东后的表决权股份总数的 97.78%。
```

### 错误 3：中小投资者计票基数与整体混用

**错误**：
```
中小投资者表决结果：同意 195,000,000 股，占出席股东所持有表决权股份总数的 24.38%。
```

**问题**：中小投资者的比例应以"中小投资者持股"为分母；用总股东数做分母会使比例看起来很低（错误暗示议案未获通过）。

**正确**：
```
中小投资者表决结果：同意 195,000,000 股，占出席中小投资者所持有表决权股份总数的 97.50%。
```

### 错误 4：特别决议的 2/3 阈值判断基数错误

**错误**：用总股本（10 亿股）做分母，认为"同意需超 10 × 2/3 = 6.67 亿股"

**问题**：特别决议阈值分母应为"**出席**股东所持有表决权股份总数"，而非"公司总股本"。

**正确**：用出席股东 8 亿股为分母，"同意需超 8 × 2/3 = 5.33 亿股"。

## 四、meeting-docs 的自动计算

在用户提供了表决结果（同意 / 反对 / 弃权股数）和关联回避名单的情况下，meeting-docs 按以下步骤自动计算：

1. 读取公司总股本
2. 读取签到表的出席股东及其持股数
3. 计算出席股东所持有表决权股份总数
4. 对每项议案：
   - 若普通议案：分母 = 出席股东所持有表决权股份总数
   - 若关联议案：分母 = 出席股东所持有表决权股份总数 - 关联股东持股数
   - 若特别决议：检查同意票 ≥ 分母 × 2/3；否则标注"未获通过"
   - 若中小投资者单独计票：计算整体 + 中小投资者两套比例
5. 同意 / 反对 / 弃权百分比：`票数 / 分母 × 100%`，保留两位小数
6. 比例总和应为 100%（允许 ±0.01% 四舍五入误差）；超出则告警

---

## 五、meeting-docs 的校验

当用户**已提供表决结果**（会后起草场景），meeting-docs 自动验证：

| 校验点 | 期望 |
|-------|------|
| 同意 + 反对 + 弃权 = 分母 | 相等（允许 ±1 股四舍五入） |
| 各票比例之和 | 100.00%（允许 ±0.01%） |
| 特别决议的同意票 ≥ 分母 × 2/3 | 是（否则通过结论错误） |
| 关联议案的分母 | 扣除关联股东持股 |
| 中小投资者分母 | 扣除 5% 以上股东和董监高持股 |

校验不通过时在决议顶部打 "⚠️ 表决基数或结果存在异常" 并在一致性校验报告中详列。

## 六、《公司法》2023 年修订后的新情况

1. **监事会可选配**：若公司设立审计委员会替代监事会，不再产生监事会决议；meeting-docs 识别公司是否设立监事会，未设立则不生成监事会相关文件
2. **职工代表监事 / 职工董事**：若有职工代表监事 / 职工董事，该人选不由股东会选举（由职工大会选举）；meeting-docs 在股东会议案中不列入此类人员的选举
3. **书面决议制度（有限责任公司）**：有限公司可以全体股东书面一致同意方式作出决议，不召开股东会；meeting-docs 在用户指定"有限公司 + 书面一致同意"时，生成简化版文件（股东签字文本 + 决议）

## 七、基数校验的用户提示

meeting-docs 在生成决议 / 表决统计表时，若发现用户提供的数据可能存在基数错误（例如关联股东未在出席股东名单、或关联议案的分母未扣除关联股东），会主动提示：

```
⚠️ 基数校验告警：
  议案 3（关联交易议案）的表决结果使用了"出席股东所持有表决权股份总数"8 亿股为分母，
  但根据关联关系识别，本议案关联股东 A（持股 2 亿）、B（持股 1.5 亿）需回避。
  请确认：
    A. 基数应为 8 - 2 - 1.5 = 4.5 亿股（按关联回避规则）
    B. 用户提供的数据已按 4.5 亿股计算，只是披露时写成了"出席股东所持有表决权股份总数"
  哪种情况，请告知以便修正。
```

用户确认后 meeting-docs 再继续，不得自行决策。
