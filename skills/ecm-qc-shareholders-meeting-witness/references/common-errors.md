# 常见错误库（Common Error Catalog）

本清单基于 A 股市场见证业务公开披露的实务经验汇总，覆盖股东（大）会
法律见证意见书常见的错误类型。**这份清单是本 skill 最有"料"的部分——
反映的是同类错误在行业内反复出现的高频模式**，审查时请按本清单逐条筛查。

---

## A 类 — 基础信息类（事实错，最高频扣分项）

这类错误本质是"笔误/复制粘贴残留"，但影响严重（决议日期错可能
导致决议效力瑕疵）。发现即 tracked change 改正 +【必改】批注。

| 编号 | 典型错误 | 检查方法 |
|---|---|---|
| A1 | 股东（大）会**召开日期**错 | 三方对比：意见书 vs 股东会通知 vs 决议公告 |
| A2 | 股东（大）会**召开时间**错（AM/PM 混淆、时点错） | 核对通知中的时间 |
| A3 | **年份**错（跨年度的跨年错误，如当年会议误写上年年份） | 全文搜索年份 |
| A4 | **届次**错（第几次临时股东会） | 全文搜索届次编号 |
| A5 | **董事会届次/日期**错 | 核对董事会决议公告 |
| A6 | **会议地点**错 | 对照通知地址 |
| A7 | **公告披露日期**错 | 核对实际公告时间 |
| A8 | **公司简称**前后不一致 | 全文搜索简称 |
| A9 | **议案编号**错 | 逐项对照通知 |

**批注模板**：
- 【必改】经核对股东（大）会通知/决议公告，本次股东（大）会____
  （日期/时间/届次/地点）应为____，请订正。

## B 类 — 议案结构类

| 编号 | 典型错误 | 检查方法 |
|---|---|---|
| B1 | 总议案 vs 子议案混淆（将"统一表决"当成单独议案） | 核对通知议案结构 |
| B2 | 有子议案的议案只写子议案不写总议案 | 核对议案完整性 |
| B3 | 现场对议案内容临时变更，但未按董事会决议披露内容表决 | 核对议案内容与披露内容 |
| B4 | 应有"总议案 + 逐项表决"结构却缺总议案 | 核对议事体例 |

**批注模板**：
- 【必改】请区分议案____的总议案与逐项表决的子议案，并保持
  与股东（大）会通知、董事会决议一致。

## C 类 — 特别决议类（高频扣分）

> **法规依据：**《公司法》第 116 条第 2 款；《上市公司股东会规则》（2025
> 年修订）第 46 条（特殊回购情形）、第 24 条第 2 款（类别股事项）；各交易所
> 《股票上市规则》；公司章程。**完整事项清单 + 议案识别关键词见
> `voting-rules.md` §2。**

| 编号 | 典型错误 | 检查方法 |
|---|---|---|
| C1 | 应为特别决议事项，未标注"本项议案为特别决议事项" | 矩阵 X 第 3 列 |
| C2 | 应写明三分之二以上同意通过，未写 | 全文搜"三分之二""2/3" |
| C3 | 误把非特别决议议案标注为特别决议 | 关键词逆向匹配 |
| C4 | 三会议事规则作为章程附件，却未列为特别决议议案（**深交所上市公司**常见，2026 年后注意"审计委员会议事规则"替代了"监事会议事规则"） | 议案标题命中"议事规则修订" |
| C5 | 涉及特别决议的议案，未加上"本项议案为股东会特别决议事项"的表述 | 表决段是否含标准句式 |
| C6 | 特别决议议案表决同意比例 < 2/3，结论却写"通过" | 比例核算 → 比 66.67% |
| C7 | 章程附件性质的"董事会议事规则""审计委员会议事规则"（原"监事会议事规则"）修订未按特别决议处理 | 与 C4 配套，扩大到三会全部 |
| C8 | 议案性质应为特别决议（如发行公司债券、股权激励），但通知和见证意见均未标注 | 关键词识别 + 与公司核实 |
| C9 | 子议案结构中某些项目应为特别决议但未单独标注 | 子议案逐项识别 |
| C10 | 资本公积转增股本被误标为"增加注册资本"特别决议 | 议案内容辨析 |

**批注模板**（精简版；完整版见 `voting-rules.md` §8）：
- 【必改】本议案为特别决议事项（《公司法》第 116 条），请补"本项议案为
  股东会特别决议事项，已经出席本次股东会的股东及股东代理人所持表决权的
  三分之二以上同意通过"。
- 【必改】同意比例未达 2/3，议案未通过，请重写决议表述。

## D 类 — 累积投票制类

> **法规依据：**《公司法》第 119 条；《上市公司股东会规则》（2025 年修订）
> 第 33 条。**触发条件、表决格式、易误判情形见 `voting-rules.md` §3。**

| 编号 | 典型错误 | 检查方法 |
|---|---|---|
| D1 | 累积投票议案写了"反对""弃权"的表决情况 | 关键词："反对""弃权" 在累积投票议案段 |
| D2 | 非累积投票议案漏写"反对""弃权" | 选举议案是否等额 |
| D3 | 累积投票议案未列明当选人（"根据表决结果，XXX 当选为 XXX"） | 表决段末搜"当选" |
| D4 | 累积投票议案误写"同意/反对/弃权"三栏统计 | 表格结构核对 |
| D5 | 累积投票议案未说明"采用累积投票制" | 表决段开头是否说明 |
| D6 | 同时选举独立董事与非独立董事，合并计票（应当分议案分别累积投票） | 议案是否拆分 |
| D7 | 候选人数 = 应选名额（等额选举）却启用了累积投票 | 候选人 vs 应选名额 |
| **D8** | **选举两名以上独立董事或单一股东及一致行动人持股 ≥ 30% 时未采用累积投票**（《股东会规则》第 33 条强制） | 候选人数 + 控股股东持股核对 |
| D9 | 职工代表董事被纳入股东会议案（应由职代会选举） | 议案主体核对 |
| D10 | 罢免议案被误按累积投票处理 | 罢免应为普通决议 |

**批注模板**（精简版；完整版见 `voting-rules.md` §8）：
- 【必改】累积投票仅列同意票（《股东会规则》第 33 条），请删除反对/弃权。
- 【必改】请补"根据表决结果，XXX 当选为 XXX"。
- 【必改】本次选举独董 ≥ 2 名，强制采用累积投票制（同上条规）。

## E 类 — 关联回避与中小投资者单独计票类

> **法规依据（单独计票）：**《上市公司股东会规则》（2025 年修订）
> 第 32 条第 2 款；各交易所《股票上市规则》。**必属单独计票议案清单
> 见 `voting-rules.md` §4。**

| 编号 | 典型错误 | 检查方法 |
|---|---|---|
| E1 | 股东会通知中未列具体关联股东回避名单 | 通知"回避名单"段是否空 |
| E2 | 通知未列明的关联股东未核查即列在意见书中 | 名单一致性 |
| E3 | 通知明确为中小投资者单独计票议案，意见书未列明计票结果 | 见证表决段是否"中小投资者……股" |
| E4 | 隐性关联识别不彻底（亲属、曾任职、受益人等） | 关联股东穿透核查 |
| E5 | 关联股东名单与股东大会通知不一致 | 三方对照 |
| E6 | 通知"全部议案均为中小投资者单独计票"，意见书仅对部分议案列示 | 矩阵 Z 逐项检查 |
| E7 | 议案性质应单独计票（关联交易、股权激励、重大资产重组等），但通知漏标，意见书亦未列示 | voting-rules.md §4 反向识别 |
| **E8** | **见证意见"中小投资者"定义中仍含"监事"** —— 2026 年起监事会已撤销，定义应为"除上市公司董事、高级管理人员，以及单独或合计持有上市公司 5% 以上股份的股东以外的其他股东" | 全文搜"中小投资者，是指" |
| E9 | 关联回避 + 单独计票同时适用的议案，单独计票口径与关联回避混算 | 表决基数核对 |
| E10 | 单独计票结果的比例分母错误（应为出席中小投资者所持有效表决权，非全体出席股份） | 比例公式核对 |
| E11 | 同一议案的总表决结果与单独计票结果之和出现数字矛盾 | 数字交叉核算 |
| **E12** | **见证意见仍出现"监事""监事会"字样**（出席人员、回避名单、列席人员等任何位置） | 全文搜"监事" |

**批注模板**（精简版；完整版见 `voting-rules.md` §8）：
- 【必改】通知明确议案 X 为单独计票议案（《股东会规则》第 32 条第 2 款），
  请补列示中小投资者口径。
- 【必改】2026 年起监事会已撤销，请删除定义/出席人员/回避名单中的"监事"
  字样。
- 【核实】通知未列具体回避名单，请与公司确认。

## F 类 — 模板残留类（复制粘贴没删干净）

| 编号 | 典型错误 | 检查方法 |
|---|---|---|
| F1 | 不涉及网络投票，意见书保留了"网络投票的统计结果……" | 全文搜"网络投票" |
| F2 | 挂牌公司不适用《股东（大）会规则》，意见书未删除该法规 | 全文搜"股东会规则" |
| F3 | 《股东大会规则》未更新为《股东会规则》（2025 年修订） | 全文搜"股东大会规则" |
| F4 | 深交所公司引用了上交所投票系统地址（反之亦然） | 全文搜"投票系统" |
| F5 | 相关任职人员已离职但意见书仍保留其出席表述（如董秘离职未更新）| 与公司核实 |
| F6 | 结论意见段缺失（复制模板时漏了第三章节议案表决部分的结论意见段）| 章节末尾是否有结论段 |
| **F7** | **"具体时间以董事会发出的会议通知为准"冗余** —— 同一次董事会决议中已经明确了股东会召开日期，"以董事会通知为准"是模板残留；但如果公司确实开两次董事会（第一次决定开股东会但暂不发通知，第二次定日期发通知），保留无误 | 对照董事会决议公告：同次会议已定日期 → 必改；多次会议 → 保留 |
| **F8** | **"中国台湾地区""中国台湾""台湾"等非规范用语** —— 应为"中国台湾省" | 全文搜"台湾" |
| **F9** | **页眉 / Logo / 抬头含 "Mallesons" 字样** —— 2024 年金杜与 Mallesons 全球分家后，金杜律所出具的文件不应再含此字样 | 跑 `extract_images.py` 视觉读图，详见 visual-check.md |
| **F10** | **意见书载明的法律适用范围未排除港澳台**（如错写"中华人民共和国（包括…和中国台湾省）境内"，应为"不包括"） | 搜"中华人民共和国" + "包括" |

**批注模板**：
- 【必改】本次股东会不涉及网络投票，请删除"网络投票的统计结果……"相关表述。
- 【必改】挂牌公司不适用《上市公司股东会规则》，请全文删除。
- 【必改】《上市公司股东大会规则》已于 2025 年修订为《上市公司股东会规则》，请全文更新。
- 【必改】公司系____证券交易所上市公司，请更正网络投票系统地址。
- 【必改】公司相关任职人员已离职，开会时由____代为履行职责，请订正。
- 【必改】同次董事会决议已明确了股东会召开日期，"具体时间以董事会发出的会议通知为准"系模板残留，请删除。
- 【必改】"中国台湾地区"应为"中国台湾省"（国务院新闻办规范用语）。
- 【必改】页眉 / Logo 含 "Mallesons" 字样，2024 年金杜与 Mallesons 分家后金杜文件不应再含此字样，请更换。
- 【必改】法律适用范围表述"中华人民共和国（包括…和中国台湾省）境内"应为"**不包括**…"，意见书的法律适用范围仅限中国内地。

## G 类 — 章程/议事规则适用类

| 编号 | 典型错误 |
|---|---|
| G1 | 现行有效的公司章程与本次股东会审议的尚未生效的公司章程描述混乱 |
| G2 | 未核查披露公司现行有效的公司章程 |
| G3 | 章程修订版本引用错误（引用旧版或未生效版） |

**批注模板**：
- 【必改】请明确区分"截至本法律意见出具日现行有效的公司章程"与
  "本次股东（大）会审议通过后将生效的公司章程（修订稿）"，避免
  混用。
- 【核实】请核查并在底稿中落实公司现行有效的公司章程版本。

## H 类 — 底稿与查验计划类

这类问题用【底稿】批注提示，正文一般不需要 tracked change。

| 编号 | 典型错误 |
|---|---|
| H1 | 现场出席 0 人但《查验计划》未记录 |
| H2 | 底稿上传了现场照片，但出席股东的持股证明/身份证明底稿未上传 |
| H3 | 股东会召开当日未更新上传《查验计划》 |
| H4 | 会后底稿未上传股东会决议、会议记录 |
| H5 | 会后快速通过，未上传法律意见书定稿 |
| H6 | 股东大会现场会议照片只拍了律师但未拍现场全貌 |
| H7 | 视频方式参会的董监高未保存线上截图 |
| H8 | 董监高请假未出具书面请假条（仅电话/邮件/微信）|
| H9 | 客户关于真实准确完整的声明/保证仅为邮件回复确认 |
| H10 | 提交质控时项目组不上传底稿 |

**批注模板**：
- 【底稿】请落实底稿：____（具体底稿项目），并在《查验计划》中
  同步记录。
- 【底稿】现场出席股东为 0 人时，请在《查验计划》中记录该情况。
- 【底稿】保证/承诺类文件请落实底稿，一般是上市公司盖章出具书面
  承诺，或至少取得董事会办公室、证券投资部门的邮件确认。
- 【底稿】视频方式参会的董监高请保存线上截图存档。

## I 类 — 质控程序类

这类属于律师执业程序问题，意见书文本本身可能没问题，但应以
【核实】批注提示：

| 编号 | 典型错误 |
|---|---|
| I1 | 股东会召开当日才提交质控 |
| I2 | 未在股东会召开两日前提交质控，也未与质控小组提前沟通 |
| I3 | 股东会召开前一天晚上才提交 |
| I4 | 签字律师与律所质控系统中填写的见证律师不一致 |
| I5 | 对质控批注的意见大部分未进行回复 |
| I6 | 律师未出席会议 |

**批注模板**（通常加在文件开头或签字页附近的整体性批注）：
- 【核实】根据律所见证业务安排，请务必在股东（大）会召开前 2 日
  以上提交质控；如有特殊情况请提前与质控小组沟通。
- 【核实】请核对签字律师与律所质控系统中填写的见证律师是否一致；
  如不一致请及时修正。
- 【核实】会后请将定稿法律意见书及完整底稿及时上传律所归档与
  质控系统。

## 一致性交叉核查（高频扣分的元问题）

这不是某一个错误点，而是审查**方法论**。内核审查时应建立以下
三个"一致性矩阵"同步核对：

### 矩阵 1 — 基础信息三方一致性

| 核查字段 | 见证意见 | 股东（大）会通知 | 决议公告 |
|---|---|---|---|
| 召开日期 |   |   |   |
| 召开时间 |   |   |   |
| 召开地点 |   |   |   |
| 届次 |   |   |   |
| 股权登记日 |   |   |   |

**任何字段三者不一致 → tracked change + 【必改】批注**。

### 矩阵 2 — 议案一致性

| 议案编号 | 通知是否列明 | 意见书审议章节是否列明 | 决议内容是否列明 | 是否特别决议 | 是否关联回避 | 是否中小投资者单独计票 |

**逐项核对**，缺一即批注。

### 矩阵 3 — 关联股东/回避名单一致性

通知中列的回避名单 vs 意见书列的回避股东 vs 表决结果中扣减的
表决权数，三者应当对应。

## 审查顺序建议

**建议按以下顺序过文件（效率最高）**：
