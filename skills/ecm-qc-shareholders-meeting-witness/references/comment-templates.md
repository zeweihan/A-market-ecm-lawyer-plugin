# 批注标准话术模板（Comment Templates）

> **批注硬规则**（与 SKILL.md 同步）：
> 1. **每条批注 ≤ 3 句话**，律师风格——指问题 + 给改法或法规依据。
> 2. **只批问题，不批"已核查无问题"**。三种特殊表决方式、形式要件等
>    专项核查若整体合规，**不要写综合意见、不要写"经核对一致"**。
> 3. **不要给出长篇解释或绕弯子的开头**（"本批注涉及……"、"基于法规
>    依据……"等绝对禁止）。
> 4. 批注按【必改】/【核实】/【建议】/【底稿】分类，分类前缀必加。

---

## 1. 前言部分（开篇声明段 / 委托段）

### 1.1 律师执业关系（转委托）

【核实】见证律师执业关系应在签署常法服务协议并出具法律意见书的同一分所；
如否，需建立转委托关系（签署协议或取得董办/董秘的书面/邮件同意）。

### 1.2 保证段

【底稿】请落实底稿，上市公司盖章出具书面承诺，或至少取得董办/证券部邮件确认。

### 1.3 挂牌公司法规引用

【必改】挂牌公司不适用《上市公司股东会规则》，请全文删除该法规引用。

---

## 2. 召开程序部分

### 2.1 召开地点与通知不一致

【必改】见证意见载明地点"____"与股东会通知载明地点"____"不一致，请订正。

### 2.2 非董事长/副董事长主持

【核实】如非董事长、副董事长主持，请查验推举程序、收集底稿，并在《查验计划》
同步记录。

### 2.3 出席人员资格

【核实】此处请根据实际出席情况调整核查资料描述，并在《查验计划》同步记录。

### 2.4 任职人员离职/代行

【必改】公司____已离职，开会时由____代为履行职责，请订正表述。

### 2.5 召集人权限

【核实】请核查召集人权限（董事会决议/审计委员会决议/股东请求），并落实底稿。

### 2.6 通知时限

【必改】年度股东会通知应于会议召开 20 日前发出（《公司法》），本次通知披露日
____至召开日____不足____日，请核查。

---

## 3. 议案与表决部分

### 3.1 总议案与子议案结构

**默认**：议案被拆成子议案逐项表决时，行业通行做法是**只列子议案表决结果，
不再列总议案**，**不需要批注**。

**例外**（通知提案编码表把总议案单独编码并标"作为投票对象的子议案数（N）"，
意味着深交所投票系统可能对总议案有独立统计）：

【核实】通知提案编码表将议案 X.00 单独编码并标"作为投票对象的子议案数（N）"，
请核查深圳证券信息有限公司提供的投票统计中是否有针对总议案的独立票数；如有
请列示，如无可不列。

### 3.2 特别决议

- **通知已标 + 见证未写**：【必改】本议案为特别决议事项（《公司法》第 116 条），
  请补"本项议案为股东会特别决议事项，已经出席本次股东会的股东及股东代理人
  所持表决权的三分之二以上同意通过"。
- **通知未标但性质应是**：【核实】议案 X 涉及[修改章程附件/三会议事规则修订/
  发行公司债券/股权激励/重大资产重组]，是否应按特别决议审议？请与公司核实。
- **同意比例 < 2/3 却写"通过"**：【必改】同意比例 XX.XXXX% 未达 2/3，议案
  未通过，请按未通过结果重写决议表述。
- **误标特别决议**：【必改】本议案非特别决议事项，请删除"特别决议"表述。
- **三会议事规则修订**（深交所高频）：【核实】议案 X 是否为章程附件性质的三
  会议事规则修订？如是应按特别决议审议。**注意**：2026 年起监事会已撤销，
  "三会"指股东会、董事会、审计委员会。

### 3.3 关联股东回避

- **通知缺名单**：【必改】通知未列具体回避名单，请与公司确认关联股东明细。
- **名单不一致**：【核实】通知与见证意见列示的回避股东是否一致；隐性关联
  （亲属、曾任职、受益人等）是否已穿透核查。

### 3.4 中小投资者单独计票

- **通知已标 + 见证未列示**：【必改】通知明确议案 X 为单独计票（《股东会规则》
  第 32 条第 2 款），请补"其中，中小投资者表决情况：同意 XXX 股，占
  XX.XXXX%；反对……弃权……"。
- **通知未标但性质应是**：【核实】议案 X 属于关联交易/股权激励/重大资产重组，
  请与公司确认是否漏标单独计票。
- **"全部议案均单独计票" + 部分议案缺列**：【必改】通知约定本次所有议案均
  单独计票，议案 X/Y/Z 缺列，请逐项补。
- **定义中仍含"监事"**：【必改】2026 年起监事会已撤销（证监会 2024-12-27
  过渡期安排），定义应为"除上市公司董事、高级管理人员，以及单独或合计持有
  上市公司 5% 以上股份的股东以外的其他股东"，请删除"监事"二字。
- **分母错**：【必改】单独计票分母应为出席中小投资者所持有效表决权，非全体
  出席股份总数。

### 3.5 累积投票

- **出现反对/弃权**：【必改】累积投票仅列同意票（《股东会规则》第 33 条），
  请删除反对/弃权票数。
- **未说明累积投票**：【必改】请补"本次选举采用累积投票制"。
- **缺当选结论**：【必改】请补"根据表决结果，XXX 当选为 XXX"。
- **强制条件触发但未启用**：【必改】本次选举独董 ≥ 2 名（或单一股东及一致
  行动人持股 ≥ 30%），依《股东会规则》第 33 条强制采用累积投票制。

### 3.6 残留"监事"表述（任何位置）

【必改】2026 年起监事会已撤销（证监会 2024-12-27 过渡期安排），监事职权由
审计委员会承担，请删除[出席人员/回避名单/中小投资者定义/列席人员]中的
"监事""监事会"字样。

### 3.7 网络投票

- **不适用网络投票**：【必改】本次股东会不涉及网络投票，请删除网络投票相关
  表述。
- **交易所投票系统混淆**：【必改】公司系____证券交易所上市公司，请更正网络
  投票系统地址为____。

## 4. 决议内容部分

### 4.1 独立董事选举

【核实】董事会是否对独立董事候选人是否被交易所提出异议的情况进行了说明？
独立董事交易所审核情况是否已查验？请落实底稿。

### 4.2 当选人表述缺失

【必改】请补"根据表决结果，XXX 当选为 XXX"。

### 4.3 结论意见段缺失

【必改】议案表决章节缺结论性意见段，请补"本律师认为，本次股东会上述议案
的表决结果合法、有效"。

### 4.4 表决结果与结论矛盾

【必改】议案表决结果为"通过"，但结论写为"未通过"（或反之），请核查订正。

---

## 5. 结论意见与签字页

### 5.1 含糊措辞

【必改】结论性意见避免使用"基本符合""未发现"等含糊措辞（《律师事务所从事
证券法律业务管理办法》第 23 条、《12 号规则》第 8 条），请改为"符合/不符合"
等明确表述。

### 5.2 签字律师数量

【必改】本法律意见书仅有____名律师签字，根据《律师事务所证券法律业务管理
办法》第 26 条，应由 2 名以上具有证券业务执业资格的经办律师签名。

### 5.3 签字律师与质控记录不一致

【核实】请核对签字律师与律所质控系统中填写的见证律师是否一致。

### 5.4 签字页文件名

【建议】最终定稿提交系统时"文件名称"栏请写全称。

## 6. 其他特殊情形

### 6.1 0 人出席

【底稿】现场出席股东及股东代理人共 0 人，请在《查验计划》记录该情况。

### 6.2 视频方式参会

【底稿】视频方式参会的董事、高管，请保存线上截图存档。

### 6.3 董事/高管请假

【底稿】董事/高管请假应出具书面请假条，仅电话、邮件、微信沟通的应补充书面
材料或正式邮件请假。

### 6.4 占位符待填

【底稿】数字字段存在"【】"占位符，会后请按股东名册、签到表、深圳证券信息
有限公司投票统计逐一填写，并复核比例之和、口径一致性。

## 批注格式注意事项

1. **XML 实体转义**（写入 comment.py 时）：`&` → `&amp;`；`<` → `&lt;`；
   `>` → `&gt;`；`'` → `&#x2019;`；`"` → `&#x201C;` / `&#x201D;`
2. **作者名**：所有 `comment.py` 调用必须加 `--author "内核"`
3. **ID 唯一**：每条批注一个唯一 ID（0, 1, 2 ... 递增）
4. **放置位置**：`<w:commentRangeStart>` / `<w:commentRangeEnd>` 是 `<w:p>`
   的直接子节点，**不要**套进 `<w:r>` 里
5. **范围精确**：批注覆盖的文字范围尽量精确到问题所在的几个字或一句话，
   不要覆盖整段
