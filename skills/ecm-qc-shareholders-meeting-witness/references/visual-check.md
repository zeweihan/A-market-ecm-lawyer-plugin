# 视觉 / Logo 校验（Visual Check）

## 为什么单独成篇

见证意见的页眉、页脚、签字页常含**律所 Logo 图片**。这些图片是 docx 的内嵌
媒体（`word/media/imageN.png`），用纯文本提取看不见——必须用视觉能力读图。

试跑发现 AI 漏识别的典型场景：

> **莎普爱思 2025 年年度股东会见证意见**——页眉 Logo 含 "King & Wood
> **Mallesons**" 英文字样。
>
> 但 2024 年金杜（King & Wood）已与 Mallesons 全球分家，**金杜律师事务所
> 出具的任何法律文件都不应再含 "Mallesons" 字样**——这是品牌独立后的硬性
> 要求。

---

## 一、硬规则

### 规则 1：金杜律所意见书禁含 "Mallesons"

**任何**金杜律师事务所（含上海分所、深圳分所、成都分所、北京总所等所有分所
/ 总所）出具的法律意见书，**页眉、页脚、Logo 图片、签字页、抬头、水印**等
任何位置**均不应出现** "Mallesons" 英文字样。

**判定标准**：

- 律所名称含"金杜"或"King & Wood"
- 任何嵌入图片含字符串 "Mallesons" / "MALLESONS" / "Mallesons LLP"

→ 【必改】请更换页眉 / Logo / 签字页图片，删除 "Mallesons" 字样，使用金杜
独立后的统一 VI。

### 规则 2：律所抬头与出具方一致

意见书正文声明的出具方（如"北京市金杜律师事务所上海分所"）应与页眉 / 签字
页显示的律所信息一致。

不一致情形：
- 正文：上海分所；页眉：深圳分所 → 【必改】
- 正文：上海分所；签字页盖章：总所 → 【核实】

## 二、执行步骤

### Step 1: 提取 docx 内嵌图片

```bash
# 用脚本一次性提取并展示所有图片
python3 scripts/extract_images.py /path/to/file.docx
```

脚本会把 `word/media/*` 下的所有图片解压到临时目录并打印路径，便于后续逐张
读图。

### Step 2: 逐张视觉读图

对每个图片调用 Read 工具（视觉模式）。重点判断：

- 图片中是否包含 **"Mallesons"** 文字（任何大小写形式）
- 图片是律所 Logo 还是公司印章
- Logo 中显示的律所名称、地址、电话等信息

### Step 3: 写批注

发现含 "Mallesons" 字样 → 【必改】批注。其他视觉异常 → 【核实】。

**不要批注**：
- 律所 Logo 设计是否好看
- 图片清晰度
- 公司印章是否符合规范（这是公司层面的事）

## 三、典型 Logo 字样识别

### 金杜独立前 Logo（**禁止**出现在 2024 年后出具的意见书中）

```
King & Wood Mallesons
金杜律师事务所
```

或

```
KING & WOOD
MALLESONS.
金杜律师事务所
```

### 金杜独立后 Logo（**正确**）

```
King & Wood
金杜律师事务所
```

或仅中文标识：

```
金杜律师事务所
```

## 四、批注模板

### 4.1 含 "Mallesons" 字样

【必改】页眉 / 签字页 / 抬头 Logo 含 "Mallesons" 字样。2024 年金杜与 Mallesons
全球分家后，金杜律所出具的法律文件不应再含此字样，请更换为金杜独立后的统一 VI。

### 4.2 律所抬头与出具方不一致

【必改】页眉显示 [X 分所]，与正文声明的出具方 [Y 分所] 不一致，请核对并统一。

### 4.3 律所抬头不清 / 残缺

【核实】页眉 Logo 残缺 / 模糊，请确认是否为律所最新 VI，并补充完整页眉。

---

## 五、视觉校验例外

- 公司方提供的财务报表截图、会议照片、董事签到表照片等业务相关图片，**不在
  视觉校验范围**——这些是底稿，不需要批注
- 抬头页只有意见书首页和签字页相关，正文中段无需逐页检查
- 跨页页眉（如每页页眉一致）只需检查一次
