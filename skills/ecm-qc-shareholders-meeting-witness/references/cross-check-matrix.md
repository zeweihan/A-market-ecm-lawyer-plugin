# 交叉比对矩阵（Cross-Check Matrix）

本文件规定**在已获得"股东大会通知"和"董事会决议公告"两份参考文件的情况下**，
内核审查应逐字段进行的一致性核对。这是本 skill 的核心增量价值——**没有参考
文件时做的只是语法和体例审查，有了参考文件才能做事实性的交叉核对**，而事实错
恰恰是行业常见错误模式里最高频的类别（A 类常见错误中"日期/时间/届次"三连错占了
过半比例）。

---

## 工作原则

1. **三源对照**：凡参考文件中存在的字段，**都要把见证意见里的写法、股东大会
   通知里的写法、董事会决议公告里的写法三者摆出来对照**。任何两者不一致都需
   要处理。
2. **以"最源头"为准**：
   - 董事会是否召开 / 召开日期 / 届次 → 以**董事会决议公告**为准
   - 股东大会召开日期/时间/地点/议案 → 以**股东大会通知**为准
   - 见证意见里相应字段若与这两者不一致，一律 tracked change 订正 +【必改】批注，
     批注说明依据来源
3. **处理方式**：
   - 事实错（可直接从参考文件取到正确值）→ tracked change + 批注注明来源
   - 事实疑（参考文件也含糊/多种说法）→ 不改正文，只【核实】批注
   - 参考文件里没有的字段（如关联股东身份隐性核查）→ 走原有的【核实】/【底稿】批注路径

批注里引用参考文件时，用明确表述，例如：
> 【必改】经核对公司于 YYYY-MM-DD 披露的《XX 股东大会通知》（公告编号
> XXXXXXX），本次股东大会召开日期应为 YYYY-MM-DD，见证意见此处的 YYYY-MM-DD
> 有误，请订正。

## 核心交叉核对字段（12 项）

下表中"✓"表示该字段应在该文件里存在并核对；"—"表示通常不含。
"权威性"表示不一致时以哪个文件为准。

| 字段 | 见证意见 | 股东大会通知 | 董事会决议公告 | 权威性 | 处理 |
|---|:-:|:-:|:-:|---|---|
| 股东大会**召开日期** | ✓ | ✓ | ✓（决定召开日期） | 股东大会通知 | tracked change + 【必改】 |
| 股东大会**召开时间**（含上/下午、具体时点） | ✓ | ✓ | — | 股东大会通知 | tracked change + 【必改】 |
| 股东大会**召开地点** | ✓ | ✓ | — | 股东大会通知 | tracked change + 【必改】 |
| 股东大会**届次**（"2025年年度"/"2025年第X次临时"） | ✓ | ✓ | ✓ | 股东大会通知 | tracked change + 【必改】 |
| **股权登记日** | ✓ | ✓ | — | 股东大会通知 | tracked change + 【必改】 |
| **网络投票安排**（平台、开通/截止时间） | ✓ | ✓ | — | 股东大会通知 | tracked change + 【必改】 |
| **审议议案清单** | ✓ | ✓ | ✓（董事会提交部分） | 股东大会通知（总清单）+ 董事会决议（来源） | 缺失/多余：【必改】 |
| **特别决议议案**的标注 | ✓ | ✓ | ✓ | 股东大会通知 | 标注缺失：【必改】 |
| **关联股东回避**名单 | ✓ | ✓ | — | 股东大会通知 | 缺失/不一致：【必改】 |
| **中小投资者单独计票**议案标注 | ✓ | ✓ | — | 股东大会通知 | 缺失：【必改】 |
| **董事会届次**（"第 X 届董事会第 Y 次会议"） | ✓ | ✓（通知引用） | ✓ | 董事会决议公告 | tracked change + 【必改】 |
| **董事会召开日期** | ✓ | ✓（通知引用） | ✓ | 董事会决议公告 | tracked change + 【必改】 |

**加分项**（不在核心 12 项，但有参考文件就能顺手做的）：

| 字段 | 见证意见 | 股东大会通知 | 董事会决议公告 | 说明 |
|---|:-:|:-:|:-:|---|
| 公告**披露日期**（董事会决议 + 股东大会通知） | ✓ | ✓（自身） | ✓（自身） | 用于计算是否满足法定通知期（年度 ≥20 日、临时 ≥15 日、新三板临时 ≥10 日） |
| 公司**全称**与**简称** | ✓ | ✓ | ✓ | 跨文档简称一致性检查 |

## 推导性检查（基于参考文件可算出来的）

这几项不是简单字段对照，而是需要基于参考文件里的字段做计算：

### 1. 法定通知期是否满足

- **年度股东大会**：股东大会通知披露日 → 召开日之间 **≥ 20 个自然日**
- **临时股东大会**（沪深）：**≥ 15 个自然日**
- **新三板临时股东会**：**≥ 10 个自然日**
- 计算方式：股东大会通知的"公告时间"（从 cninfo `announcementTime` 字段得到）
  → 见证意见/通知中载明的"召开日期"

不足的处理：
> 【必改】经核对，《股东大会通知》披露于 YYYY-MM-DD，本次股东大会召开日为
> YYYY-MM-DD，通知期为 N 日，**不足**《公司法》第 XX 条及公司章程第 XX 条
> 规定的 M 日法定通知期，请项目组核查。

### 2. 董事会召集 → 股东大会通知 → 股东大会召开 的时序逻辑

合规时序应是：董事会决议日 ≤ 股东大会通知披露日 ≤ 股东大会召开日。
如果出现倒挂（比如见证意见写的董事会日期 > 通知披露日期），几乎一定是事实错。

### 3. 议案清单完整性矩阵

把三份文件里的议案逐条摆出来画个对照表：

| 议案编号 | 股东大会通知中 | 董事会决议公告中（提交审议） | 见证意见中审议章节 | 见证意见中决议章节 |
|---|---|---|---|---|

三列都有 × 见证意见两列都有 = PASS。任何一列缺失都要批注。
**特别注意**：有"总议案"和"子议案"结构的（如议案 1 下设 1.1、1.2、1.3），
见证意见须对总议案和每个子议案分别写审议结果。

### 4. 特别决议 / 累积投票 / 中小投资者单独计票的标注一致性

- 如果股东大会通知标注某议案为"特别决议事项"，见证意见里该议案表决段必须
  有"本项议案为股东（大）会特别决议事项，已经出席本次股东（大）会的股东及
  股东代理人所持表决权的三分之二以上同意通过"字样。
- 如果股东大会通知标注某议案为"中小投资者单独计票"，见证意见必须列明该议案
  的中小投资者表决结果。
- 如果是累积投票议案（一般是董事/监事选举且候选人数 > 应选数时启用），见证
  意见里该议案的表决结果只能有"同意"票数，**不应有"反对""弃权"票**。

## 输出到批注的标准话术（带参考文件证据版）

相比 `comment-templates.md` 里的通用话术，有了参考文件后批注应当**引用
参考文件的具体字段作为证据**，这样项目组看到批注能直接复核，不用再翻原始公告。

### 日期/时间/届次类

> 【必改】经核对《XX 股东大会通知》（披露于 YYYY-MM-DD，公告编号 XXXXX），
> 本次股东（大）会召开日期/时间/地点/届次应为「XX」，见证意见此处表述为「YY」
> 与通知不符，请订正。

### 董事会届次/日期类

> 【必改】经核对《第 X 届董事会第 Y 次会议决议公告》（披露于 YYYY-MM-DD），
> 决定召开本次股东（大）会的董事会为"第 X 届第 Y 次会议"，召开日期 YYYY-MM-DD。
> 见证意见此处表述为「...」，请核对订正。

### 议案缺失类

> 【必改】经核对《XX 股东大会通知》，本次股东（大）会审议议案包括议案 X
> "XXX"，但见证意见中未见对该议案的审议与表决段落，请补充。

### 特别决议标注类

> 【必改】《XX 股东大会通知》明确议案 X "XXX"为股东（大）会特别决议事项，
> 见证意见未标注，请补充表述："本项议案为股东（大）会特别决议事项，已经
> 出席本次股东（大）会的股东及股东代理人所持表决权的三分之二以上同意通过。"

### 中小投资者单独计票类

> 【必改】《XX 股东大会通知》明确议案 X 为中小投资者单独计票议案，见证意见
> 未列明该议案的中小投资者表决结果（同意/反对/弃权票数及比例），请补充。

### 关联回避类

> 【必改】《XX 股东大会通知》明确关联股东 [A、B、C] 应回避议案 X 的表决。
> 见证意见回避名单为 [X、Y]，与通知不一致，请核对订正。

### 法定通知期不足类

> 【必改】《XX 股东大会通知》披露于 YYYY-MM-DD，本次会议召开日为 YYYY-MM-DD，
> 间隔 N 日，不足《公司法》关于[年度/临时]股东大会提前 M 日通知的规定，请核查
> 补救措施（如补充披露、推迟召开等）。

## 特殊场景

### 场景 A：找到多份董事会决议公告候选

同一公司往往一年开多次董事会。`fetch_cninfo_announcements.py` 脚本默认挑选
"公告时间最接近股东大会召开日、且在其之前"的那份。但如果候选列表中有多份
都很接近（比如同一天披露了两份董事会决议公告），Claude 应当：

1. 查看见证意见里引用的董事会届次（如"第八届董事会第十五次会议"）
2. 在候选列表（`board_resolution_candidates`）中找标题精确匹配该届次的那份
3. 如果脚本自动挑的不对，用 `board_resolution_candidates` 里该项的 `pdf_url`
   手动下载（或改用该 announcement_id 再跑一次脚本）

### 场景 B：找到的股东大会通知不止一份

常见于有"补充通知"或"更正通知"的情况。脚本默认排除标题含"补充/更正"的，
只挑主通知。如果见证意见明确基于"补充通知"（即通知内容经修订后召开），
Claude 应当同时参考主通知和补充通知。

### 场景 C：公司更改披露板块或新老代码转换

如北交所新老代码对照情形，用老代码搜不到时用新代码再试。
脚本的 `--market` 参数允许在"沪深京"和"三板"之间切换。

### 场景 D：挂牌公司（新三板）

- `--market` 传 `三板`
- 审查时移除《上市公司股东会规则》的适用性判断（这是挂牌公司的高频扣分点）
- 通知期按新三板规则：年度 20 日、临时 10 日

## 如果没有参考文件（降级路径）

SKILL.md 的 Step 1.5 规定三级降级：自动拉取 → 用户上传 → 跳过交叉比对。
如果最终没有参考文件，本矩阵中**所有依赖参考文件证据的批注都不要写**——
只做 `form-requirements.md` / `substantive-checklist.md` / `common-errors.md`
三个清单规定的审查，并在输出文档开头加一条"整体性批注"说明未做交叉比对，
详见 SKILL.md Step 1.5。
