# ecm-dd-assets

## 定位与边界

本 skill **负责**：
- 核查发行人不动产（房产 + 土地使用权）权属清晰性，包括国有划拨 vs 出让、集体土地、
  宅基地、建筑物合法性（是否违建）、权属瑕疵与历史遗留问题
- 核查发行人知识产权（专利、商标、著作权、软件著作权、域名、商业秘密、非专利技术、
  特许经营权）的权属、有效期、维持缴费、侵权风险
- 核查发行人重大生产设备的权属、是否存在融资租赁 / 保留所有权买卖等特殊安排
- 核查发行人承租的重大资产（房屋、设备、土地）的租赁合同合法性、租赁备案、出租方权属
- 核查上述资产的权利受限情况（抵押、质押、查封、冻结、扣押、轮候查封、优先权等）
- 核查权属纠纷（诉讼、仲裁）对主要财产的潜在影响
- 出具编报规则第 9 章对应的 DD Memo

本 skill **不负责**：
- 资产相关的借款 / 担保 / 抵押合同条款（归 `ecm-dd-debt`；本 skill 只记录权利受限事实）
- 资产相关的关联交易（归 `ecm-dd-related-party`）
- 会计科目层面的资产负债表分析（财务专业领域，DD 律师一般不介入）
- 资产相关的税务处理（归 `ecm-dd-tax`）
- 资产形成的主营业务合法性（归 `ecm-dd-business`）
- 资产独立性的独立性结论（归 `ecm-dd-independence`；本 skill 提供资产权属输入）
- 募投项目所需预留地 / 房产 / 设备（归 `ecm-dd-fundraising`，BATCH-04）
- 资产相关的诉讼 / 仲裁（归 `ecm-dd-litigation`，BATCH-04；本 skill 只关注涉及权属本身的纠纷）
- 文件读取 / 搬运（归 `ecm-dd-file-review` / `ecm-setup-file-organize`）

## 核心工作流（四步）

1. **生成核查要点清单**：照 [references/checklist.md](./references/checklist.md) 输出本项目的核查清单
2. **文件审阅与标注**：对照清单审阅 `02-08-主要财产/` 目录下的不动产权证、IP 证书、
   设备采购合同、租赁合同、权利受限证明；必要时调用 `ecm-dd-data-verify` 对不动产查询 /
   知识产权局 / 商标局等公开数据做交叉验证
3. **风险分级**：按下方口径给每个问题打高 / 中 / 低
4. **结论与建议**：汇总主要财产权属状况和权利受限情形，列出补办证照、解除权利限制、
   租赁备案、IP 续展等建议

## 统一风险分级口径

| 级别 | 定义 | 本 skill 场景的典型例子 |
|------|------|------------------------|
| **高（红）** | 足以构成发行 / 交易的实质性障碍 | 核心不动产存在重大权属纠纷或被查封且金额重大；关键生产基地土地性质违规（集体土地上自建厂房用于经营）；核心专利 / 商标被宣告无效或撤销且无备份 IP；核心 IP 权属存在争议或登记在关联方名下未转回；核心设备处于融资租赁期且有终止风险 |
| **中（黄）** | 程序或文件存在瑕疵但可补正 | 部分 IP 即将到期但可续展（商标 10 年 / 专利 20 年）；租赁房屋未办理租赁备案；权属证书名称与发行人名称历史差异但已变更；少量设备权属不清晰但金额不重大；部分厂房存在报批手续滞后（需补办或取得主管部门证明） |
| **低（蓝）** | 不影响实质的形式瑕疵 | 权属证书扫描不清；IP 证书副本缺失可向原登记机关补办；商标分类覆盖不完整但主业类别齐全；域名续费周期临近但可自行续费 |

## 输出格式契约（强制）

严格遵循 [shared/templates/dd-skill-template.md](../../shared/templates/dd-skill-template.md) 的
"输出格式契约"五段式结构：

1. 一级标题：`# DD Memo：发行人的主要财产`
2. 二级标题：`一、核查要点清单` / `二、审阅发现` / `三、风险分级汇总` / `四、结论与建议` / `五、参考资料`
3. 风险等级列取值限"高 / 中 / 低"
4. 输出落地：`02-尽职调查/02-08-主要财产/DD-Memo-主要财产-{YYYYMMDD}.md`

**额外要求**（本 skill 专属）：

1. Memo 中必须附**不动产清单表**，字段：位置 / 权属类型（自有 / 租赁）/ 权证号 / 权利人 /
   建筑面积 / 土地面积 / 土地性质（国有出让 / 国有划拨 / 集体）/ 取得方式 / 是否抵押 /
   是否查封 / 备注
2. Memo 中必须附**知识产权清单表**，按类别（专利 / 商标 / 著作权 / 软件著作权 / 域名 /
   其他）分类列示，字段：名称 / 权利人 / 证号 / 类别 / 申请日 / 授权日 / 有效期至 /
   是否质押 / 是否有纠纷 / 是否支撑核心业务
3. Memo 中必须附**重大设备清单表**（账面价值占比较高或对生产经营关键）：设备名称 /
   权利人 / 取得方式（自有购置 / 融资租赁 / 经营租赁 / 无偿占用）/ 价值 / 权利受限情况
4. Memo 中必须附**承租资产清单表**：资产类型 / 出租方 / 出租方权属情况 / 租赁期限 /
   租金 / 是否备案 / 是否关联交易

## 与邻近 DD skill 的边界

| 场景 | 归属 skill | 分工说明 |
|------|-----------|---------|
| 房产 / 土地 / IP / 设备权属 | **本 skill** | 本 skill 的核心 |
| 资产抵押 / 质押背后的借款合同 | `ecm-dd-debt` | 本 skill 只记录权利受限事实；对应的主债权归 debt |
| 资产相关的关联交易（租赁给关联方等） | `ecm-dd-related-party` | 本 skill 记录租赁关系；关联属性归 related-party |
| 资产支撑的主营业务是否合规 | `ecm-dd-business` | 本 skill 关注权属；业务合法性归 business |
| 资产相关的环评 / 消防 / 安监 | `ecm-dd-environment`（BATCH-04）/ `ecm-dd-compliance` | 本 skill 只关注权属 |
| 权属纠纷诉讼 | `ecm-dd-litigation`（BATCH-04）| 本 skill 记录纠纷对权属的影响；诉讼本体归 litigation |
