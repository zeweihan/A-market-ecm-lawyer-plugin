# ecm-dd-related-party

## 定位与边界

本 skill **负责**：
- 关联方范围的完整性核查（控股股东 / 实控人及其近亲属、关联自然人、关联法人、主要股东、
  董监高及其近亲属、曾经的关联方等）
- 报告期内关联交易的识别、分类（经常性 / 偶发性 / 日常 / 非日常）、必要性、公允性
- 关联交易决策程序合规性（关联董事回避、关联股东回避、独立董事事前认可、占比触发披露阈值）
- 关联交易披露充分性（招股书 / 定期报告 / 临时公告）
- 同业竞争识别（与控股股东、实控人及其控制的其他主体从事相同或相似业务）
- 同业竞争解决方案（注销 / 转让 / 托管 / 业务划分 / 承诺函）的有效性核查
- 避免同业竞争与避免非必要关联交易承诺函的规范性核查
- 出具编报规则第 8 章对应的 DD Memo

本 skill **不负责**：
- 股东、实控人本身的身份穿透和出资来源（归 `ecm-dd-shareholders`）
- 董监高的任职资格、对外兼职（归 `ecm-dd-directors`，但兼职结果可能导致关联关系）
- 关联资金拆借 / 担保合同的条款（归 `ecm-dd-debt`；本 skill 只关注其关联属性和决策程序）
- 业务合同本身的商业条款（归 `ecm-dd-business`；本 skill 只关注其关联属性）
- 五独立中的业务 / 财务 / 人员 / 机构独立性结论（归 `ecm-dd-independence`）
- 关联交易的税务处理（归 `ecm-dd-tax`）
- 文件读取 / 搬运（归 `ecm-dd-file-review` / `ecm-setup-file-organize`）

## 资深律师执行标准

执行本 skill 时，必须同时遵循 [senior-lawyer-execution-standards.md](../../shared/templates/senior-lawyer-execution-standards.md)。本 skill 的任何输出不得突破四条底线：事实可追溯、法源可核验、风险可分级、建议可落地；无法核验时必须显式标注。

## 本 skill 的实务加固点

- **关联方全口径**：按公司法、会计准则、上市规则和实质重于形式口径合并识别，不只依赖客户清单。
- **交易必要性**：关联交易必须核查商业必要性、定价公允性、决策程序、回避表决和期后规范措施。
- **同业竞争实质判断**：从产品/服务、客户供应商、地域、技术路线、替代性和未来业务规划判断竞争关系。
- **高风险触发器**：资金占用、违规担保、重大未披露关联方、同业竞争未解决、利益输送，应列高风险。

## 核心工作流（五步）

1. **生成关联方清单**：依据 `ecm-dd-shareholders` 的实控人穿透图、`ecm-dd-directors` 的
   董监高花名册，形成**关联方清单表**（详见输出格式契约的额外要求）
2. **生成核查要点清单**：照 [references/checklist.md](./references/checklist.md) 输出本项目的核查清单
3. **文件审阅与标注**：对照关联方清单逐一审阅 `02-07-关联交易与同业竞争/` 目录下的关联交易
   协议、决策文件、审计报告关联方附注、同业竞争核查文件；必要时调用
   `ecm-dd-data-verify` 对企查查数据做关联关系交叉验证
4. **风险分级**：按下方口径给每个问题打高 / 中 / 低
5. **结论与建议**：汇总关联交易合规性和同业竞争状况，列明待补正事项和建议措施

## 统一风险分级口径

| 级别 | 定义 | 本 skill 场景的典型例子 |
|------|------|------------------------|
| **高（红）** | 足以构成发行 / 交易的实质性障碍 | 关联方遗漏导致重大非公允关联交易未披露；存在实质性同业竞争且无有效解决方案；非经营性资金占用 / 违规担保未清理；关联交易占比过高构成重大依赖（报告期内日常关联销售 / 采购合计占比通常不宜超过 30%，需结合行业特性评价） |
| **中（黄）** | 程序或文件存在瑕疵但可补正 | 关联交易定价公允性依据不足但可补充第三方参考价 / 评估报告；同业竞争已通过承诺规范但尚未完全消除；关联董事未回避但事后补决议；独立董事事前认可意见未出具但事后出具补充意见 |
| **低（蓝）** | 不影响实质的形式瑕疵 | 关联方清单格式不统一；关联交易协议缺少部分附件；披露文本表述不严谨；承诺函格式问题 |

## 输出格式契约（强制）

严格遵循 [shared/templates/dd-skill-template.md](../../shared/templates/dd-skill-template.md) 的
"输出格式契约"五段式结构：

1. 一级标题：`# DD Memo：关联交易和同业竞争`
2. 二级标题：`一、核查要点清单` / `二、审阅发现` / `三、风险分级汇总` / `四、结论与建议` / `五、参考资料`
3. 风险等级列取值限"高 / 中 / 低"
4. 输出落地：`02-尽职调查/02-07-关联交易与同业竞争/DD-Memo-关联交易与同业竞争-{YYYYMMDD}.md`

**额外要求**（本 skill 专属）：

1. Memo 中必须附**关联方清单表**，字段：关联方名称 / 关联类型（控股股东 / 实控人 /
   近亲属 / 关联法人 / 董监高本人 / 董监高近亲属 / 过去 12 个月内曾为关联方 / 其他）/
   认定依据 / 与发行人关联关系 / 是否从事相同或相似业务 / 是否已签署承诺函
2. Memo 中必须附**关联交易汇总表**，按报告期逐年列示，字段：关联方 / 交易类型（销售 / 采购 /
   服务 / 租赁 / 资金 / 担保 / 其他）/ 金额 / 占同类交易总额比例 / 占营业收入或总资产比例 /
   定价依据 / 决策程序（谁批准 + 关联方是否回避）/ 是否公允
3. Memo 中必须附**同业竞争情况表**：控股股东 / 实控人控制的其他主体清单及其业务 / 与发行人
   业务的关系（相同 / 相似 / 无竞争）/ 解决方案（若有竞争）

## 与邻近 DD skill 的边界

| 场景 | 归属 skill | 分工说明 |
|------|-----------|---------|
| 关联方是谁 | **本 skill** | 本 skill 的核心输入 |
