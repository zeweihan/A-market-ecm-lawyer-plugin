# 法律意见书 — 常见错误库（Common Error Catalog）

本清单汇集 A 股证券律师意见书在律所内核、保荐人审阅、监管反馈环节反复出现的高频错误模式。审查时按本清单逐条扫描。

---

## A 类 — 基础信息类（事实错，最高频）

| 编号 | 典型错误 | 检查方法 |
|---|---|---|
| A1 | **客户简称**前后不一致（封面 / 释义 / 正文 / 签字页） | 全文 grep 简称 |
| A2 | **律所简称 / 全称**不一致 | 全文 grep 律所名 |
| A3 | **出具日期**三处（封面 / 声明 / 签字页）不一致 | 三处对照 |
| A4 | **律所文号**不符合命名规则（应为 XX 律字[YYYY]第 XXX 号 / XX-1 号） | 核对封面文号 |
| A5 | **项目名称**（如"首次公开发行 A 股股票并在深交所创业板上市"）跨段不一致 | 全文 grep 项目名 |
| A6 | **报告期**（如"2022 年度、2023 年度、2024 年度及 2025 年 1-6 月"）跨段不一致 | 全文 grep 期间 |

**批注模板**：
> 【必改】{字段} 在意见书 {位置 1} 为 "…"，在 {位置 2} 为 "…"，请统一。

---

## B 类 — 五段骨架结构类

| 编号 | 典型错误 |
|---|---|
| B1 | 缺"二、释义"段 |
| B2 | "四、结论性意见" 被合并到正文最后一章里 |
| B3 | "五、特别事项提示" 整段缺失（即使工作报告有"高"级风险） |
| B4 | 一级标题编号错误（"二、释义" 被误写为"二 释义" 或 "2、释义"） |
| B5 | 章节顺序颠倒 |

---

## C 类 — 三步法结构类（每章正文）

| 编号 | 典型错误 |
|---|---|
| C1 | 章节内三级标题缺失（没有"1、事实陈述" / "2、核查工作" / "3、法律意见"） |
| C2 | 三级标题命名不统一（如写"1、情况介绍" / "（一）核查情况" / "（二）律师意见"） |
| C3 | 三步顺序颠倒（先意见再事实） |
| C4 | "2、核查工作"段只写"本所律师核查了……"一句，无具体核查资料范围 |
| C5 | "3、法律意见"段无法规引用（空口定论） |

---

## D 类 — 结论口径类（超出工作报告认定范围）

| 编号 | 典型错误 |
|---|---|
| D1 | 工作报告识别了"高"级风险（如控股股东曾受行政处罚），意见书结论为"合法有效" |
| D2 | 意见书使用"基本符合"、"大致合规"等含糊措辞（223 号令 §23 禁止） |
| D3 | 意见书对某事项发表结论，但 DD Memo 未核查过（意见书脱离底稿发表意见） |
| D4 | 意见书独立性结论比 `ecm-dd-independence` Memo 的结论更乐观 |
| D5 | 意见书结论句只说"发行人本次发行符合《xx 法》"，未说明是否符合发行人相关条件 |

---

## E 类 — 特别事项提示类

| 编号 | 典型错误 |
|---|---|
| E1 | 未设"五、特别事项提示"段，但 DD Memo 有"高"级风险 |
| E2 | "五、特别事项提示"段存在但事项不完整（遗漏部分 DD "高"级风险） |
| E3 | 特别事项措辞与 DD Memo 原文差别过大（dd-output-schema §2.3 要求原样引用） |
| E4 | 特别事项"本所意见"段为空或写"按现行法规处理" |
| E5 | 特别事项之间逻辑冲突（如事项 1 说"已整改"，事项 2 说"该瑕疵构成障碍"） |

---

## F 类 — 释义类

| 编号 | 典型错误 |
|---|---|
| F1 | 释义段少于 15 条 |
| F2 | 正文使用缩写未在释义段定义（"中国证监会"未定义即直接使用） |
| F3 | 释义与工作报告冲突 |
| F4 | 释义之间相互冲突（"本次发行"在两处定义不同） |
| F5 | 释义段有已废除的术语（"本所前身 XX 律师事务所"与本次意见书无关） |

---

## G 类 — 签字页类

| 编号 | 典型错误 |
|---|---|
| G1 | 经办律师 < 2 人（违反 223 号令 §26） |
| G2 | 律所负责人签字缺失 |
| G3 | 律所盖章位置错误（未盖在律所全称下方） |
| G4 | 签字律师未标注执业证号 |
| G5 | 签字律师姓名与引言段"经办律师简况"不一致（高频错） |

---

## H 类 — 法规引用类

| 编号 | 典型错误 |
|---|---|
| H1 | 《公司法》版本滞后（引用 2018 修订或 2013 修订，应为 2023 修订） |
| H2 | 《股东大会规则》未更新为《股东会规则》（2025 修订后已改名） |
| H3 | 《首次公开发行股票并上市管理办法》（已废止）引用为现行 —— 应改为《首次公开发行股票注册管理办法》（2023） |
| H4 | 项目类型与法规清单不匹配（如 IPO 意见书引用了《上市公司重大资产重组管理办法》却未引用《注册管理办法》） |
| H5 | 《交易所股票上市规则》对应板块错（如深交所主板公司引用了科创板规则） |

---

## I 类 — 模板残留类

| 编号 | 典型错误 |
|---|---|
| I1 | 涉及上市公司的话术残留在非上市公司意见书里（"上市公司"、"交易所"、"中小投资者单独计票") |
| I2 | 涉及 IPO 的话术残留在再融资意见书里（"首次公开发行"、"拟上市板块") |
| I3 | "{xxx}" 占位符未替换（例 "截至 {日期}"、"{律所名}"） |
| I4 | 前次项目的客户名未替换彻底（意见书主体是 A 公司，但个别段落仍提 B 公司） |
| I5 | 意见书类型与正文不匹配（封面标"补充法律意见书"但正文按正式稿写） |

---

## 执行建议

内核审查时，建议按 "A → B → C → D → E → F → G → H → I" 顺序扫一遍，每类用关键字 grep 定位典型错误点。**A 类 / D 类 / E 类**是含金量最高的三类，优先保证这三类覆盖完全。
