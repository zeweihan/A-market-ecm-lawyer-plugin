# 法律意见书实质审查清单

实质审查 = 在形式合规的前提下，审查意见书的"实质内容"是否经得起监管 / 内核推敲。共 **8 类审查项**，每类按"核查要点 / 高频问题 / 批注模板"组织。

---

## 类别 1 — 正文各章意见结构完整性

**核查要点**：
- 每章"3、法律意见"段是否发表**明确结论**（合法有效 / 有条件 / 保留）
- 每章是否引用**对应法规条文**（空口定论不接受）
- 每章结论是否与该章"1、事实陈述"和"2、核查工作"段内容形成闭环（事实-核查-意见三步法）

**高频问题**：
- 第 X 章"3、法律意见"段只有"本所认为，上述事项合法有效"一句，未引用法规
- 事实陈述段描述了某瑕疵（如主体资格期间某出资未到位），核查工作段写"已核查"，但法律意见段只说"合法有效"，不提瑕疵如何处理
- 合并章节处理（如将"批准和授权"与"主体资格"合并）但未保留各章独立结论

**批注模板**：
> 【建议】本章 "3、法律意见" 段结论句仅 {结论原文}，建议引用支撑该结论的具体法规条文（如"符合《公司法》（2023 年修订）第 XX 条的规定"），便于监管和未来诉讼场景的证据链。
>
> 【必改】本章"1、事实陈述"段载明"{瑕疵描述}"，但"3、法律意见"段未针对该瑕疵发表意见，建议补充 "该瑕疵 {已 / 未} 经 {整改程序}，本所认为 {结论}"。

---

## 类别 2 — 结论三级措辞规范

**核查要点**：
每章 / 整体结论必须归入三级之一，且措辞规范：

1. **肯定（绿）** — `本所认为，{事项} 合法有效，符合 {法规} 的规定`
2. **有条件肯定（黄）** — `除 {瑕疵} 外，本所认为 {事项} 合法有效`；或 `{事项} 履行了 {程序}，但 {瑕疵}，建议 {整改}`
3. **保留 / 限定（红）** — `本所对 {事项} 保留意见，理由是 {具体原因}`

**判定规则**（按 dd-output-schema 风险级别）：
- 该章对应 DD Memo 无 `级别=高` 事项 → 可用"肯定"
- 有 `级别=中` 事项 → 宜用"有条件肯定"
- 有 `级别=高` 事项 → 必须用"保留"或"限定"（如用"肯定"则违反 `legal-opinion-format §一`，走【必改】）

**批注模板**：
> 【必改】本章结论为"本所认为，……合法有效"，但经核对同项目 DD-Memo-{章节}-{YYYYMMDD}.md 的"三、风险分级汇总"，存在 `级别=高` 事项 "{事项描述}"。按 `legal-opinion-format §一`，意见书结论不得超出工作报告认定范围；请调整为"除 {瑕疵} 外……" 或改为保留意见，或在"五、特别事项提示"段补充披露。

---

## 类别 3 — 法规引用版本与条号

**核查要点**：
- 法规版本是否现行有效
- 条号是否准确
- 条号引用方式是否一致（《公司法》第 46 条 vs 《公司法》第 XLVI 条；阿拉伯 vs 中文）

**高频问题**：
- 2024 年 7 月 1 日后引用《公司法》老版（1993 / 2005 / 2013 / 2018）
- 《首次公开发行股票并上市管理办法》（已废止）引用为现行 —— 应改为《首次公开发行股票注册管理办法》
- 《上市公司章程指引》版本未注明（2023 年最新修订）
- 《股东大会规则》未更新为《股东会规则》

**批注模板**：
> 【必改】本章引用 "{法规原名} 第 {X} 条"。现行有效版本为 "{法规新名 / 新版}"；条号可能随版本更新变化，请核对并更新至现行版本。

---

## 类别 4 — 释义闭环性

**核查要点**：
- 正文每个缩写 / 专有名词是否在"二、释义"段定义
- 释义段定义的术语是否在正文中使用（未使用的删除 —— 工作报告遗留的术语）
- 释义之间是否存在冲突（如"报告期"在释义 A 定义为"2023-2025"，释义 B 定义为"最近三年")

**批注模板**：
> 【必改】正文第 {N} 部分使用了缩写"{缩写}"，但"二、释义"段未定义该缩写。请补充释义条目或改为全称。

---

## 类别 5 — 保留意见情形规范

**核查要点**（223 号令 §24）：保留意见应说明：
- 具体情形（哪些事项 / 哪些文件 / 哪些时间段）
- 影响程度（对发行条件是否构成实质障碍）
- 风险提示（是否可能导致监管问询 / 行政处罚）

**高频问题**：
- 保留意见只说"本所对 xx 保留意见"但未说明影响程度
- 保留意见与"特别事项提示"重叠但未相互引用
- 保留意见措辞模糊（"存在 xx 不确定性"而不明确保留范围）

**批注模板**：
> 【必改】本所对 "{事项}" 保留意见的表述未说明 {影响程度 / 风险提示}，请按《律师事务所从事证券法律业务管理办法》第 24 条补充 "对本次发行构成 {实质 / 非实质} 障碍 / 可能导致 {具体风险}" 等要素。

---

## 类别 6 — 项目类型对应法规清单

**核查要点**：意见书类型不同，法规清单需调整：

| 项目类型 | 必引法规（除《公司法》《证券法》外） |
|---------|-----------------------------|
| IPO（首次公开发行） | 《首次公开发行股票注册管理办法》（2023）《深圳 / 上海 / 北京证券交易所股票上市规则》对应板块版本 |
| 再融资（定增 / 配股 / 可转债） | 《上市公司证券发行注册管理办法》（2023）《可转换公司债券管理办法》（如涉及） |
| 重大资产重组 | 《上市公司重大资产重组管理办法》（2023）《上市公司收购管理办法》（如触发要约） |
| 股份回购 | 《上市公司股份回购规则》（2022） |
| 新三板挂牌 | 《全国中小企业股份转让系统挂牌办法》《公司章程指引》 |
| 并购（收购 / 要约） | 《上市公司收购管理办法》（2020 修订） |

**批注模板**：
> 【必改】本意见书类型为 {项目类型}，编制依据缺失 "{法规名}"。请按 `shared/templates/legal-opinion-format §六` 的项目类型法规清单补齐。

## 类别 7 — 意见书与工作报告的结论口径

（详见 cross-check-matrix §五 J1-J4；本清单不重复）

核心要求：意见书结论 ≤ 工作报告认定范围。

---

## 类别 8 — 补充法律意见书的 "承上启下" 要求

适用于第二次 / 第三次 / 第 N 次补充意见书。

**核查要点**：
- 首段明确引用原意见书（律所文号 + 出具日期）
- 说明本次补充的原因（回复监管反馈 / 补充核查 / 变更事项）
- 重申或修正原意见书的结论
- 释义段沿用原意见书（如未变化，可引用"同原法律意见书")
- 签字律师尽量与原意见书一致（如变化，需说明原因）

**高频问题**：
- 补充意见书完全不提原意见书，像独立意见书
- 文号未使用 "XX-1 / XX-2 号" 规则
- 结论与原意见书冲突但未说明修正理由

**批注模板**：
> 【必改】本意见书系对 {原意见书文号} 的补充，但首段未引用原意见书、未说明补充原因。请按 `legal-opinion-format §二` 补充意见书格式补充承上启下段。

---

## 实质审查的执行流程

建议按 "类别 1 → 类别 2 → 类别 3 → 类别 4 → 类别 8（如适用） → 类别 5（如适用） → 类别 6 → 类别 7" 顺序过一遍。类别 7 依赖工作报告参考源，若 Step 1.5 未取得，跳过类别 7。
