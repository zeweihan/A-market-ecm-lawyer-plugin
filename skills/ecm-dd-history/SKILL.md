# ecm-dd-history

## 定位与边界

本 skill **负责**：
- 按时间线复原发行人自设立以来的全部股本变动（增资 / 减资 / 股权转让 / 股份回购 / 员工持股 /
  股权激励）
- 每次变动的四要素核查：决策程序 / 定价公允性 / 款项支付 / 工商变更（与税务处理）
- 涉及国有股权的国资审批、评估和进场交易合规
- 涉及外资股权的外汇登记和审批合规
- 历次代持情形的排查与清理核查
- 历史出资瑕疵的识别和补正方案

本 skill **不负责**：
- 首次设立时的出资（归 `ecm-dd-establishment`）
- 现行股东身份和实控人认定（归 `ecm-dd-shareholders`）
- 现行章程条款核查（归 `ecm-dd-charter`）
- 股权激励计划的税务和会计处理（仅标记是否合规；深核归 `ecm-dd-tax`）

## 本 skill 的实务加固点

- **逐次变动台账**：历次增资、减资、转让、股改、股份拆细、代持还原必须形成时间轴和证据索引。
- **价款闭环**：股权转让和增资必须核对协议、决议、付款凭证、完税/税务说明和工商登记。
- **高风险触发器**：代持未还原、权属争议、瑕疵出资未补正、股份支付或突击入股解释不足，应列高风险。
- **申报敏感期**：报告期内及申报前一年股权变动需单列监管关注点和披露建议。

## 核心工作流（四步）

1. **生成核查要点清单 + 股本演变时间线**：输出一张贯穿全部股本变动的时间线表
2. **文件审阅与标注**（每一笔变动都按"决议 / 协议 / 定价 / 付款 / 工商 / 税务"6 列核对）
3. **风险分级**
4. **结论与建议**

## 统一风险分级口径

| 级别 | 定义 | 本 skill 场景的典型例子 |
|------|------|------------------------|
| **高（红）** | 实质性障碍 | 未经批准的国有股权转让、国有资产流失嫌疑、股转款项未实际支付、严重低价 / 无偿转让且无合理解释、股权代持未清理、工商记载与实际股权严重不符 |
| **中（黄）** | 瑕疵但可补正 | 股转定价偏低但有合理解释（亲属 / 同事 / 激励等）且已补缴税款、验资报告形式瑕疵、工商变更晚于决议时间较多、内外资切换未及时办外汇登记 |
| **低（蓝）** | 形式瑕疵 | 历史文件签字页缺失、协议版本多版混用、转让方签章字体与身份证不完全一致 |
