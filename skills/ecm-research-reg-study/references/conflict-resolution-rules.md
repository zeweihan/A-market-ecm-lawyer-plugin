# 法规冲突解决规则库（Conflict Resolution Rules）

本文件列举 `ecm-research-reg-study` 在 Phase 4（冲突识别与解决）时使用的**标准冲突解决
规则**，按冲突类型分类。

---

## 一、总体冲突解决原则

中国法律规范冲突解决的基本规则：

1. **上位法优先**：高位阶规范优于低位阶规范
2. **新法优先**：同位阶同主题，后发布的规范优于先发布的规范
3. **特别法优先**：同位阶不同主题（一般/特别），特别规范优于一般规范
4. **经济特区立法 / 民族自治条例**：在特定地域可优先于一般法律（ECM 实务较少涉及）

当同时存在多个原则冲突时的判断顺序（从强到弱）：
```
上位法优先 > 新法优先 > 特别法优先
```

即：**上位法优先高于新/特别规则**；先看位阶，再看新旧/特别。

---

## 二、具体冲突类型及解决规则

### 2.1 上位法 vs 下位法

**冲突表现**：下位规范的规定与上位规范不一致。

**解决规则**：

| 情形 | 解决 |
|------|------|
| 下位法**减损**上位法权利 | 下位法该条无效 |
| 下位法**增加**上位法义务 | 若超出上位法授权范围，下位法该条无效；若在授权范围内细化，合法 |
| 下位法**细化**上位法（更具操作性） | 合法，累积适用 |
| 下位法**创设**上位法未规定的新制度 | 视是否有明确授权 —— 有授权合法，无授权无效 |

**实务判断要点**：
- 看上位法是否有明确的**授权条款**（如"具体办法由国务院证券监督管理机构制定"）
- 看下位法是否在**授权范围内**
- 看下位法是否**必要**（只是对上位法已有规定的重复，尚非超越）

**ECM 实例**：
- 《证券法》规定证券发行实行注册制 → 《IPO 注册管理办法》具体化为申请材料、审核程序、
  发行条件等。属于合法细化。
- 若某部委规章规定 IPO 申请人必须具备"超出证券法条件"的某项资质，需审查证监会有否授权。

### 2.2 新法 vs 旧法（同一位阶同一主题）

**冲突表现**：旧法和新法的规定不同。

**解决规则**：

| 情形 | 解决 |
|------|------|
| 新法**明文废止**旧法 | 全部按新法 |
| 新法**隐含废止**旧法（未明文但实质冲突） | 实质冲突部分按新法；其余旧法保留 |
| 新法**部分修订**旧法 | 修订部分按新法；未修订部分按旧法原文 |
| 新法**变更适用条件** | 过渡期问题 → 见 transition-analysis-framework.md |

**实务判断要点**：
- 先查新法的**附则**/**废止条款**
- 若未明文废止但存在冲突，按"新法优于旧法"
- 旧法中的条款只要未明文废止且无实质冲突，**仍可适用**（即使新法整体替代了旧法）

**ECM 实例**：
- 《IPO 注册管理办法》（205 号，2023）第 XX 条规定"本办法施行之日起，《首次公开发行股票
  并上市管理办法》（证监会令第 32 号）同时废止"—— 明文废止，全部按新法。
- 《上市公司股东会规则》（2025）对原 2022 版的修订，若修订部分和未修订部分并存，修订部分
  按 2025 版，其余按原版（若整体重发，则全按新版）。

### 2.3 特别法 vs 一般法

**冲突表现**：同位阶两部规范对同一事项的规定不同，一部是针对该事项的专门规范，另一部是
一般性规范。

**解决规则**：特别法优于一般法。

**识别要点**：
- **事项范围更窄** = 特别法
- **主体范围更窄** = 特别法
- **程序特定** = 特别法

**ECM 实例**：
- 《公司法》（一般）vs 《证券法》（对上市公司更专门）→ 涉及上市公司时，证券法优先
- 《证券法》（一般）vs 《上市公司证券发行注册管理办法》（针对上市公司证券发行的特别规定）
  → 涉及再融资时，注册管理办法优先
- 《上市公司重大资产重组管理办法》（一般）vs 《境内企业境外发行证券和上市管理试行办法》
  （针对境外上市的特别规定）→ 涉及境外上市时，207 号办法优先

### 2.4 部门规章之间的冲突

**冲突表现**：两个部委对同一事项的规定不同。

**解决规则**：
- 由**国务院裁决**（《立法法》第 95 条）
- 实务中律师判断方法：
  1. 看两部委的**职权划分**：哪个部委是该事项的主管机关？→ 主管机关的规章通常优先
  2. 看**行业监管边界**：证券事项以证监会为准；外汇事项以外汇局为准；会计事项以财政部为准

**ECM 实例**：
- 证监会规章 vs 国资委规章：涉及**上市公司国有股权管理**时，两者有联合规章（36 号令），
  实务中由联合规章统一口径
- 证监会规章 vs 财政部规章：会计处理按财政部会计准则；披露按证监会信息披露规则

### 2.5 规范性文件加码部门规章

**冲突表现**：规范性文件（指引、通知）对部门规章提出更严要求。

**解决规则**：
- 若规范性文件在部门规章的**授权范围内细化** → 合法，累积适用
- 若规范性文件**突破**部门规章（明显超出）→ 以部门规章为准，规范性文件超出部分无效

**判断要点**：
- 多数情况下，证监会规范性文件属于合法细化（监管机关对本机关规章的解释、适用指引）
- 极少数情况下，规范性文件可能超越规章——此时需仔细分析

**ECM 实例**：
- 《监管规则适用指引——发行类第 4 号》对《205 号办法》第 13 条的细化 → 合法加码
- 某些窗口指导口径要求超出规章 → 属于非正式监管口径，法律层面不具强制力，但事实上影响审核

### 2.6 交易所规则严于部委规章（自律加码）

**冲突表现**：交易所规则对某项要求严于证监会规章。

**解决规则**：
- 原则：部门规章优先
- 但：交易所规则属于**自律加码**，市场主体必须遵守（否则会受自律处分）
- **实务结果**：市场主体应**同时满足**规章和规则，即"从严适用"

**法律救济路径**：
- 市场主体违反规章：行政处罚
- 市场主体违反规则：自律处分（不是行政处罚）
- 市场主体对规章的合法性有异议：行政复议、行政诉讼
- 市场主体对规则的合法性有异议：法律救济路径复杂（证券法修订后有所加强）

**ECM 实例**：
- 证监会某规章对信息披露频率的要求为"每半年" → 上交所《自律监管指引》要求"每季度" → 
  市场主体必须按每季度披露（从严）

### 2.7 司法解释 vs 部门规章

**冲突表现**：最高法司法解释对某条款定性为无效或作不同于部门规章的解释。

**解决规则**：
- 对法院：**按司法解释**（法院裁判权限范围内）
- 对行政机关：理论上按部门规章执法，但**实务中法院判决会影响监管口径**——部委通常会
  调整解释或修订规章
- 对当事人：若涉及诉讼，按司法解释；若涉及监管审核，按监管口径

**ECM 实例**：
- 最高法"九民纪要"对对赌协议效力的认定（与公司对赌的效力限制） → 法院裁判遵循；监管审核
  口径同步调整为"IPO 申报前对赌必须清理"

### 2.8 国际条约 vs 国内法

**冲突表现**：中国签订的国际条约与国内法规定不一致。

**解决规则**：
- 原则：国际条约优先（相同事项有不同规定时适用条约）
- 例外：中国声明保留的条款按国内法
- ECM 实务中较少涉及，主要在跨境并购、涉外合同中出现

## 三、冲突解决的输出模板

对每组识别的冲突，按以下模板输出：

```markdown
### 冲突组 N：<简短描述>

**涉及规范**：
- [L-0X] 《规范 A》第 X 条：<规定 A>
- [L-0Y] 《规范 B》第 Y 条：<规定 B>

**冲突性质**：<上位/下位 | 新/旧 | 一般/特别 | 规范加码 | 交易所加码 | 司法解释与规章>

**适用规则**：<按本文件第 2 节对应条目引用的解决规则>

**解决结论**：
<在本项目场景下具体适用哪部规范，是否累积适用，是否排他适用>

**实务建议**：
<基于本结论对本项目应采取的具体动作>

**论据**：
- 法理依据：<引用《立法法》第 XX 条 或 通说>
- 案例支撑：[C-0X] （如有）
```

## 四、冲突识别的常见盲点

### 盲点 1：只看"明文"不看"实质"

- 新法没明文废止旧法，但旧法的某些条款已不可能适用
- 应识别为"事实失效" → 参考 `ecm-research-reg-search/references/effect-status-rules.md`

### 盲点 2：未识别"同主题"

- 两部法规表面不同主题，但实质对同一事实的调整
- 应同时列出，分析是否真冲突

### 盲点 3：忽视授权条款

- 部门规章细化上位法 → 看是否在授权范围内
- 不看授权就断言"越权无效"是错误的

### 盲点 4：忽视过渡条款

- 新法常有过渡条款（"本办法施行前已受理的……"）
- 忽视过渡条款会导致对"本项目是否跨越施行日"的误判

### 盲点 5：把监管实践混为法规

- 监管实践（窗口指导、答复）不是法规
- 冲突分析只应针对**规范层面**，不应把"监管口径"作为独立一方纳入冲突对比

## 五、当冲突结论不确定时

若经分析仍无法确定冲突解决路径：

1. **两种可能并列列出**，由律师据项目具体事实选择
2. 建议调用 `ecm-research-case-search` 寻找**最近的监管/裁判先例**作为指导
3. 建议与监管机关**预沟通**
4. **绝不**给出武断结论

输出示例：

```markdown
### 冲突组 1：205 号办法第 13 条与指引第 4 号的细化要求

**两种理解并存**：
- 理解 A：指引属于合法细化，累积适用（通说）
- 理解 B：指引某条款超出规章授权，规章优先（少数观点）

**建议**：在意见书中按理解 A 为主，同时披露理解 B 的存在和潜在风险。调用 
`ecm-research-case-search` 查最近 2 年监管口径是否有明确表态。
```
