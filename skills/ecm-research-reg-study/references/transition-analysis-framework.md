# 新旧法衔接分析框架（Transition Analysis Framework）

本文件为 `ecm-research-reg-study` 在涉及**法规修订导致的过渡**问题时提供分析框架。

---

## 一、新旧衔接的基本原则

### 1.1 法律不溯及既往

**基本原则**：新法不适用于其施行前已经发生的法律事实，除非：
- 新法明文规定溯及
- 新法规定的程序性或有利于当事人的条款可以溯及

**ECM 实务中的应用**：
- 新法通常不影响**新法施行前已完成**的交易行为
- 新法通常适用于**新法施行后**的交易行为
- **跨越施行日**的行为（签约在前、履行在后）最复杂，需具体分析

### 1.2 过渡条款优先

若新法明文规定过渡条款（"本办法施行前已受理的 XXX 案件按原办法处理"），**以过渡条款
为准**。

### 1.3 程序优于实体

程序性规定一般适用于施行后的**程序行为**（即使该程序涉及的实体争议发生在施行前）。
- 例如：诉讼法的新程序规定适用于施行后的诉讼，即使案涉事实发生在施行前
- ECM 实例：新审核程序规定适用于施行后开始的审核流程，即使申报材料是旧法时期准备的

## 二、过渡条款的四种常见形式

### 2.1 "施行前已受理/已批准的案件按原规定"

典型表述：
> "本办法施行前，已经受理但尚未完成审核的 XXX 案件，按照原规定办理。"

**处理原则**：以**受理日期**为分界点
- 受理在新法前 → 原规定
- 受理在新法后 → 新规定

### 2.2 "施行前已发生的行为不溯及"

典型表述：
> "本办法施行前已经发生的 XXX 行为，不因本办法的施行而改变其效力。"

**处理原则**：以**行为发生日期**为分界点

### 2.3 "老办法和新办法双轨运行一段时间"

典型表述：
> "本办法施行之日起 X 个月内，发行人可以选择按本办法或原办法办理。X 个月后，全部按本办法。"

**处理原则**：窗口期内由当事人**选择**，并应在申请中明示选择

### 2.4 "行为须在 X 日内整改"

典型表述：
> "本办法施行前已存在的 XXX 行为，发行人应当在本办法施行后 X 个月内完成整改。"

**处理原则**：给出**整改期**，期间内老行为不被追究，过期后按新法。

## 三、跨越施行日的分析框架

研究场景：某项行为的**启动**在新法前，**完成/效力展现**在新法后。

### 3.1 分析步骤

**第一步**：确定关键时间节点

```
│────────│─────── 新法施行日 ──────│────────│
           │
        行为节点 1：意向书签订
                 │
              行为节点 2：申报材料递交
                       │
                    行为节点 3：审核
                             │
                           审议通过
                                  │
                              注册生效
                                        │
                                     发行实施
```

**第二步**：针对每个节点，判断适用法律：
- 在新法前的节点 → 一般按旧法
- 在新法后的节点 → 一般按新法
- 跨越节点 → 按过渡条款；无过渡条款则按"行为完成地点"判断

**第三步**：识别**关键节点**（决定整体适用法律的那个节点）：
- IPO：通常以**受理日期**为关键节点
- 再融资：通常以**董事会/股东会决议日**为关键节点
- 重大资产重组：通常以**首次披露日**为关键节点（证监会专门对 2023 修订的重组办法有过渡
  安排）
- 合同行为：通常以**合同签订日**为关键节点
- 诉讼：通常以**起诉日**或**一审开庭日**为关键节点

### 3.2 典型场景分析

**场景 A：意向书签署在前，申报在后**

```
2024-05：签署 IPO 意向协议
2024-07-01：新公司法施行
2024-09：递交 IPO 申请
```

- 意向协议的效力：以 2024-05 法律为准（旧公司法）
- IPO 申请的审核：以 2024-09 法律为准（新公司法）
- 冲突点：意向协议中的**公司治理条款**可能基于旧公司法，但 IPO 申报时需符合新公司法
  → 发行人应**修订意向协议**或**出具补充承诺**使其符合新法

**场景 B：受理在前，审议在后**

```
2024-03：IPO 申请受理
2024-07-01：新证券法修订版施行（假设）
2024-11：上市委审议
```

- **关键节点**：通常监管机关有明示过渡规则
- 若过渡条款规定"已受理的按原规定审核" → 全过程按旧法
- 若没有过渡 → 一般按**审议时的新法**（对程序性要求）+ 按**受理时的旧法**（对实体要求）

**场景 C：完成后遭遇新法**

```
2024-05：IPO 成功发行、注册完成
2024-07-01：新法施行
```

- 原行为不溯及：按旧法成立的发行合法有效
- 但**持续合规义务**可能按新法：如新法规定的持续披露要求，适用于新法施行后的披露期间

**场景 D：跨越多部法规修订**

```
2023-02-17：证监会 205 号《IPO 注册管理办法》施行
2024-07-01：新公司法施行
2026-M-D：某 IPO 受理
```

- **现时适用**：2026-M-D 之后，按 205 号办法 + 新公司法 同时适用
- 无新旧衔接问题（两部新法早已生效）

## 四、常见的新旧衔接场景清单

### 4.1 2023-02-17 注册制全面改革的衔接

**改革内容**：证监会发布 205 号、206 号、207 号、208 号等注册制配套办法，并同步调整重组监管规则，全面替代旧制度。

**过渡安排**：
- 已受理的 IPO 在改革后切换为注册制程序，但部分实体要求仍按受理时的旧法
- 已公告的重大资产重组继续按原重组办法完成

**现状**：2026 年的项目绝大多数已在新制度下运行，衔接问题主要涉及：
- 历史上**在旧制度下完成**的事项的合规性评估（例如回顾性分析历史上的发行）
- 跨制度的**长链路交易**（若 2022 年启动但尚未完成的极少数案例）

### 4.2 2024-07-01 新公司法施行

**主要变化**：
- 股东出资制度变化（实缴期限、出资方式）
- 董事会、监事会结构变化（可不设监事会改设审计委员会）
- 公司治理要求加强（董事忠实勤勉义务、股东权利保护）

**对 ECM 的影响**：
- 申报材料中关于公司治理的**章程、三会制度**需符合新公司法
- 历史沿革中**出资未按新法期限实缴**的情况需整改或说明
- 董监高义务、实控人认定可能受影响

**衔接要点**：
- 2024-07-01 之前设立的公司的章程，需在**合理期限内**按新公司法修订（具体期限见新公司
  法附则和相关通知）
- 原章程中与新法冲突的条款，自 2024-07-01 起失效；需及时修订

### 4.3 上市公司持续规则的修订

如《上市公司信息披露管理办法》《上市公司股东会规则》《上市公司治理准则》《上市公司章程指引》等定期修订。

**衔接要点**：
- 披露规则修订后，披露义务适用新规则（从修订日起的披露期间）
- 股东会/董事会决议的合法性**按决议作出日**的规则判断
- 若历史上某决议按旧规则合法，新规则实施后该决议仍然有效（除非新法明文追溯）

### 4.4 司法解释的修订

如《虚假陈述司法解释》2022 年大修。

**衔接要点**：
- 新司法解释**施行日后**受理的案件适用新解释
- 施行日前受理但尚未终审的案件：通常按新解释
- **已终审的案件不受影响**（既判力保护）

## 五、新旧衔接的输出模板

在 reg-study 的输出中，新旧衔接分析段按以下模板：

```markdown
### 新旧衔接分析

**涉及法规**：<法规 A（旧）→ 法规 B（新）>

**分界点**：<新法施行日期 YYYY-MM-DD>

**过渡条款**：
- 法规 B 附则第 X 条：<过渡条款原文>
- 配套通知 YYY 号：<补充过渡规则>
- （如无过渡条款，注明"无明文过渡条款"）

**本项目时间线**：
- YYYY-MM-DD：<事件 1>
- YYYY-MM-DD：<事件 2>
- …

**跨越新法施行日的判断**：
- 事件 X 发生在施行日前 → 按旧法
- 事件 Y 发生在施行日后 → 按新法
- 事件 Z 跨越施行日 → 按过渡条款 / 按程序行为发生地点

**结论**：本项目整体适用 <新/旧/混合> 法；具体地：
- 实体要求：按 <旧/新>
- 程序要求：按 <旧/新>
- 披露要求：按 <旧/新>

**建议措施**：
- 若原合同条款基于旧法需调整 → <具体调整>
- 若需要出具新法合规承诺 → <承诺内容要点>
- 若需要向监管预沟通 → <沟通要点>
```

## 六、衔接分析的常见误区

| 误区 | 正确做法 |
|------|---------|
| 一律适用新法 | 先识别过渡条款；识别关键节点；再定适用法律 |
| 一律适用旧法（"签约时效法律"） | 程序性规定仍按新法；持续性义务按现时法律 |
| 忽视过渡条款 | 过渡条款是**优先考虑**的依据 |
| 把"溯及力"和"追溯调整"混淆 | 不溯及既往 ≠ 历史行为不受影响；持续合规仍按现时法律 |
| 假设"发布日 = 施行日" | 大多数法律明确规定施行日期，与发布日期分开；有过渡期的更要注意 |

## 八、fallback

若新旧衔接涉及的问题复杂、通过本分析仍不能确定适用法律：

1. 输出**两种可能的结论**并明示条件
2. 建议律师**直接与监管机关预沟通**
3. 建议**参考最近同类项目**的实际做法（调用 `ecm-research-case-search`）
4. 在意见书中**明确披露过渡期不确定性**，由签字律师承担最终判断责任
