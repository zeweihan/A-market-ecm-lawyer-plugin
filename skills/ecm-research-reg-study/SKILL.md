# ecm-research-reg-study

## 定位与边界

本 skill **负责**：
- 在**已有法规清单**（来自 `ecm-research-reg-search` 或用户直接提供）基础上，对特定法律
  问题开展**体系性、适用性**研究
- 梳理法规的**效力层级**（法律 > 行政法规 > 部门规章 > 规范性文件 > 交易所业务规则）
- 分析一般法 vs 特别法、新法 vs 旧法的关系
- 识别法规之间的**冲突**并给出**冲突解决路径**
- 分析**新旧衔接**（修订前签订的合同、跨越修订日期的行为、溯及力问题）
- 产出"法规适用路径图"+ "冲突解决结论"+ "合规建议"

本 skill **不负责**：
- 从零开始检索法规（由 `ecm-research-reg-search` 做——本 skill 预期已有法规清单）
- 案例检索（由 `ecm-research-case-search` 做，但本 skill 会**引用已检索的案例**作论据）
- 替代签字律师作出最终法律意见（本 skill 输出"分析结论"，律师作"法律意见"）

## 免责声明

本 skill 产出的法规适用性分析仅为研究辅助。法规冲突解决是一项**高度依赖具体事实**的判断，
本 skill 给出的结论是**一般性分析**，律师必须结合项目具体情境、最新监管口径、近期裁判案例
再做独立判断。完整免责声明见仓库顶层 [DISCLAIMER.md](../../DISCLAIMER.md)。

## 资深律师执行标准

执行本 skill 时，必须同时遵循 [senior-lawyer-execution-standards.md](../../shared/templates/senior-lawyer-execution-standards.md)。本 skill 的任何输出不得突破四条底线：事实可追溯、法源可核验、风险可分级、建议可落地；无法核验时必须显式标注。

## 本 skill 的实务加固点

- **问题导向**：研究结论必须回应具体交易/发行问题，不只做法规摘要。
- **适用性三段论**：事实前提、规则要件、适用结论必须分开写；事实缺口不得用法规解释补足。
- **过渡安排**：新旧规则切换时必须核查受理日、披露日、决议日、实施日等关键时点。
- **不确定性输出**：监管口径不稳定或规则冲突时，必须给出保守方案、激进方案和需外部确认事项。

## 前置依赖

- 输出格式：[shared/templates/research-output-format.md](../../shared/templates/research-output-format.md)
- 法规清单输入：由调用方提供，或通过 `ecm-research-reg-search` 先行检索
- 案例支撑：需要裁判或监管先例支撑时，调用 `ecm-research-case-search`

## 核心任务

**5 个 Phase 顺序执行**：

1. **Phase 1 — 输入与澄清**：确认研究主题、法规清单、具体事实场景
2. **Phase 2 — 效力层级图**：按效力层级梳理法规清单，建立"法源图"
3. **Phase 3 — 关系分析**：分析一般法 vs 特别法、新法 vs 旧法、上下位法
4. **Phase 4 — 冲突识别与解决**：识别冲突并按规则给出解决路径
5. **Phase 5 — 适用性结论 + 合规建议**：结合事实给出适用结论和合规建议

- 若法规清单缺失且调用方不是 `reg-search`，追问：

  > 请直接提供？"

## Phase 2 — 效力层级图

将输入的法规清单按效力层级重新组织，建立**法源图**。

### 效力层级（中国法规范层级）

```
宪法
 ├── 法律（全国人大及其常委会）
 │    ├── 行政法规（国务院）
 │    │    └── 部门规章（部委令）
 │    │         └── 规范性文件（证监会公告、部委通知等）
 │    │              └── 交易所业务规则（自律规则，属"自律规范"范畴）
 │    └── 地方性法规（省级人大）
 │         └── 地方政府规章
 └── 司法解释（最高法、最高检）：事实上具备准法律效力
```

注意：
- **司法解释**不在上述"行政—监管"序列内，而是独立的法律适用解释体系，对法院具有**直接
  约束力**，实务中其效力近似于法律。
- **自律规则**（交易所业务规则、证券业协会规则）的性质法律定位仍有争议，一般认为其不属于
  严格意义的法律规范，但通过合同/章程约定对市场主体产生约束。监管实践中，违反自律规则可
  导致纪律处分，属于一种"**自律监管**"机制。

### 输出的法源图

以**表格**形式展示层级，便于阅读：

```markdown
## 2. 法规适用法源图

| 层级 | 法规 | 条款 | 对本主题的地位 |
|------|------|------|---------------|
| 法律 | 《证券法》（2019 修订）| 第 9 条 | 上位法，建立发行的核准/注册原则 |
| 部门规章 | 《IPO 注册管理办法》（205 号）| 第 13 条 | 直接适用的发行条件 |
| 规范性文件 | 《监管规则适用指引——发行类第 X 号》| — | 对 205 号办法第 13 条的解释性文件 |
| 交易所规则 | 《科创板上市规则》| 第 2.1 条 | 科创板场景下的细化要求 |
```

或**树状图**（对复杂主题更直观）：

```markdown
《证券法》第 9 条（上位法）
│
├── 《IPO 注册管理办法》205 号 第 13 条（一般规则）
│   │
│   ├── 《监管规则适用指引——发行类第 4 号》第 X 条（解释性文件）
│   └── 《监管规则适用指引——发行类第 7 号》（特别场景）
│
└── 场景 1：科创板
    └── 《科创板上市规则》第 2.1.1 条
    
    场景 2：创业板
    └── 《创业板上市规则》第 X 条
```

## Phase 3 — 关系分析

针对法源图中的法规，做以下三组关系分析：

### 3.1 一般法 vs 特别法

- **特别法优于一般法**（特别法优先适用）
- 判断要点：
  - 范围：《证券法》是一般法；《科创板上市规则》是针对"科创板这一特定市场"的特别法
  - 事项：《公司法》是一般法；《上市公司重大资产重组管理办法》是针对"上市公司重组"的
    特别法
- 注意：**特别法优于一般法**仅在**同一位阶**规范之间或经明确授权的下位规范优于上位一般性
  规范（如特别授权的行政规定）。跨位阶时，应先看上下位关系

### 3.2 新法 vs 旧法（同一位阶同一主题）

- **新法优于旧法**（后法优于前法）
- 判断要点：
  - 施行日期的先后
  - 是否明文废止旧法
  - 新旧之间是否存在**实质冲突**
- 注意：新法未明文废止旧法但存在冲突时，应判断"**同主题同位阶**"旧法事实失效

### 3.3 上下位法（跨位阶）

- **上位法优于下位法**
- 判断要点：
  - 下位法与上位法冲突时，下位法**无效**
  - 下位法只能在上位法授权范围内作出补充规定，不得突破上位法
- 实务中常见：
  - 某证监会规章规定的条件严于《证券法》——通常认为证监会规章对上位法做了具体化，合法
  - 某证监会规章放宽了《证券法》的条件——需警惕是否超出授权

### 3.4 规范性文件和业务规则的特殊分析

- 规范性文件（如《监管规则适用指引》）**解释性地细化**部门规章，不得突破规章
- 交易所业务规则属于**自律规则**，通过章程和合同约定对市场主体产生约束；其规定严于
  证监会规章时，市场主体仍需遵守（但法律救济路径与国家立法不同）

## Phase 4 — 冲突识别与解决

### 4.1 冲突类型

| 冲突类型 | 举例 | 解决规则 |
|---------|------|---------|
| **上下位法冲突** | 下位法缩减了上位法权利 | 上位法优先；下位法无效 |
| **同位阶同主题新旧冲突** | 新办法与旧办法规定不同 | 新法优先（若新法未明确废止旧法，视为实质废止） |
| **同位阶不同主题冲突** | 办法 A（一般）与办法 B（特别） | 特别法优先 |
| **规范性文件与规章冲突** | 指引的要求超出规章 | 规章优先；规范性文件无权突破 |
| **交易所规则与部委规章冲突** | 交易所规则严于部委规章 | 属于自律加码，市场主体仍需遵守；但对法律救济路径有影响 |
| **司法解释与部门规章冲突** | 最高法认定某条款无效，而部门规章要求该条款 | 司法解释对法院有直接约束力；当事人如对部门规章提诉讼，法院按司法解释 |

### 4.2 冲突解决路径输出

对每组识别出的冲突，按以下格式输出：

```markdown
### 冲突组 1：XXX

**两份规范**：
- [L-03] 《IPO 注册管理办法》（205 号）第 13 条：要求 A
- [L-05] 《监管规则适用指引——发行类第 4 号》第 X 条：要求 A 的**更严格形式** A'

**冲突性质**：规范性文件对上位规章的加码

**适用规则**：监管规则适用指引在证监会规章的授权范围内细化规章要求；A' 对 A 的加码属
  合法补充。实务中**必须同时满足** A 和 A'。

**依据**：<援引的冲突解决法理，如"上位法优于下位法 + 授权补充合法"；必要时援引 [C-01]
  案例支撑此解释方向>

**对本项目的建议**：本项目拟适用 205 号办法第 13 条时，必须同时满足指引第 X 条的 A' 要求。
```

### 4.3 新旧衔接分析

若研究主题涉及新旧衔接（法规修订导致的过渡），必须分析：

1. **时间分界点**：新法施行日期
2. **时间性效力**：
   - 新法原则上**不溯及既往**
   - 但程序性规定、有利当事人的实体规定可能溯及
3. **过渡安排**：新法是否规定过渡条款（例如"本办法施行前已受理的案件按旧法办理"）
4. **本项目的时间位置**：
   - 发生在新法前：原则适用旧法
   - 发生在新法后：适用新法
   - **跨越新法施行日**（如 2022 年签约、2024 年交割）：需按新旧衔接规则具体分析

输出示例：

```markdown
### 新旧衔接分析

- **分界点**：《XXX 办法》（2024 版）于 2024-07-01 施行
- **过渡条款**：本办法第 XX 条规定"本办法施行前已受理的 IPO 申请，可选择按本办法或原
  办法进行审核"
- **本项目时间位置**：
  - 项目意向书签订日期：2023-12
  - IPO 受理日期：2024-03（在新法施行前）
  - 审议会议：2024-10（新法施行后）
- **适用规则**：因受理在新法前，按过渡条款可选择适用原办法；但 2024-10 后可能有部分程序性
  要求适用新办法（例如审议程序本身）
- **建议**：发行人应在报告期内通过补充说明或承诺函，明确承诺适用新办法的实体要求，以避免
  审核问询
```

## Phase 5 — 适用性结论 + 合规建议

   承诺适用新办法实体要求
5. 风险点：……

3. **结论必须紧扣事实**：不得给出脱离具体事实的通用分析
4. **冲突必须给解决路径**：识别到冲突不能只列出"存在冲突"，必须给出解决路径
5. **新旧衔接必须明示**：涉及法规修订时必须单独分析新旧衔接
6. **结论可溯源**：每条结论用 [L-XX][C-XX] 方括号编号反向引用
7. **不得代替律师判断**：分析结论需含"律师应独立核对"的提示，不得直接定性为"法律意见"

## 被其他 skill 调用

本 skill 常被以下 skill 调用：

- `ecm-draft-opinion-letter`：意见书的"适用法律分析"段落
- `ecm-draft-disclosure-review`：披露文件"法律法规"章节
- `ecm-design-*`：方案设计时的法律依据论证
- `ecm-dd-*`：DD 章节遇到**法律适用不确定**时（例如新旧法衔接）

被调用时，本 skill 的输出的"1. 核心结论"和"4. 下一步建议"两段最常被上游 skill 直接消费。
