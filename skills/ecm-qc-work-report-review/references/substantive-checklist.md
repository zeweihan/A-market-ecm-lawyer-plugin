# 律师工作报告实质审查清单

实质审查 = 在形式合规前提下，审查报告的"实质内容"是否经得起监管 / 内核推敲。共 **8 类审查项**：

---

## 类别 1 — 17 章章节完整性

**核查要点**：
- 工作报告正文覆盖的章节数 vs 实际 DD Memo 覆盖数
- 缺章处理是否合规（按 `dd-output-schema §7` 规定的占位说明）

**缺章处理的正确格式**：

```markdown
第 {N} 部分  {章节标题}

注：本章 Memo 尚未出具（对应 skill: `ecm-dd-{xxx}`），本次报告暂缺此部分。
```

并在报告首页"编制说明"段列出"未完成章节"清单。

**常见问题**：
- 章节直接跳过，无占位说明
- 占位说明格式不规范（漏掉 skill 名 / 缺"未完成章节"清单）
- 缺章但工作报告结论说"本次发行符合所有条件"（结论与缺章冲突）

**批注模板**：
> 【必改】工作报告正文缺 "第 {N} 部分 · {章节名}"，但首页"编制说明"段未列出该章为"未完成章节"。按 `dd-output-schema.md §7`，缺章须在正文插入占位说明 "本章 Memo 尚未出具（对应 skill: ecm-dd-{xxx}）"，并在首页"编制说明"段列明未完成章节清单。请补齐。

---

## 类别 2 — 五段式保留的完整性

**核查要点**（详见 cross-check-matrix §二 / form-requirements §七）：每章是否保留 DD Memo 的五段式。

**常见问题**：
- 起草人将"一、核查要点清单"缩减为几句话（应保留完整 checklist）
- "二、审阅发现"各条未按 `### 发现 N：` 格式（dd-output-schema §2.2 硬字段）
- "三、风险分级汇总"表列定义有误（dd-output-schema §2.3 要求 5 列固定）
- "五、参考资料"段被删（认为"工作底稿不对外" —— 但 `dd-output-schema §2.5` 要求保留）

---

## 类别 3 — 专属表原样保留

**核查要点**（详见 cross-check-matrix §三）：8 类专属表是否原样保留：

- `history` 股本演变时间线表
- `shareholders` 股权结构图 + 实控人穿透图
- `directors` 董监高花名册
- `business` 业务资质清单 / 前五大客户供应商集中度
- `related-party` 关联方清单 / 关联交易汇总 / 同业竞争
- `assets` 不动产 / 知识产权 / 重大设备 / 承租资产清单
- `debt` 重大借款 / 对外担保 / 大额应收应付 / 或有负债
- `independence` 五独立对照评估表 / 独立性重大依赖汇总

**常见问题**：
- 专属表被改成文字摘要
- 表格被按 Word 格式要求转成图片，导致后续不可编辑
- 股权结构图的 ASCII art 转图片时丢失

---

## 类别 4 — 全项目风险汇总表聚合正确性

**核查要点**（详见 cross-check-matrix §四 / form-requirements §九）：

- 列定义 7 列
- 聚合完整（17 份 Memo 的全部风险行）
- 序号跨章节连续
- 级别限 `高/中/低`
- "所属章节"列格式统一

**常见问题**（高频）：
- 聚合不完整（常见漏掉个别"中"级或"低"级事项）
- 序号按章节重置（1.1, 1.2, 2.1, 2.2…）
- 级别出现"严重 / 一般"等别名
- "建议措施"列为空或写"无"（应聚合 DD Memo "四、结论与建议"）

---

## 类别 5 — 编制说明段的 DD 违约提示

**核查要点**（`dd-output-schema §7`）：
如有 DD Memo 违反契约（缺段 / 表头错 / 级别非法），工作报告应在首页"编制说明"段给出提示：

```
注：下列章节对应 DD Memo 因格式不符合 shared/schemas/dd-output-schema.md 契约 vX.Y.Z，未能纳入本次拼接。请回到对应 skill 重新出具：
- ecm-dd-{xxx}：Memo 文件路径，具体违约点
```

**常见问题**：
- DD Memo 存在违约但工作报告未提示
- 违约被起草人"静默纠正"（补段 / 改表头），违反 `dd-output-schema §7`

**批注模板**：
> 【必改】经核对，`DD-Memo-{章节}-{YYYYMMDD}.md` 存在 "{具体违约点}"（例："三、风险分级汇总" 表级别列出现"严重"非法取值），但工作报告"编制说明"段未按 `dd-output-schema.md §7` 给出违约提示。请补充提示或回源 DD skill 重新出具 Memo。

---

## 类别 6 — 附件完整性

**核查要点**（form-requirements §八）：5 个附件是否齐备 + 内容完整。

**常见问题**：
- 附件一"核查文件清单"未按 17 个 DD 章节分组
- 附件二"律师出具人员清单"缺执业证号
- 附件三"律所资质证明"缺律所执业许可证扫描件或号码
- 附件五"工作底稿索引"缺（虽可选，但正式版建议有）

**批注模板**：
> 【必改】附件一"核查文件清单"未按 17 个 DD 章节分组，而是按文件类型分组。按 `work-report-format §4.2`，应按 17 个 DD 章节分组列示，便于监管回查。

---

## 类别 7 — 跨章节交叉引用

**核查要点**（form-requirements §十三）：
- 禁用"上文 / 下文 / 前述"
- 跨章节引用应用"第 X 部分 / 附件 X"
- 引用页码需与正文同步（报告调整后页码可能变化）

**常见问题**：
- "上文所述本公司……" / "如前所述……" —— 排版调整后可能错位
- 引用附件但附件名已改（起草中改过附件名但未同步正文）

---

## 类别 8 — 报告类型与正文匹配

**核查要点**：
- 律师工作报告 → 对监管披露；封面标题固定"律师工作报告"；末页不加"仅供客户内部使用"脚注
- 尽职调查报告 → 对客户内部；首页可加"仅供客户内部使用，不得对外披露"脚注

**常见问题**：
- 封面标"律师工作报告"但首页加了"仅供客户内部使用"脚注 —— 矛盾
- 封面标"尽职调查报告"但编制依据中有"适用于本次发行审核" —— 类型错配

**批注模板**：
> 【核实】本报告封面为 "{报告类型}"，但首页含 "{与报告类型冲突的脚注 / 用语}"。请核实报告类型，按 `work-report-format §一` 律师工作报告对监管披露 / 尽职调查报告对客户内部 的划分明确定位。

---

## 执行建议

按 "类别 1 → 类别 2 → 类别 3 → 类别 4 → 类别 5 → 类别 6 → 类别 7 → 类别 8" 顺序。类别 5 依赖 DD Memo 参考源，Step 1.5 未取得时跳过。
