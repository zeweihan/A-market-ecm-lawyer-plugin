# 律师工作报告内核审查 — 交叉比对矩阵

适用 Step 1.5 Level 1 成功获得 DD Memo / 法律意见书作为参考坐标时。14 项核心字段，覆盖"拼接契约"和"形式配套"两大方向。

> 参考源优先级：17 份 DD Memo > 法律意见书（形式配套）> 更早版本工作报告。

---

## 一、章节完整性（拼接契约）

按 `dd-output-schema.md §0` 表格（skill → NN → 章节名 对应关系），工作报告正文应含 17 章，章节号"第一部分…第十七部分"。

| 编号 | 核对项 | 核对方法 | 偏差处理 |
|------|------|---------|---------|
| T1 | 17 章是否齐全 | 比对工作报告章节号与 DD Memo 清单；缺章要判断是**起草遗漏**还是**DD 未做** | 若 DD Memo 有但工作报告缺：【必改】补入 / 若 DD 未做：【必改】要求工作报告在"编制说明"段加占位说明（按 `dd-output-schema §7`） |
| T2 | 章节号按 NN 升序，无断号 | 扫描一级标题"第 X 部分" | tracked change 修正编号 + 【必改】批注 |
| T3 | "独立性"作为横向章节在 NN=04 | 独立性章节位于正文第四部分 | 若位置错：tracked change 调整 + 【必改】批注 |
| T4 | NN=03 共用目录内的"设立"与"历史沿革" | 先设立后历史沿革；两个独立小节 | 合并写、顺序错：tracked change + 【必改】 |

## 二、章节内部五段式保留（拼接契约核心）

DD Memo 必须保留五段式（见 `dd-output-schema §2`）。工作报告每章内部应呈现：

```
第 N 部分  {章节标题}

一、核查要点清单
二、审阅发现
{专属表，如有}
三、风险分级汇总
四、结论与建议
五、参考资料
```

| 编号 | 核对项 | 偏差处理 |
|------|------|---------|
| S1 | 五个二级标题齐全 | 【必改】批注要求起草人回源 DD Memo 补齐 |
| S2 | 标题名与 DD Memo 一致（含顿号、序号） | tracked change 改齐 + 【必改】批注 |
| S3 | 顺序无调整 | tracked change + 【必改】批注 |

**批注模板**：
> 【必改】本章（第 {N} 部分 · {章节名}）缺少 "{缺失的二级标题}" 段。按 `shared/schemas/dd-output-schema.md §2`，DD Memo 的五段式骨架必须原样进入律师工作报告对应章节。请回源 `DD-Memo-{章节}-{YYYYMMDD}.md` 补齐本段。

## 三、专属表原样保留（拼接契约）

按 `dd-output-schema §3`，以下章节必须保留其专属表：

| skill | 专属表 | 位置（在"二、审阅发现"之后、"三、风险分级汇总"之前） |
|-------|------|------------------------------------|
| `ecm-dd-history` | 股本演变时间线表 | 位于本章"二、审阅发现"后 |
| `ecm-dd-shareholders` | 股权结构图 + 实控人穿透图 | 位于本章"二、审阅发现"后 |
| `ecm-dd-directors` | 董监高花名册 | 位于本章"二、审阅发现"后 |
| `ecm-dd-business` | 业务资质清单表 / 前五大客户供应商集中度表 | 位于本章"二、审阅发现"后 |
| `ecm-dd-related-party` | 关联方清单表 / 关联交易汇总表 / 同业竞争情况表 | 位于本章"二、审阅发现"后 |
| `ecm-dd-assets` | 不动产清单 / 知识产权 / 重大设备 / 承租资产 | 位于本章"二、审阅发现"后 |
| `ecm-dd-debt` | 重大借款 / 对外担保 / 大额应收应付 / 或有负债 | 位于本章"二、审阅发现"后 |
| `ecm-dd-independence` | 五独立对照评估表 / 独立性重大依赖汇总表 | 位于本章"二、审阅发现"后 |

| 编号 | 核对项 | 偏差处理 |
|------|------|---------|
| P1 | 专属表原样保留（不被改成文字摘要） | 【必改】要求恢复原表 |
| P2 | 专属表位置在"二、审阅发现"后 | tracked change 调整位置 |
| P3 | 独立性的"五独立对照评估表"同时供法律意见书引用 | 若内容与 DD Memo 原表不一致：【必改】统一 |

## 四、全项目风险分级汇总表（附件四，核心）

按 `shared/templates/work-report-format.md §七`，附件四"全项目风险分级汇总表"必须：

- **列定义（7 列，严格顺序）**：`序号 | 所属章节（第 N 部分） | 问题 | 级别 | 所涉文件 | 法规依据 | 建议措施`
- **聚合完整性**：聚合 17 份 Memo "三、风险分级汇总"表的**全部**行
- **序号跨章节连续编号**（不按章节重置）
- **级别限 `高/中/低`**（不接受"严重/一般/轻微"等别名）

| 编号 | 核对项 | 偏差处理 |
|------|------|---------|
| A1 | 7 列定义是否正确 | tracked change 改表头 + 【必改】批注 |
| A2 | 是否聚合 17 份 Memo 的全部风险行 | 扫描 DD Memo 总风险数 N，对比汇总表总行数 M；差异： 【必改】批注列出缺失行 |
| A3 | 序号是否跨章节连续 | tracked change 重新编号 |
| A4 | "级别"列取值合法 | tracked change 改枚举 + 【必改】 |
| A5 | "所属章节"列格式统一（"第 N 部分  {章节标题}"） | tracked change 改齐 |

**批注模板**：
> 【必改】经比对 17 份 DD Memo 的"三、风险分级汇总"表共计 {N} 行，其中"级别=高" {X} 行、"级别=中" {Y} 行、"级别=低" {Z} 行，但工作报告附件四"全项目风险分级汇总表"仅含 {M} 行，缺失 {N-M} 行：
> 1. 第 {NN} 部分 · {章节名}：{问题描述} — {级别}
> 2. …
> 请补齐聚合。按 `work-report-format §七`，附件四应聚合所有 Memo 的风险行。

## 五、元信息一致性（跨 Memo 校验）

按 `dd-output-schema §1`，每份 Memo 元信息含 5 字段：
- `company_short_name`
- `chapter_no`
- `编制日期`（YYYY-MM-DD）
- `lawyer_name`
- `skill_version`

工作报告"编制说明"段应对跨 Memo 元信息不一致做提示。

| 编号 | 核对项 | 偏差处理 |
|------|------|---------|
| M1 | `company_short_name` 17 份 Memo 全一致 | 不一致 → 【必改】批注要求工作报告"编制说明"段加提示；若工作报告已用某一版简称，tracked change 改齐 |
| M2 | `编制日期` 与工作报告出具日期的前后关系合理（Memo 日期 ≤ 报告日期） | 违反 → 【必改】或【核实】 |
| M3 | `lawyer_name` 跨 Memo 一致（多个经办律师时以顿号分隔并按姓氏笔画排序） | 不一致 → 【核实】批注 |
| M4 | `skill_version` 跨 Memo 一致（避免用不同版本 DD skill 产出） | 不一致 → 【必改】批注要求工作报告"编制说明"段提示 |

## 六、与法律意见书形式配套

同项目《法律意见书》与工作报告"形式配套"（`legal-opinion-format.md §一`）。

| 编号 | 核对项 | 偏差处理 |
|------|------|---------|
| F1 | 律所名称（中英文）一致 | tracked change + 【必改】 |
| F2 | 签字律师（姓名 / 人数 / 顺序）一致 | tracked change + 【必改】 |
| F3 | 出具日期一致（同一天） | tracked change + 【必改】 |
| F4 | 律所文号规则一致（XX 律字[YYYY]第 XXX 号；补充稿 XX-1 号） | tracked change + 【必改】 |
| F5 | 编制依据（法规清单）一致或意见书为子集 | 意见书法规清单不得**超出**工作报告；超出走【核实】或【必改】 |

**批注模板**：
> 【必改】本工作报告 {字段} 为 "{工作报告原文}"，但同项目《法律意见书》{字段} 为 "{意见书原文}"。两份文书为"形式配套"关系，{字段} 必须完全一致。请统一（建议以 {先出具的文书 / 项目质控系统记录} 为准）。
