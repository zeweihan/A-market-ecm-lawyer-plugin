# ecm-dd-fundraising

## 定位与边界

本 skill **负责**：

- 募投项目与发行人**主营业务的相关性**、是否符合**产业政策**（属于鼓励类 /
  非限制非淘汰类）
- 募投项目的**立项 / 备案 / 核准**合规性（发改委、行业主管、商务、国资等，
  视项目性质）
- 募投项目必需的**环评、安全、用地、用房**等前置合规要件是否具备或具备可行性
- 补充流动资金 / 偿还银行贷款 / 收购 / 并购等**无项目投向类**募投的合规性
  （额度、占比、用途表述、监管口径）
- 超募资金 / 节余募集资金 / 置换预先投入资金的合规性要件
- **募集资金管理制度**与**专户监管协议**的合规性
- 募投变更 / 终止的决策程序和披露义务（报告期内已发生过的）
- 出具《编报规则第 12 号》第 15 章对应的 DD Memo

本 skill **不负责**：

- 发行条件本身（《注册办法》第 12 条等）是否满足——归 `ecm-dd-approval` 与板块规则
- 募投项目环评批复的实质合规性——归 `ecm-dd-environment`；本 skill 仅核查"是否
  取得"这一程序性要件
- 募投项目土地 / 房产权属瑕疵——归 `ecm-dd-assets`（BATCH-03）；本 skill 仅
  核查"项目用地 / 用房是否落实"的可行性
- 募投项目的税务影响——归 `ecm-dd-tax`；本 skill 仅在必要时交叉引用
- 募投收购标的的尽职调查（财务、业务、资产、债务）——另走完整的标的 DD 流程
- 发行方案本身的合规性（发行条件、发行方式、价格）——归设计类 skill 和
  `ecm-dd-approval`

## 核心工作流（四步）

1. **生成核查要点清单 + 募投项目档案**：照 [references/checklist.md](./references/checklist.md)
   生成清单；同时按"每个募投项目"建立独立档案（项目名 / 投资总额 / 拟投入募集
   资金 / 建设期 / 建设内容 / 主要手续 / 用地情况 / 产品与产能 / 市场空间 / 效益测算
   的法律合规要点）
2. **文件审阅与标注**：对照每个项目的必备手续清单逐项审阅。重点识别：
   - 项目与主营业务相关性（是否偏离主业）
   - 产业政策适用（《产业结构调整指导目录》《外商投资准入负面清单》等）
   - 备案 / 核准层级是否正确（内资境内投资以备案为主，特定行业需核准）
   - 前置手续完备性（环评 / 用地 / 规划 / 安全 / 能评等）
   - 补流 / 还贷类：额度占比、期间费用合理性
   - 收购类：标的清晰、评估合规、是否形成商誉风险
3. **风险分级**
4. **结论与建议**

## 统一风险分级口径

| 级别 | 定义 | 本 skill 场景的典型例子 |
|------|------|------------------------|
| **高（红）** | 足以构成发行上市实质性障碍 | 募投项目属产业政策限制类 / 淘汰类；募投项目未取得必需备案 / 核准或用地 / 环评重大不确定性难以解决；募投项目与主营业务明显无关构成"跨主业扩张"被否决风险；主要募投项目涉及重大土地违法或无法取得用地；补充流动资金或偿还贷款占比超出板块规则规定上限 |
| **中（黄）** | 程序 / 文件瑕疵可整改 | 募投项目备案文件表述与可研报告存在差异需一致性修订；部分募投手续尚在办理中；可行性研究报告假设过于乐观、与同行业比对偏高；募集资金管理制度个别条款不符合上市公司规范；收购类募投 pre-IPO 投入置换未履行充分决策 |
| **低（蓝）** | 形式瑕疵 | 备案证印章不清晰；可研报告版本编号小差异；三方监管协议拟稿版本与最终签署版本的小差异；投资明细与台账存在可调节的小差异 |

**特别约束**：各板块上市规则对**补充流动资金 / 偿还贷款**、**财务性投资**、**房地产
业务**、**募集资金置换 pre-IPO 投入**等有明确监管口径（主板 / 创业板 / 科创板 / 北交所
有差异，再融资规则亦不同）。本 skill 在结论里必须结合项目所在板块及发行类型
（IPO / 定增 / 可转债等）给出匹配判断。

严格遵循 [shared/templates/dd-skill-template.md](../../shared/templates/dd-skill-template.md) 的
"输出格式契约"五段式结构：

1. 一级标题：`# DD Memo：募集资金运用`
2. 二级标题：`一、核查要点清单` / `二、审阅发现` / `三、风险分级汇总` / `四、结论与建议` / `五、参考资料`
3. 风险等级列取值限"高 / 中 / 低"
4. 输出落地：`02-尽职调查/02-14-募集资金运用/DD-Memo-募集资金运用-{YYYYMMDD}.md`

**本 skill 专属"额外要求"**：在"二、审阅发现"前插入**募投项目台账**（逐项目）：

| 项目编号 | 项目名称 | 项目类型 | 投资总额（万元） | 拟使用募集资金（万元） | 占比 | 建设期 | 主营业务相关性 | 产业政策适用 | 备案 / 核准 | 环评 | 用地 / 用房 | 安全 / 能评 | 效益测算口径 | 主要合规风险 | 合规结论 |
|---------|---------|---------|------------------|-------------------------|------|--------|----------------|---------------|--------------|-----|--------------|--------------|---------------|---------------|---------|

> 项目类型取值：新建生产线 / 扩建改造 / 研发中心 / 数字化智能化 / 营销服务
> 网络 / 补充流动资金 / 偿还银行贷款 / 收购（并购）/ 投资基金等。

此外对**补流和还贷**单独做一张小表：占募集资金总额比例、对应的合规依据
（各板块对 IPO / 再融资的不同口径）。

## 常见误用 / FAQ

- **Q：还未开工的"拟投资"项目，备案什么时候办？**
  A：项目备案应在申报前取得（板块差异：科创板 / 创业板 / 北交所的具体要求略有
  不同；常态是"可由发行人在取得发行批文前取得"，但尽量在申报节点已备案）。
  未备案的风险等级应按项目规模评估。

- **Q：补流 / 还贷超过 30% 是否一定为高风险？**
  A：不同板块、不同发行类型有不同上限和例外口径（例如高新 / 轻资产企业在特定
  板块可放宽）。必须结合板块规则给结论，不要一刀切。

- **Q：募投置换 pre-IPO 投入的决策程序？**
  A：应在发行人董事会、股东大会审议募投方案时一并审议；置换应与审计机构复核的
  已投入金额匹配；置换资金到账后应及时披露。
