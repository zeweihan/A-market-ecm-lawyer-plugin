# 拼接前 / 拼接后 自检清单

本文件供 `ecm-draft-report-assembly` 在拼接律师工作报告前后自检使用。

---

## 一、拼接前自检（Step 1 之前）

### 项目元信息齐备

- [ ] `company_short_name`（客户简称）已确认
- [ ] 律所全称已确认
- [ ] 签字律师至少 2 名（姓名 + 执业证号）
- [ ] 律所文号规则已确认（如 `XX 律字 [2026] 第 XXX 号`）
- [ ] 报告日期已确认（默认今天）
- [ ] 项目类型已确认（IPO / 再融资 / 并购 / 重组 / 新三板挂牌 / 其他）
- [ ] 报告类型已确认（律师工作报告 / 尽职调查报告）
- [ ] 报告版本已确认（初稿 / 第 N 稿）

### DD Memo 扫描

- [ ] 扫描 `02-尽职调查/02-{01..17}-*/` 目录，列出所有 `DD-Memo-*-*.md` 文件
- [ ] 每目录取 `YYYYMMDD` 最大的一份作为本次拼接版本
- [ ] 生成 Memo 清单（章节号 / skill 名 / Memo 文件路径 / issue_date / skill_version）

### Memo 结构校验（每份独立进行）

- [ ] 一级标题存在且符合 `dd-output-schema.md` §0 映射表
- [ ] 元信息引用块完整（含 5 个字段：项目、对应章节、编制日期、编制人、skill 版本）
- [ ] 五个二级标题齐全：`一、核查要点清单` / `二、审阅发现` / `三、风险分级汇总` / `四、结论与建议` / `五、参考资料`
- [ ] `二、审阅发现` 段每条发现含四个硬字段（相关文件 / 具体问题 / 风险级别 / 法规依据）
- [ ] `三、风险分级汇总` 表头为 5 列（编号 / 问题 / 级别 / 所涉文件 / 法规依据），级别取值限 `高 / 中 / 低`
- [ ] `四、结论与建议` 含三个三级标题（总体结论 / 待补正事项 / 建议措施）
- [ ] `五、参考资料` 含两个三级标题（法规引用 / 客户文件清单）

### 元信息一致性校验

- [ ] 跨 Memo 对比 `company_short_name`（所有 Memo 必须一致）
- [ ] 跨 Memo 对比 `lawyer_name`（不强制一致，但收集 union 集）
- [ ] 跨 Memo 对比 `issue_date`（不强制一致；跨度超过 6 个月时警示 Memo 老化）
- [ ] 跨 Memo 对比 `skill_version`（不一致时给出 Memo 版本矩阵）

### 违约 / 冲突清单（供 Step 6 编制说明使用）

- [ ] 违约跳过的 Memo 清单（文件路径 + 具体违约点）
- [ ] 元信息冲突清单
- [ ] 缺章清单（未出 Memo 的 DD skill）

---

## 二、拼接中自检（Step 3-5 之间）

### 首尾件

- [ ] 封面 6 个元素齐全（律所中英文名、主标题、事由、客户、文号、日期）
- [ ] 目录占位 / [TOC] 标签已插入
- [ ] 引言段 5 个小节齐全
- [ ] 附件一至附件四占位已插入；附件五按需
- [ ] 签字页格式正确（律所盖章、律所负责人、至少 2 名经办律师、日期）

### 正文拼接

- [ ] 按 `dd-output-schema.md` §0 NN 升序拼接
- [ ] 章节编号使用汉字（"第一部分"/"第二部分"……）
- [ ] Memo 的元信息引用块**已移除**（不出现在正文）
- [ ] Memo 的五个二级标题**原样保留**为每部分的内部结构
- [ ] skill 专属表（如股权结构图）**原样嵌入**，未解析
- [ ] 跨部分交叉引用使用形如"详见本报告第X部分"的绝对引用，未用"上文 / 下文"等相对引用
- [ ] 违约跳过的 Memo 用占位说明代替（"本章 Memo 尚未出具"）

### 风险聚合

- [ ] 附件四全项目风险汇总表已生成
- [ ] 新增"所属章节"列（格式 `第N部分  {章节标题}`）
- [ ] "序号"跨章节连续编号（1, 2, 3, ...），不按章节重置
- [ ] 按 "级别" 降序排序（高 → 中 → 低）；同级别内按章节 NN 升序
- [ ] "建议措施"列已填充（从对应 Memo 的"四、结论与建议"段抽取简化措辞）
- [ ] 表上方导读段已加
- [ ] 表头行在 Word 套版时会自动重复（Markdown 阶段无需特别处理）

---

## 三、拼接后自检（Step 6 之前）

### 文件落地

- [ ] 报告 Markdown 落到 `04-文件输出/律师工作报告/律师工作报告-{company_short_name}-{YYYYMMDD}.md`
- [ ] 编制说明 Markdown 落到同目录 `编制说明-{YYYYMMDD}.md`

### 编制说明内容完整

- [ ] 合规拼接的 Memo 清单（章节号 / skill 名 / skill_version）
- [ ] 违约跳过的 Memo 清单（文件路径 + 违约点）
- [ ] 元信息冲突清单
- [ ] 缺章清单（未出 Memo 的 DD skill）
- [ ] 报告首页"编制说明"段落已同步列出上述四项

### 整体合理性检查

- [ ] 报告字数 > 10,000（17 章齐备时典型字数 50,000+）
- [ ] 报告内所有引用的法规均出现在编制依据清单中
- [ ] 报告内所有"第N部分"标题都出现在目录占位中
- [ ] 报告日期与封面日期一致
- [ ] 签字律师在报告正文"（二）本所及经办律师承办概况"段已列出

---

## 四、交付前最终自检

- [ ] 已告知用户下一步：
  - 手工复核后可调用 `ecm-draft:format-adjust` 做 Word 套版
  - 或直接调用外部 `docx` skill 做 Markdown → Word 转换
  - 内核审查可调用 `ecm-qc:work-report-review`（BATCH-09）
- [ ] 已提示用户：本报告为初稿，需签字律师复核后方可作为申报材料
