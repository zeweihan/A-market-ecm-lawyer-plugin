# 风险聚合 / 去重 / 排序规则

本文件是 `ecm-draft-report-assembly` 聚合 17 份 DD Memo 的"三、风险分级汇总"表时的细节规则说明。

---

## 一、聚合的数据源

**仅**读取各 DD Memo 的"三、风险分级汇总"段落下的 Markdown 表（表头固定为 `| 编号 | 问题 | 级别 | 所涉文件 | 法规依据 |`）。

**不读取**：
- "二、审阅发现" 段的三级标题细节（太底稿，聚合汇总表已覆盖）
- "四、结论与建议" 的自由文字（只取"建议措施"小节做简化抽取）

## 二、聚合后的表结构

汇总表列定义（顺序固定）：

```markdown
| 序号 | 所属章节 | 问题 | 级别 | 所涉文件 | 法规依据 | 建议措施 |
|------|---------|------|------|---------|---------|---------|
```

| 列 | 来源 | 处理 |
|----|------|------|
| 序号 | 新生成 | 跨章节连续编号（1, 2, 3, ...） |
| 所属章节 | Memo 的一级标题 + dd-output-schema.md §0 | 格式 `第N部分  {章节标题}` |
| 问题 | Memo 风险表"问题"列 | 原样 |
| 级别 | Memo 风险表"级别"列 | 原样（高 / 中 / 低） |
| 所涉文件 | Memo 风险表"所涉文件"列 | 原样 |
| 法规依据 | Memo 风险表"法规依据"列 | 原样 |
| 建议措施 | Memo "四、结论与建议" / "建议措施" 段 | 按"问题"做关键词匹配，抽对应简化措辞；匹配不到时留空 |

## 三、去重规则

**默认不去重**。每份 Memo 的风险事项独立呈现，即使跨章节出现同一事实（如"某项关联交易"在 related-party 和 debt 两章都被提及），各章节的风险视角不同，保留各自条目。

**例外**：如果用户明确要求"压缩重复项"，可按以下规则去重：
- 问题列全文完全相同 + 所涉文件列全文完全相同 → 视为同一事项
- 合并时"所属章节"列保留两份章节名（用顿号分隔，如"第五部分  股东及实控人、第八部分  关联交易与同业竞争"）
- 级别取两者中**更高**的级别（高 > 中 > 低）

## 四、排序规则

### 第一层：按"级别"降序

```
高（红） → 中（黄） → 低（蓝）
```

### 第二层：按章节 NN 升序

同级别内，按 dd-output-schema.md §0 表格的 NN（01-17）升序排列。

### 第三层：按原 Memo 内"编号"列升序

同级别 + 同章节内，按原 Memo "三、风险分级汇总" 表的"编号"升序。

### 序号重新编号

所有排序完成后，从 1 开始重新编"序号"列。

## 五、"建议措施"列的抽取

从对应 Memo 的"四、结论与建议" → "建议措施" 段（bullet list）抽取：

```markdown
- 补充材料：...
- 程序整改：...
- 披露安排：...
- 获取承诺函：...
```

抽取策略：
- 在 bullet list 中搜索与"问题"列关键词匹配的 bullet（启发式：文字相似度 > 0.6 或关键实体名匹配）
- 匹配到的 bullet 文字，去掉前缀（"补充材料："等）后作为"建议措施"值
- 匹配不到的，留空，并在编制说明中提示用户人工补充

## 六、导读段（汇总表上方）

```
本表汇总发行人在全部 17 项尽调维度下识别的问题，按风险级别提示，具体分析详见各部分。
本次拼接共纳入 {M} 份 DD Memo（共 17 份；缺 {17-M} 份见编制说明）。识别高风险事项 {H} 项、
中风险事项 {M_} 项、低风险事项 {L} 项。建议优先处置高风险事项后进入申报程序。
```

其中 M、H、M_、L 的取值由 report-assembly 根据实际聚合结果填入。

## 七、表后脚注（可选）

若用户的报告会附电子版（而非单纯打印），可在表后加脚注：
```
注：本表电子版中"所涉文件"列的文件名支持跳转到项目底稿目录对应文件；正式打印版本以文字为准。
```

纸质报告无需此脚注。

## 八、违约表的处理

若某份 Memo 的"三、风险分级汇总" 表违约（列名不符 / 级别取值超出枚举），按 `dd-output-schema.md` §7：

- 该 Memo 整体跳过（所有章节内容及风险表均不进入拼接）
- 在编制说明中列明：`ecm-dd-{xxx}: Memo 文件路径, 违约点描述`
- 汇总表**不补齐**该章节的占位行（占位行属于章节层的缺章处理，不是风险聚合层）

## 九、重复风险的告警（跨 Memo）

即使默认不去重，仍对以下情形生成告警（写入编制说明）：

- 两条风险的"问题"列文字相似度 > 0.8 → 疑似重复，请人工确认
- 两条风险的"所涉文件"列完全一致但"级别"不同 → 级别定性冲突，请人工核对

告警不改变聚合结果，只提示律师人工复核。
