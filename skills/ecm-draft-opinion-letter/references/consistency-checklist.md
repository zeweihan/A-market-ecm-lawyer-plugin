# 与律师工作报告 — 形式配套校验清单

本文件是 `ecm-draft-opinion-letter` 在 strict 模式（默认）下，与同项目的律师工作报告做形式配套校验的检查清单。

---

## 一、首页元信息层（强一致）

校验意见书与工作报告的以下字段**完全一致**：

| 字段 | 校验方式 | 不一致时 |
|------|---------|---------|
| 律所全称 | 完全相等 | 阻塞流程，追问用户 |
| 律所英文名 | 完全相等 | 阻塞 |
| 律所文号规则 | 规则一致（如均为 `XX 律字 [2026] 第 XXX 号`，编号可不同但规则一致） | 阻塞 |
| 出具日期 | 完全相等 | 阻塞 |
| 客户全称 | 完全相等 | 阻塞 |
| 客户简称 | 完全相等 | 阻塞 |
| 项目名 | 完全相等 | 阻塞 |
| 签字律师姓名 | 完全相等且顺序一致 | 阻塞 |
| 签字律师人数 | 完全相等 | 阻塞 |
| 律所负责人姓名 | 完全相等 | 阻塞 |

## 二、编制依据层（强一致）

- 意见书"一、引言（五）核查范围与编制依据"列出的法规清单 **必须完全包含**工作报告"（三）编制依据"的法规清单
- 意见书可以多列（意见书层可能引用更具体的法规），但不得少列
- 重叠部分的法规版本必须一致

## 三、核查覆盖范围层（强一致）

- 意见书发表意见的章节清单 **必须完全包含**工作报告已出 Memo 的章节清单
- 意见书不得对工作报告**未覆盖**的章节发表结论性意见
- 工作报告覆盖但意见书跳过的章节必须在意见书首页编制说明中标注"暂缺"原因

## 四、结论一致性层（强一致，重点）

| 校验点 | 期望 |
|-------|------|
| 意见书每章"法律意见"的结论级别 | 与工作报告该章 Memo "三、风险分级汇总" 的最高级风险匹配（无 / 低 → 肯定；中 → 有条件肯定；高 → 保留） |
| 意见书"五、特别事项提示" | **必须**包含工作报告附件四"全项目风险汇总表" `级别=高` 的全部事项（按去重规则） |
| 意见书"特别事项提示"不得包含工作报告未列出的事项 | 硬约束 |
| 意见书结论性意见的措辞 | 不得超出工作报告的核查范围 |

## 五、释义一致性层（弱一致）

- 释义条目可以不完全相同（意见书可精简）
- 但同一术语的定义**必须**完全相同
- 工作报告中出现的缩写，若在意见书正文出现，必须在意见书"二、释义"段定义

## 六、页眉页脚层（弱一致）

- 意见书页眉奇数页"法律意见书"、偶数页律所名；工作报告页眉奇数页"律师工作报告"、偶数页律所名 —— 这是**差异**（不需一致）
- 页码规则一致（封面、目录、释义页不显示页码；从引言起算）

## 七、签字页层（强一致）

- 律所盖章位置、大小、样式一致
- 律所负责人签字栏一致
- 经办律师签字栏**完全一致**（姓名、人数、顺序、排版）
- 日期完全一致

## 配置项交互

- **strict 模式**：任何层的"阻塞"级不一致都会暂停意见书起草，追问用户 **不允许继续**
- **advisory 模式**：不一致时生成警告，继续起草，最终输出的校验报告清晰列明所有不一致
- **none 模式**：不做配套校验（单独出意见书；在编制说明中显著标注"本意见书未与律师工作报告做配套校验"）

---

## 常见不一致类型与处理

1. **经办律师名单顺序不同**：strict 模式阻塞；最常见原因是 opinion-letter 和 report-assembly 在不同会话中调用，未共享元信息
2. **特别事项提示遗漏工作报告的高风险**：在 opinion-letter Step 5 的自动注入环节出错；检查 DD Memo 的"风险分级汇总"表是否完整
3. **意见书多列了工作报告未覆盖的章节**：意见书不得超出工作报告核查范围；删除多余章节或补出工作报告对应 Memo
4. **日期差 1 天**：多数因 opinion-letter 和 report-assembly 跨天运行；以工作报告日期为准，更新意见书首页
5. **律所文号有误**：文号是律所手工分配的，opinion-letter 直接采用用户输入；对不齐时追问
