# ecm-dd-business · 核查要点清单

本 skill 在项目中生成的 DD Memo 应照本清单逐条核查。法规依据以 `shared/regulations/` 下条文为准。

## 一、经营范围与实际业务一致性

### A. 经营范围

- [ ] 工商登记经营范围覆盖发行人实际从事的全部业务
- [ ] 实际业务不超出经营范围（超范围经营需评估合规风险）
- [ ] 经营范围中的前置审批事项已取得对应许可证 / 批文
- [ ] 经营范围中的后置审批事项已取得对应许可证 / 批文
- [ ] 近 3 年经营范围变更及其原因（与业务战略调整是否一致）

### B. 主营业务突出

- [ ] 主营业务收入占营业收入比例通常不低于 70%（板块规则另有规定的从其规定）
- [ ] 主营业务与招股说明书 / 申报材料表述一致
- [ ] 业务模式（B2B / B2C / 平台 / 订阅制 / 项目制等）清晰可验证
- [ ] 报告期内主营业务未发生重大不利变化（《注册办法》第 12 条第 2 款）

## 二、经营资质与行业许可

### A. 行业资质清单

逐项核查发行人业务所需的经营资质 / 许可证，形成**资质清单表**（见 SKILL.md 额外要求）：

- [ ] 普通许可证类（食品经营 / 药品经营 / 医疗器械经营等）
- [ ] 特许经营类（成品油 / 烟草 / 盐业 / 邮政等）
- [ ] 特殊行业类（金融 / 保险 / 证券 / 期货 / 基金 / 支付 / 征信 / 保险经纪 / 小贷 / 融资担保等）
- [ ] 高新技术类（高新技术企业证书 / 技术先进型服务企业 / 研发机构认定）
- [ ] 认证类（ISO 质量管理体系 / CMMI / 等保测评 / 安全生产标准化等，视行业而定）
- [ ] 进出口 / 外贸类（对外贸易经营者备案 / 海关 AEO 认证等）
- [ ] 互联网 / 数据类（ICP 许可证 / EDI 许可证 / SP 许可证 / 网络文化经营许可证 / 网络出版服务许可证等）

### B. 资质核查要点（对每个资质）

- [ ] 发证机关适格（中央 vs 地方 / 行业主管部门）
- [ ] 证号、发证日期、有效期
- [ ] 许可范围覆盖实际业务
- [ ] 是否在续展程序中
- [ ] 是否因违规被暂扣 / 吊销 / 撤销
- [ ] 跨省 / 跨境经营所需的异地备案是否齐全

### C. 特许经营 / 特殊行业

- [ ] 特许经营权取得方式（招拍挂 / 授权 / 转让）
- [ ] 特许经营期限、独占性、续展条件
- [ ] 金融类业务的行业主管部门事前 / 事后批准
- [ ] 外资受限行业的 FDI 准入审查（负面清单）

## 三、重大业务合同

### A. 合同清单与抽样

- [ ] 按金额 / 战略重要性从重大合同中抽样（建议覆盖每个业务线前 3 大合同 + 报告期内金额前 20 大合同）
- [ ] 合同分类：销售 / 采购 / 技术服务 / 加工承揽 / 特许经营 / 框架协议 / 补充协议

### B. 合同条款核查

- [ ] 签约主体适格（对方身份证明、法定代表人、授权委托）
- [ ] 合同标的、金额、数量、期限明确
- [ ] 付款条款（预付 / 验收后 / 节点付款）及对应账期
- [ ] 质量标准、验收条件、违约责任
- [ ] 终止 / 变更 / 不可抗力 / 争议解决条款
- [ ] 对发行人 IPO / 重大资产重组 / 控制权变更的限制性条款（"change of control"）
- [ ] 保密 / 竞业 / 知识产权归属条款
- [ ] 合同签署时间、盖章、签字完整
- [ ] 合同是否与业务合同台账一致

### C. 客户 / 供应商集中度

报告期内逐年分析（形成**前五大客户 / 供应商集中度表**，见 SKILL.md 额外要求）：

- [ ] 前五大客户销售占比（整体 + 分业务线）
- [ ] 前五大供应商采购占比
- [ ] 单一客户 / 供应商依赖度超过 30% 的合理性说明
- [ ] 客户 / 供应商是否为关联方（交由 `ecm-dd-related-party` 进一步核查）
- [ ] 客户 / 供应商稳定性（合作年限、可替代性、在手订单）
- [ ] 新增大客户 / 供应商的背景和商业合理性

## 四、业务合法合规（横向合规）

### A. 反垄断与反不正当竞争

- [ ] 是否存在限制竞争的协议、决定、协同行为（价格联盟 / 市场分割 / 限制产量 / 限制技术等）
- [ ] 是否存在滥用市场支配地位行为
- [ ] 近 3 年内经营者集中申报情况（收购、合资、新设）
- [ ] 行政处罚或约谈记录（若有，交叉送 `ecm-dd-litigation`）
- [ ] 是否涉及"平台反垄断"监管（互联网平台类发行人）

### B. 数据合规与个人信息保护

- [ ] 是否收集、处理、存储、传输个人信息
- [ ] 是否属于《数据安全法》《个人信息保护法》《网络安全法》《生成式人工智能服务管理暂行办法》
      的适用对象
- [ ] 关键信息基础设施（CII）认定情况
- [ ] 数据分类分级制度
- [ ] 跨境数据传输 / 数据出境安全评估 / 标准合同备案 / 认证（2022 年网信办规定）
- [ ] 近 3 年内的网络安全审查情况
- [ ] 数据泄露事件与处置记录

### C. 行业监管政策

- [ ] 近 3 年内行业重大监管政策变化（是否对发行人业务构成重大不利影响）
- [ ] 是否处于"双减""清退""反垄断约谈"等强监管行业
- [ ] 新出台政策对发行人未来业务持续性的影响

### D. 外资准入与产业政策

- [ ] 业务是否涉及《外商投资准入负面清单》限制 / 禁止领域
- [ ] 业务是否属于《产业结构调整指导目录》鼓励 / 限制 / 淘汰类
- [ ] 业务是否涉及《市场准入负面清单》

### E. 国家安全审查

- [ ] 外资并购涉及关键领域的国家安全审查情况
- [ ] 关键信息基础设施运营者采购网络产品和服务的网络安全审查情况
- [ ] 境外上市前的网络安全审查情况（适用于收集 100 万以上用户个人信息的境外上市主体）

## 五、业务完整性与连续性

### A. 业务完整性

- [ ] 生产经营所需的主要原材料 / 设备 / 技术 / 渠道不依赖控股股东 / 实控人 / 关联方
- [ ] 发行人具有独立的销售体系、品牌、客户资源
- [ ] 关键业务流程（采购、生产、销售、售后）不受控股股东 / 实控人控制
- [ ] 若存在重大依赖：已通过资产重组 / 合同安排 / 承诺函等方式化解（视具体情形评估，与
      `ecm-dd-independence` 协同）

### B. 业务连续性

- [ ] 主要业务合同存续期覆盖上市后合理期间（通常 1 年以上或视行业惯例）
- [ ] 核心客户 / 供应商关系稳定性（合作年限、终止风险）
- [ ] 核心技术 / 商标 / 专利（由 `ecm-dd-assets` 主导核查）的稳定性
- [ ] 行业政策 / 市场环境的重大变化风险

## 六、关键文件清单（客户需提供）

- [ ] 营业执照（载明经营范围）
- [ ] 业务资质证书、经营许可证、特许经营权文件（全量）
- [ ] 高新技术企业证书 / 研发机构认定 / 创新型企业证明（如有）
- [ ] 重大业务合同（销售 / 采购 / 技术服务 / 加工 / 特许经营）
- [ ] 报告期内前五大客户 / 供应商清单（含金额、占比、关联关系）
- [ ] 业务合同台账 / 在手订单清单
- [ ] 行业监管政策文件、行业协会证明（如适用）
- [ ] 反垄断申报文件、反垄断行政处罚文件（如有）
- [ ] 数据合规评估报告、个人信息保护影响评估报告、等保测评证书
- [ ] 跨境数据传输相关文件（安全评估申报 / 标准合同备案 / 认证）
- [ ] 网络安全审查相关文件（如适用）
- [ ] 业务流程文件（采购 / 生产 / 销售 / 售后 / 研发）

## 七、特别关注事项

- **平台反垄断 / 互联网监管**：平台类发行人须专项核查"二选一""大数据杀熟""算法歧视"等问题
- **数据跨境**：向境外提供重要数据 / 超过 100 万人个人信息，通常需网信办安全评估
- **AI / 算法**：生成式 AI 服务需做算法备案、安全评估（2023 年《生成式人工智能服务管理暂行办法》）
- **医药 / 医疗器械**：GCP / GMP / GSP 合规，临床试验数据的合规性
- **教育培训**：学科类 vs 非学科类、"双减"政策适用
- **房地产 / 城投**：政策限制与融资要求（通常不适合 A 股 IPO，但可参与并购）
- **境外上市备案**：若同步申报境外上市，依据《境内企业境外发行证券和上市管理试行办法》（2023）
  需完成备案
