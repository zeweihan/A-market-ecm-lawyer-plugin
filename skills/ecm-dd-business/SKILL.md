# ecm-dd-business

## 定位与边界

本 skill **负责**：
- 核查发行人主营业务与工商登记经营范围、实际经营活动的一致性与业务模式清晰度
- 核查开展业务所需的行业资质、经营许可证、特许经营权、行业准入许可是否齐备有效
- 核查业务的合法合规性，特别是反垄断、反不正当竞争、数据与个人信息保护、网络安全审查、
  国家安全审查等横向合规
- 核查重大业务合同（销售 / 采购 / 技术服务 / 特许经营等）的真实性、履行状态、关键条款
- 核查主要产品 / 服务的市场地位、核心竞争力、前五大客户 / 供应商的集中度与合理性
- 核查业务完整性（是否存在生产经营对关联方的重大依赖）、业务连续性（行业政策不利变化风险）
- 出具编报规则第 7 章对应的 DD Memo

本 skill **不负责**：
- 发行人主体是否合法存续、36 个月消极条件（归 `ecm-dd-entity`）
- 业务合同项下的关联交易公允性、同业竞争（归 `ecm-dd-related-party`）
- 不动产、知识产权、租赁等主要财产权属（归 `ecm-dd-assets`）
- 借款合同、担保合同、应收 / 应付款等债权债务（归 `ecm-dd-debt`）
- 税务合规（归 `ecm-dd-tax`）
- 环保、产品质量、安全生产（归 `ecm-dd-environment`）
- 募投项目的业务合同与可行性（归 `ecm-dd-fundraising`）
- 诉讼、仲裁、行政处罚（归 `ecm-dd-litigation`）
- 文件读取 / 搬运（归 `ecm-dd-file-review` / `ecm-setup-file-organize`）

执行本 skill 时，必须同时遵循 [senior-lawyer-execution-standards.md](../../shared/templates/senior-lawyer-execution-standards.md)。本 skill 的任何输出不得突破四条底线：事实可追溯、法源可核验、风险可分级、建议可落地；无法核验时必须显式标注。

## 本 skill 的实务加固点

- **业务模式拆解**：必须拆分采购、生产/服务、销售、结算、客户供应商集中度和外包/经销/加盟模式。
- **资质许可闭环**：业务许可证照应核对主体、范围、期限、地域、年检/续期和与实际业务的一致性。
- **高风险触发器**：无证经营、超范围经营、核心资质即将到期、重大客户供应商依赖、商业模式合法性存疑，应列高风险。
- **披露联动**：业务描述与招股书、审计报告、重大合同和官网宣传不一致时，必须提示信披自查。

## 核心工作流（四步）

1. **生成核查要点清单**：照 [references/checklist.md](./references/checklist.md) 输出本项目的核查清单。
   项目类型（IPO / 再融资 / 重大资产重组 / 新三板 / 债券）会影响资质核查的深度，优先核对项目类型。
2. **文件审阅与标注**：对照清单审阅 `02-06-业务资质/` 目录下的营业执照、资质证书、重大业务
   合同、行业监管文件，重点比对经营范围覆盖性、资质有效期、合同金额与期限、客户 / 供应商
   集中度。必要时调用 `ecm-dd-data-verify` 对企查查 / Tushare 数据做交叉验证。
3. **风险分级**：按下方口径给每个问题打高 / 中 / 低。
4. **结论与建议**：汇总业务合规状况和持续经营能力，列出缺失资质、合同瑕疵、集中度风险和
   补正建议。

## 统一风险分级口径

| 级别 | 定义 | 本 skill 场景的典型例子 |
|------|------|------------------------|
| **高（红）** | 足以构成发行 / 交易的实质性障碍 | 核心业务无证经营（如涉及前置审批的金融 / 医疗 / 教育 / 文化业务）、核心资质即将到期且无法续展、业务合法性存疑（涉嫌违反《反垄断法》经营者集中、数据出境违规、涉及国家安全审查未申报）、对单一客户 / 供应商依赖度超过 50% 且不可替代、主营业务在报告期内发生重大不利变化且无法复原 |
| **中（黄）** | 程序或文件存在瑕疵但可补正 | 经营范围未覆盖全部实际业务但可做工商变更、部分辅助业务资质尚在续展中、前五大客户集中度 30%—50% 但有合理商业解释、技术服务类重大合同签署程序不完备、合同关键条款（违约、付款节奏）存在对发行人不利的条款 |
| **低（蓝）** | 不影响实质的形式瑕疵 | 经营范围表述与实际业务措辞差异、合同模板字体 / 编号不统一、业务资质附件扫描不清、行业协会证明样式陈旧但内容真实 |

## 输出格式契约（强制）

严格遵循 [shared/templates/dd-skill-template.md](../../shared/templates/dd-skill-template.md) 的
"输出格式契约"五段式结构：

1. 一级标题：`# DD Memo：发行人的业务`
2. 二级标题：`一、核查要点清单` / `二、审阅发现` / `三、风险分级汇总` / `四、结论与建议` / `五、参考资料`
3. 风险等级列取值限"高 / 中 / 低"
4. 输出落地：`02-尽职调查/02-06-业务资质/DD-Memo-业务-{YYYYMMDD}.md`

**额外要求**（本 skill 专属）：

1. Memo 中必须附**业务资质清单表**，字段：资质名称 / 发证机关 / 证号 / 发证日期 / 有效期至 /
   是否续展 / 覆盖业务范围
2. Memo 中必须附**前五大客户 / 供应商集中度表**，分别列示报告期内每一期的客户（或供应商）
   名称、销售（或采购）金额、占比、合作年限、是否关联方

## 与邻近 DD skill 的边界

| 场景 | 归属 skill | 分工说明 |
|------|-----------|---------|
| 主营业务的实际开展情况 | **本 skill** | 本 skill 的核心 |
| 客户 / 供应商构成关联方 | `ecm-dd-related-party` | 本 skill 只列客户 / 供应商的**商业**集中度；关联方属性和关联交易公允性归 related-party |
| 销售合同对应的应收账款 | `ecm-dd-debt` | 本 skill 关注合同履行与关键条款；应收账款余额和账龄归 debt |
| 生产所需的专利 / 商标 / 软件著作权 | `ecm-dd-assets` | 本 skill 关注 IP 是否支撑当前业务；IP 权属、续展、质押归 assets |
| 主营业务是否能独立于控股股东 | `ecm-dd-independence` | 本 skill 关注业务开展本身；业务独立性归 independence |
