# ecm-dd-compliance

## 定位与边界

本 skill 是《编报规则第 12 号》第 17 章"其他合规事项"的**兜底核查**——只覆盖
**其他 DD skill 未覆盖的剩余合规事项**。本 skill **不应**作为主要 DD skill 被
用来核查已有专章的事项。

本 skill **负责**：

- **劳动用工合规**：社保 / 公积金缴纳基数、缴纳比例、覆盖率；劳动合同签订率
  与期限；非全日制用工、实习生、劳务派遣的使用情况；工会 / 职代会设置
- **海关合规**：进出口经营资质、海关 AEO 认证、加工贸易手册、保税物流资质
  等特殊海关合规；海关行政处罚的登记由 `ecm-dd-litigation` 处理
- **外汇合规**：FDI / ODI / 外债 / 货物贸易 / 服务贸易的外汇登记与备案；
  资本项目登记、跨境人民币、37 号文登记
- **反垄断 / 反不正当竞争 / 反洗钱 / 反商业贿赂 / 反恐怖融资等合规管理体系**
- **数据合规**：网络安全等级保护备案、关键信息基础设施、数据出境安全评估
  或标准合同备案、个人信息保护合规（PIA）、数据分类分级
- **出入境管理**：外籍员工工作许可 / 居留许可 / 外国人来华就业证
- 其他未归入前述章节的合规事项（**严格限"其他"**）

本 skill **不负责**（务必识别并转给正确 skill）：

- 税务合规 → `ecm-dd-tax`
- 环保 / 安全生产 / 产品质量 → `ecm-dd-environment`
- 募集资金运用 → `ecm-dd-fundraising`
- 诉讼 / 仲裁 / 行政处罚 → `ecm-dd-litigation`
- 业务经营性许可 → `ecm-dd-business`
- 关联交易 → `ecm-dd-related-party`
- 独立性 → `ecm-dd-independence`
- 治理 / 章程 / 三会 → `ecm-dd-charter`
- 主体资格 / 设立 / 历史沿革 / 股东及实控人 / 董监高 / 资产 / 债务 → 分别
  对应 `ecm-dd-entity` / `ecm-dd-establishment` / `ecm-dd-history` /
  `ecm-dd-shareholders` / `ecm-dd-directors` / `ecm-dd-assets` / `ecm-dd-debt`

## 免责声明

本 skill 产出的 DD Memo 为工作底稿，不构成最终法律意见，更不构成任何专项
合规意见（劳动 / 海关 / 外汇 / 数据等均有独立专业领域）。完整免责声明见
[DISCLAIMER.md](../../DISCLAIMER.md)。

执行本 skill 时，必须同时遵循 [senior-lawyer-execution-standards.md](../../shared/templates/senior-lawyer-execution-standards.md)。本 skill 的任何输出不得突破四条底线：事实可追溯、法源可核验、风险可分级、建议可落地；无法核验时必须显式标注。

## 本 skill 的实务加固点

- **兜底不泛化**：仅承接未被其他 DD skill 覆盖但可能影响发行/交易的合规事项，不能替代专项 DD。
- **监管记录全扫**：证监会、交易所、股转、发改、商务、外汇、市场监管、海关、网信等记录按项目类型筛查。
- **重大违法判断**：违法事实、处罚机关、处罚依据、金额、整改、主管证明和规则口径必须同时列示。
- **高风险触发器**：报告期重大违法、监管立案未结、整改无证明、影响发行条件的诚信问题，应列高风险。

## 核心工作流（五步）

1. **先做分流**：识别用户请求所涉事项是否属于邻近具体章节（tax /
   environment / fundraising / litigation 等）。**如属，先触发对应 skill**，
   再由本 skill 兜底其余部分
2. **建立合规议题清单**：按本 skill 的 7 大议题组（劳动 / 海关 / 外汇 /
   反垄断反腐 / 数据 / 出入境 / 其他）逐议题评估是否适用，不适用的明确
   标注"N/A 及理由"
3. **文件审阅与登记**：照 [references/checklist.md](./references/checklist.md)
   分议题逐项核查
4. **风险分级**
5. **结论与建议**：对每一议题给出"是否合规 / 是否需披露"结论

## 统一风险分级口径

| 级别 | 定义 | 本 skill 场景的典型例子 |
|------|------|------------------------|
| **高（红）** | 足以构成发行上市实质性障碍 | 大规模未依法缴纳社保 / 公积金且无可行整改计划；应报未报反垄断经营者集中；严重违规向境外传输数据并构成重大违法；外汇重大违规未整改；劳务派遣比例长期超标且被监管要求整改未完成；外籍员工大规模无工作许可用工；反洗钱 / 反商业贿赂领域被行政处罚构成"情节严重"；关键信息基础设施运营者未履行重大义务被监管纳入重点监管 |
| **中（黄）** | 程序 / 文件瑕疵可整改 | 部分员工社保 / 公积金缴纳基数不足；历史小额海关 / 外汇处罚已缴纳完毕；劳务派遣比例略高（接近 10% 上限）但有整改计划；合规管理制度有缺漏但无处罚；数据合规制度未全面建立；个别外籍员工工作许可即将到期 |
| **低（蓝）** | 形式瑕疵 | 员工名册格式不规范；保密 / 竞业协议版本 minor 差异；工会登记材料不齐但工会实际设立；合规证明出具时间稍早 |

**特别约束**：
- **社保 / 公积金**是 A 股发行核查最高频、最敏感的合规议题。监管口径在不同
  地方、不同板块略有差异，但原则是"应缴尽缴 + 重大欠缴必须补缴 + 当地主管
  部门不追究的书面说明"三件套——缺一件都应调高风险分级
- **数据合规**自 2021 年《数据安全法》《个人信息保护法》起成为上市审核高
  关注议题，涉及数据出境、APP 信息收集的，在本 skill 核查时建议结合行业
  最新审核案例口径
- **反垄断经营者集中**有"申报但未被批准即实施"的"未申报先实施"历史情形，
  监管对此高度敏感

## 常见误用 / FAQ

- **Q：如果一份文件同时涉及多个 DD 章节，应贴什么标签？**
  A：按 `shared/terminology/classification-labels.md` 的 many-to-many 规则贴
  多个标签。例如"海关行政处罚决定书"应同时贴 `compliance` 和 `litigation`，
  分别触发两个 DD skill 从不同角度核查。

- **Q：社保 / 公积金"是不是必须按全员全额缴纳"？**
  A：法律上"应缴尽缴"，但监管对"小范围未缴纳 + 当地政策照顾 + 已补缴"的
  情形有宽限度，审核倾向看是否取得"不追究的书面意见"。建议在 Memo 中
  注意这三件套的获取。

- **Q：反垄断"未申报先实施"可以补救吗？**
  A：可以，监管近年有补申报实践（《国务院关于经营者集中申报标准的规定》及
  其实施细则）。补申报同时需核查是否存在行政处罚风险（如有，登记到
  `ecm-dd-litigation`）。
