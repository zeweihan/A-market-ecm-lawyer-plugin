# 格式检查清单（按文档类型分）

本文件列出 `ecm-draft-format-adjust` 在 Step 7 生成格式检查清单时要核对的具体条目。按文档类型独立列示。

---

## 一、律师工作报告 / 尽职调查报告（`work-report`）

### 封面

- [ ] 律所中文名（居中，三号宋体加粗）
- [ ] 律所英文名（居中，小四 Times New Roman 斜体）
- [ ] 主标题（居中，二号黑体加粗）
- [ ] 事由（居中，三号楷体_GB2312）
- [ ] 致：客户全称（小四宋体）
- [ ] 律所文号（小四宋体）
- [ ] 日期（小四宋体，中文格式）
- [ ] 封面独占一页（末尾分页符）

### 目录

- [ ] 位于封面之后、引言之前
- [ ] 深度 2 级（章 + 一级小节）
- [ ] 第一级小四宋体加粗；第二级小四宋体不加粗
- [ ] 页码右对齐，中间虚线引导
- [ ] 标题"目　录"（居中，三号黑体加粗，字间加全角空格）

### 正文章节

- [ ] 章节标题"第N部分"使用自动编号（Heading 1 绑定列表）
- [ ] 一级小节"一、"使用 Heading 2 + 自动编号
- [ ] 二级小节"（一）"使用 Heading 3 + 自动编号
- [ ] 章节编号连续无断号
- [ ] DD Memo 的五个二级标题（"一、核查要点清单"等）作为章节内部结构保留
- [ ] skill 专属表（股本演变时间线等）原样嵌入

### 附件

- [ ] 附件一：核查文件清单（按 17 章分组）
- [ ] 附件二：律师出具人员清单（含执业证号）
- [ ] 附件三：律所资质证明
- [ ] 附件四：全项目风险分级汇总表（自动生成）
- [ ] 附件五（可选）：专项核查工作底稿索引
- [ ] 每个附件独占起始页（分页符）
- [ ] 附件标题"附件 N"使用三号黑体加粗

### 签字盖章页

- [ ] 律所全称（盖章位）
- [ ] 律所负责人签字栏
- [ ] 至少 2 名经办律师签字栏
- [ ] 日期（与封面一致）
- [ ] 独占一页

### 字体 / 段落

- [ ] 正文：宋体，小四号，行距 1.5 倍，首行缩进 2 字符，两端对齐
- [ ] 表格表头：黑体加粗，五号，居中
- [ ] 表格单元格：宋体，五号，两端对齐

### 页眉页脚

- [ ] 奇数页页眉：右对齐 "律师工作报告 / 尽职调查报告"
- [ ] 偶数页页眉：左对齐 "律所全称"
- [ ] 页眉字号小五宋体，灰色
- [ ] 页脚中央：`第 X 页，共 Y 页`
- [ ] 封面、目录不显示页码
- [ ] 附件部分页码重新编号（`附件 X 第 Y 页`）

### 其他

- [ ] 全项目风险汇总表跨页自动重复表头
- [ ] 交叉引用使用 Word REF 域（非硬编码章节号）
- [ ] 所有法规引用首次出现用全称（《xx 法》{修订年份}）
- [ ] 金额格式：大写 + 小写并列，小写带千分位和币种
- [ ] 日期：中文全格式（`2026 年 4 月 24 日`）

## 二、法律意见书（`legal-opinion`）

### 封面

- [ ] 律所中英文名
- [ ] 主标题"{律所全称}关于 {发行人} {项目名} 之法律意见书"
- [ ] 致：客户全称
- [ ] 律所文号（与工作报告文号配套）
- [ ] 日期（与工作报告一致）

### 正文

- [ ] 5 段骨架：引言 / 释义 / 正文 / 结论性意见 / 特别事项提示
- [ ] "二、释义" 段条目数 15-30 条（少于 15 或多于 35 时警示）
- [ ] 正文每章节"事实陈述 → 核查工作 → 法律意见"三步法
- [ ] 结论措辞严格按肯定 / 有条件肯定 / 保留 / 限定 三级
- [ ] "五、特别事项提示"包含所有工作报告"全项目风险汇总表" `级别=高` 事项

### 形式配套

- [ ] 律所名 / 律所英文名 / 签字律师 / 律所文号规则 / 日期 与工作报告一致
- [ ] 编制依据 / 法规清单 与工作报告一致（意见书可多列，不得少列）

### 签字页

- [ ] 律所盖章
- [ ] 律所负责人签字
- [ ] 经办律师签字（至少 2 名，与工作报告一致）
- [ ] 日期

### 字体 / 段落 / 页眉页脚

同律师工作报告，除页眉文字改为"法律意见书"。

## 三、法律备忘录（`legal-memo`）

### 首页元信息

- [ ] 致 / 自 / 日期 / 事由 / 保密 五行（按 `legal-memo-format.md` §一 顺序）
- [ ] 字段填写齐全，无空占位

### 主标题

- [ ] 与"事由"一致或取核心短语
- [ ] 16 磅三号加粗，居中
- [ ] 段前段后各 24 磅

### 各级标题

- [ ] 一级"一、"、二级"（一）"、三级"1、"、四级"（1）"（按 `legal-memo-format.md` §三）
- [ ] 12 磅加粗，首行缩进 2 字符
- [ ] 编号连续无断号

### 正文

- [ ] 字体：楷体_GB2312，小四号
- [ ] 行距 1.5 倍，段前段后 0，首行缩进 2 字符
- [ ] 两端对齐

### 表格 / 法规引用 / 数字格式

- [ ] 表格样式统一（外框 / 内框 / 表头底色）
- [ ] 法规引用首次用全称
- [ ] 数字格式统一（金额大小写并列 + 千分位）

---

## 四、会议文件（`meeting-docs`）

### 通用

- [ ] 主标题（会议名）黑体二号加粗居中
- [ ] 正文宋体小四号，首行缩进 2 字符

### 会议通知

- [ ] 章节结构按 `meeting-docs-format.md` §二（"一、基本情况"至"六、联系方式"）
- [ ] 通知期限合规（或在顶部打 ⚠️）
- [ ] 公司章 + 日期

### 议案

- [ ] 每项议案独立一份
- [ ] "议案背景 / 内容 / 决策依据 / 表决方式"四段
- [ ] 特别决议议案末尾标注"**特别决议（需 2/3 以上）**"
- [ ] 关联议案末尾标注"**关联股东 {名单} 需回避表决**"
- [ ] 中小投资者议案末尾标注"**中小投资者单独计票**"

### 决议

- [ ] 会议开头段：召开时间、地点、出席情况、代表股份 / 占比
- [ ] 每项议案按"普通决议 / 特别决议 / 关联议案 / 中小投资者"格式列示表决结果
- [ ] 表决比例基数正确（出席股东 / 扣除关联回避 / 中小投资者）

### 会议记录

- [ ] 主持人 / 记录人 / 监票人 / 唱票人 / 见证律师 签字栏

### 签到表

- [ ] 列：序号 / 姓名 / 身份证 / 持股数 / 占总股本比例 / 出席方式 / 代理人 / 到场时间 / 签字
- [ ] 末尾：出席股东合计、代表股份合计、占总股本比例

### 授权委托书（股份公司 / 上市公司必备）

- [ ] 议案清单与通知完全一致
- [ ] 同意 / 反对 / 弃权勾选栏
- [ ] 委托有效期明确
- [ ] 委托人签字 / 盖章栏

### 一致性

- [ ] 通知议案 = 议案文本全集 = 决议审议议案 = 表决统计议案（完全一致）
- [ ] 总股本 / 总表决权股份在各文件一致
- [ ] 日期、届次跨文件一致

---

## 五、律所自定义样式（`custom`）

### 必检

- [ ] 律所 `.dotx` 模板中的核心样式（"标题 1"/"标题 2"/"正文"）已识别并应用
- [ ] 律所特定元素（抬头 / 落款 / 水印）已正确嵌入
- [ ] 冲突时的优先级已按配置项执行（章节结构按规范、字体字号按律所）

---

## 六、通用检查（所有文档类型）

### 页面设置

- [ ] A4 页面
- [ ] 页边距按格式规范
- [ ] 奇偶页不同（律师工作报告 / 法律意见书）

### 标题 / 段落结构

- [ ] 所有 Heading 样式应用正确，无断层（不出现从 H1 直接跳到 H3 的情况）
- [ ] 段前段后间距按规范
- [ ] 行距一致

### 自动化元素

- [ ] TOC 域代码已插入并刷新
- [ ] 交叉引用使用 REF 域
- [ ] 脚注 / 尾注字号正确

### 表格

- [ ] 外框 0.5 磅、内框 0.25 磅
- [ ] 表头加粗 + 浅灰底
- [ ] 跨页自动重复表头

### 字体

- [ ] 中文字体按规范（宋体 / 黑体 / 楷体_GB2312 按不同元素）
- [ ] 英文字体按规范（Times New Roman / Arial）

### 页眉页脚

- [ ] 按规范设置
- [ ] 首页（封面）不显示页码

### 收尾

- [ ] 签字页单独一页
- [ ] 无空白页（特别是文档末尾）
- [ ] 文件命名规范：`{原文件名}-{YYYYMMDD}.docx`

---

## 七、格式检查清单输出模板

format-adjust Step 7 输出的 Markdown 清单按以下模板组织：

```markdown
# 格式检查清单

**文档**：{文档路径}
**原始输入**：{Markdown / Word}
**目标格式**：{work-report / legal-opinion / legal-memo / meeting-docs / custom}
**格式规范来源**：{shared/templates/xxx-format.md 的 shasum / 版本}
**处理时间**：{YYYY-MM-DD HH:MM}

## 一、应用统计

- 处理段落数：{N}
- 应用 Heading 样式数：H1 {n1}、H2 {n2}、H3 {n3}、H4 {n4}
- 应用正文样式数：{N}
- 应用表格样式数：{N}
- 自动编号绑定数：{N}
- 交叉引用域代码数：{N}
- 目录深度：{level}，条目数：{N}
- 页眉页脚应用：奇偶页不同 = {true/false}；页码起算章节 = {引言}

## 二、异常 / 警示

- {按具体文档类型的 checklist 逐条列示未通过的项}

## 三、未处理项

- {无法自动处理、需手工修改的项（如未识别的占位符、合并单元格的复杂表格）}

## 四、下一步建议

- {按文档类型给出后续处理建议}
```
