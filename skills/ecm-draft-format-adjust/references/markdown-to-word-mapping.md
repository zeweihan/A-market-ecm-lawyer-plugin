# Markdown → Word 元素映射表

本文件是 `ecm-draft-format-adjust` Step 3 "规范化转换"的核心查找表。涵盖 Markdown 常见元素到 Word 样式的映射。

> 字体 / 字号 / 行距等具体参数详见各格式规范（`shared/templates/work-report-format.md` 等）；本文件只列**映射关系**。

---

## 一、段落元素映射

| Markdown | Word 样式名 / 处理 | 备注 |
|----------|------------------|------|
| `# 一级标题` | Heading 1（按格式规范绑定） | 律师工作报告绑定"章标题"样式，自动编号 |
| `## 二级标题` | Heading 2 | 对应小节 1 或一级标题（依文档类型） |
| `### 三级标题` | Heading 3 | 对应小节 2 或二级标题 |
| `#### 四级标题` | Heading 4 | 对应小节 3 |
| `##### 五级标题` | Heading 5 | 较少用 |
| `###### 六级标题` | Heading 6 | 更少用 |
| 普通段落（非空行间的连续文本） | Normal / 正文 | 首行缩进 2 字符 |
| 空行 | （段落间距）| 不生成空段落 |
| `> 引用块` | Quote / 引用段 | 左右缩进 + 灰色背景（视格式规范） |
| `---` 水平线 | 无（或空段落）| 一般不需要（封面与正文用分页符） |

## 二、行内元素映射

| Markdown | Word 格式 | 处理 |
|----------|---------|------|
| `**加粗**` | 加粗 | 保留 |
| `*斜体*` 或 `_斜体_` | 斜体 | 保留 |
| `***加粗斜体***` | 加粗 + 斜体 | 保留 |
| `~~删除线~~` | 删除线 | 保留（可配置关闭） |
| `` `代码` `` | 等宽字体 | 用 Courier New / 等线 等宽字体 |
| `[链接文字](URL)` | 超链接 | 保留（可配置去除） |
| `![图片](路径)` | 内嵌图片 | 居中对齐，图片上方加图号 |

## 三、列表映射

### 3.1 无序列表（bullet list）

Markdown:
```markdown
- 项目 1
- 项目 2
  - 子项目
```

Word：
- 应用"项目符号"样式
- 第一级默认用 "•"（实心圆点）
- 第二级用 "○"（空心圆点）
- 第三级用 "▪"（实心方块）

### 3.2 有序列表（numbered list）

Markdown:
```markdown
1. 项目 1
2. 项目 2
   1. 子项目
```

Word：
- 应用"编号"样式（自动编号）
- 第一级默认用 "1. 2. 3."
- 第二级用 "(1) (2) (3)"
- 第三级用 "①②③"

**注意**：律师文书中若有序列表对应章节编号（如"一、二、三、"），应该用**标题（Heading）**而非列表；文档结构更清晰。

### 3.3 任务列表（checkbox list）

Markdown:
```markdown
- [ ] 未完成项
- [x] 已完成项
```

Word：
- 保留复选框占位（Word 可用"复选框"控件，需 docx 支持）
- 若不支持，用 "□" / "✓" 字符替代

## 四、表格映射

Markdown:
```markdown
| 列 1 | 列 2 | 列 3 |
|------|------|------|
| A    | B    | C    |
```

Word：
- 应用"标准表格"样式（按格式规范：外框 0.5 磅、内框 0.25 磅、表头加粗 + 浅灰底）
- 表格宽度 100%
- 列宽：自动 / 均等 / 按内容适应（默认均等）
- 表格上方插入"表 N-M  {表名}"占位（由 format-adjust 按章节编号自动填充，或由律师手工填入表名）

### 4.1 表格的跨页处理

- 所有表格的"表头行"设置为"跨页时自动重复"
- 表格行不设置"允许跨页拆分"（避免单行被拆成两页）

### 4.2 复杂表格（合并单元格）

Markdown 不原生支持合并单元格；如输入 Markdown 有合并单元格需求（使用了扩展语法），本 skill 会在格式检查清单中提示"该表格存在合并单元格，转换效果请复核"。

## 五、分页 / 分节

### 5.1 分页符

Markdown 中约定用 `<!-- pagebreak -->` 注释或 `\newpage` 命令插入分页。本 skill 识别后在 Word 中插入"分页符"。

**自动分页**：
- 封面之后自动分页（封面独占一页）
- 目录之后自动分页
- 附件每一页的标题后自动分页（每个附件起始页独立）

### 5.2 分节符

若格式规范要求"附件部分页码重新从 1 开始"，需在"正文末 → 附件一起始"之间插入**分节符**（非分页符），以便页码重新计算。

## 六、页眉页脚

### 6.1 页眉

按格式规范，典型设置：

```
- 奇数页：右对齐 "{文件类型}"（如"律师工作报告"/"法律意见书"）
- 偶数页：左对齐 "{律所全称}"
- 字号：小五号宋体
- 颜色：灰色（#808080）
- 下方加 0.5 磅灰色水平线
```

### 6.2 页脚

```
- 中央：`第 X 页，共 Y 页`（小五宋体，灰色）
- 左右：留空（或按律所要求）
```

### 6.3 页眉页脚的跨节设置

- 封面：不显示页眉页脚
- 目录：不显示页码（可显示页眉）
- 引言起：显示页码（阿拉伯数字，从 1 开始）
- 附件部分：新分节，页码重新从 1 开始（格式 `附件 X 第 Y 页`）

## 七、目录（TOC）

### 7.1 TOC 的插入位置

- 律师工作报告 / 法律意见书：封面之后、引言之前，单独占一页
- 法律备忘录：主标题之后、第一个小节之前（较短时可省略）
- 会议文件：一般不需要目录

### 7.2 TOC 深度

- 默认：2 级（章 + 一级小节）
- `deep` 模式：3 级

### 7.3 TOC 的 Word 实现

使用 Word 的 TOC 域代码：

```
{ TOC \o "1-2" \h \z \u }   （2 级，含超链接，使用标题样式）
{ TOC \o "1-3" \h \z \u }   （3 级）
```

### 7.4 TOC 的样式

- 第一级：小四宋体加粗
- 第二级：小四宋体不加粗
- 第三级：五号宋体不加粗
- 每级段前段后 3 磅
- 页码右对齐，中间虚线引导

## 八、交叉引用

### 8.1 Markdown 中的占位

推荐格式：

```markdown
详见本报告{{XREF: 第五部分.（一）}}
```

本 skill 识别 `{{XREF: 目标}}` 模式后，在 Word 中生成 REF 域代码，指向对应标题。

### 8.2 降级处理

若 Markdown 中用的是普通文本（如"详见本报告第五部分"），本 skill 保留文本，但不生成 Word 域代码（因为目标不明确，可能误匹配）。在格式检查清单中标注"存在未使用占位符的交叉引用，建议改为 `{{XREF: 第N部分.小节M}}`"。

## 九、脚注 / 尾注

Markdown 脚注语法：

```markdown
正文[^1]

[^1]: 脚注内容
```

本 skill 识别后在 Word 中使用**尾注**（律师文书惯例），位置在文档末尾 / 章节末尾（可配置）。

字号：小五号；字体：宋体。

## 十、图片

### 10.1 Markdown 中的图片

```markdown
![图片描述](路径/到/图片.png)
```

### 10.2 Word 中的处理

- 图片居中对齐
- 图片下方自动加"图 N-M  {图片描述}"（图编号按章节）
- 尺寸：自动适应页面宽度（不超过 6 英寸）
- 图片环绕方式：嵌入型（in line with text，不浮动）

## 十一、代码块

Markdown:

````markdown
```python
def hello():
    print("hello")
```
````

Word 处理：
- 用"代码段"样式（等宽字体 Courier New 或 等线，小五号，浅灰底）
- 代码内的缩进保留
- 律师文书中代码块很少见；若出现通常是协议中的公式或数据格式

## 十二、元数据（Front matter）

若 Markdown 以 `---` 分隔的 YAML front matter 开头：

```markdown
---
title: ...
author: ...
---

正文内容
```

本 skill 识别但**不渲染**到 Word 正文（front matter 是文档元数据，不是正文内容）。用户可用 front matter 指定：

- `title: ` 用于文档属性（Word 的"文件 → 属性 → 标题"）
- `author: ` 用于文档属性
- `subject: ` 用于文档属性
- 其他自定义字段保留为元数据

## 十三、特殊字符

- 全角 / 半角：保留 Markdown 原文
- 中英文混排的空格：保留 Markdown 原文（中文排版规范建议中英文间加半角空格，但本 skill 不自动添加以避免误伤）
- 标点：保留（不做中英文标点转换）

## 十四、Word 样式表的选择

若用户提供律所 `.dotx` 模板，本 skill 按以下优先级：

1. 使用 `.dotx` 中定义的 Style（若有"章标题"/"正文"/"签字栏"等律所自定义样式）
2. 未在 `.dotx` 中定义的样式，回退到本仓库格式规范的对应样式
3. 都没有定义的元素，使用 Word Normal 样式

## 十五、关于 Markdown → Word 转换工具的选择

本 skill 实际执行转换时，可能使用以下工具链（按优先级）：

1. **docx 外部 skill 的 Markdown 导入 + 样式注入**（首选，本仓库内已声明依赖）
2. **pandoc**（若 docx 不支持特定语法时）
3. **python-docx + 自定义解析器**（底层兜底；复杂样式用这条）

具体调用模式见 [docx-skill-invocation-patterns.md](./docx-skill-invocation-patterns.md)。
