# ecm-research-reg-search

## 定位与边界

本 skill **负责**：
- 按**五级效力层级**（法律 / 行政法规 / 部门规章 / 规范性文件 / 交易所业务规则）对研究主题
  进行全面检索
- 核实每部法规的**时效性**（现行有效 / 已修订 / 已废止），给出最新版本和被替代关系
- 摘录与主题直接相关的**具体条款原文**
- 输出按 [shared/templates/research-output-format.md](../../shared/templates/research-output-format.md)
  统一格式组织的法规清单、时效性说明、可靠性等级

本 skill **不负责**：
- 对法规体系做**适用性分析、冲突解决、新旧衔接**（由 `ecm-research-reg-study` 做）
- 检索案例（由 `ecm-research-case-search` 做）
- 直接得出合规结论（本 skill 只提供**法源**，结论由调用方 skill 或签字律师作出）

## 免责声明

本 skill 产出的法规清单仅为研究辅助。法规持续修订、部委经常发布新规范性文件、交易所业务规则
更新频繁；律师在作为意见书或尽调报告依据前，**必须亲自到官方数据库（flk.npc.gov.cn / 证监会官网 /
交易所官网）核对最新版本**。完整免责声明见仓库顶层 [DISCLAIMER.md](../../DISCLAIMER.md)。

## 前置依赖

- 输出格式：[shared/templates/research-output-format.md](../../shared/templates/research-output-format.md)（统一输出契约）

## 核心任务

收到法规检索请求后，按 **4 个 Phase** 顺序执行：

1. **Phase 1 — 澄清**：最多 2 轮追问确认主题、层级范围、是否含废止文件
2. **Phase 2 — 五级分层检索**：从上位法到下位规则逐层扫描
3. **Phase 3 — 时效性核验**：每部法规核实"现行有效 / 已修订 / 已废止"
4. **Phase 4 — 输出**：按统一格式输出法规清单 + 条款摘录 + 时效性说明

## Phase 1 — 澄清追问

优先解析用户输入，缺失且影响重大的字段才追问：

| 字段 | 重要性 | 取值 | 缺失时的默认处理 |
|------|-------|------|----------------|
| 研究主题 | 必填 | 自由文本（如"科创板发行条件"、"重大资产重组认定") | 必问 |
| 层级范围 | 中 | 法律 / 行政法规 / 部门规章 / 规范性文件 / 交易所规则 / 全部 | 默认"全部"，五级全扫 |
| 是否含废止文件 | 中 | 是 / 否 | 默认"否"（只列现行有效），但被替代关系必须标注 |
| 板块/发行人类型 | 中 | 主板 / 科创板 / 创业板 / 北交所 / 新三板 | 缺失不追问，检索时全板块覆盖，在条款段注明适用板块 |
| 时间截止 | 低 | 默认至当前日期 | 默认至当前（执行日期） |

**硬性限制**：追问不得超过 2 轮（比 case-search 更严格，因为法规检索主题通常已经比较明确）。

### 追问示例

- 主题过宽：
  > "您检索的是'重大资产重组'的**哪一类**规范？——(a) 上市公司重组 / (b) IPO 申报前重组 /
  > (c) 国有资产重组 / (d) 全部。"
- 层级需限定：
  > "是否只关注**部门规章和交易所规则层面**？（如只要直接可执行的审核依据，可不看上位法律）"

### 2.6 检索执行要点

- **从上到下逐层**：先法律层，再行政法规，再部门规章，再规范性文件，最后交易所规则
- **不要跳层**：即使用户只问"上市规则 XX 条"，也要回溯到上位法（证券法、注册管理办法），
  让读者看到法源链路
- **并行板块**：若主题横跨多个板块（如"再融资"既适用于主板也适用于科创板），应并列列出各板块规则
- **官方 URL 优先**：链接到法规的官方页面 URL；若页面失效，给出数据库名（flk.npc.gov.cn /
  证监会规章库）和法规全称，让用户自行检索

## Phase 3 — 时效性核验

每部被引用的法规都必须核验时效性，输出时明确标注：

### 时效状态分类

| 状态 | 含义 | 标注格式 |
|------|------|---------|
| 现行有效 | 当前仍在施行 | `[现行有效]` |
| 已修订 | 已被修订版替代 | `[已被 YYYY 年 M 月 D 日 XXX 号令修订为新版，原版自 YYYY-MM-DD 停止适用]` |
| 已废止 | 已被明文废止 | `[已废止，YYYY-MM-DD 被 XXX 公告废止，被 《新法规名》 替代]` |
| 失效 | 虽未明文废止但事实上不再适用 | `[事实失效，参照《XXX》]` |
| 部分修订 | 部分条款修订，其余条款继续有效 | `[部分修订，第 X 条自 YYYY-MM-DD 起按修订版执行]` |

### 核验方法

1. **优先查 flk.npc.gov.cn**：国家法律法规数据库对法律、行政法规、部门规章有明确的效力状态标注
2. **查证监会规章库**：证监会官网对规章的废止公告会有单独栏目
3. **查各交易所"规则变动历史"**：交易所规则频繁更新，必看变动记录
4. **交叉验证**：北大法宝等商业数据库的"版本对照"功能可辅助，但最终以官方源为准

### 常见陷阱

1. **名义未废止但实际失效**：例如《首次公开发行股票并上市管理办法》（证监会令第 32 号）在
   注册制办法施行后实际失效，但官网上可能还能找到旧版 PDF；必须标注"事实失效"
2. **征求意见稿被误作正式文件**：证监会、交易所经常发布征求意见稿；**征求意见稿不具有法律效力**，
   检索时必须过滤掉
3. **通知、公告、意见**：有些文件以"通知"、"公告"形式发布，效力等级较低但作为监管依据仍有约束力；
   列入"规范性文件层"
4. **2023 年注册制全面改革的影响**：2023-02-17 前后规章体系大变，涉及 IPO / 再融资 / 重组的
   规章很多都在当天全面替换；旧规章的引用都要警惕时效性

### 核心结论段

"1. 核心结论"应直接回答用户的主题，用 3–6 句概括**适用法源图景**：

> 1. 本主题在法律层由《公司法》第 X 条和《证券法》第 X 条提供上位法依据 [L-01][L-02]
> 2. 具体操作规范在部门规章层，由证监会令 205 号《首发注册管理办法》第 X 条规定 [L-04]
> 3. 科创板场景下还需适用交易所规则《科创板上市规则》第 X.X.X 条 [L-06]
> 4. 废止注意：《首发并上市管理办法》（32 号）已被 205 号办法全面替代，引用旧规章将导致意见书错误 [L-07]

### 时效性段必写

"3. 时效性与局限性"段必须包含：
- 本次检索截止日期
- 最近 6 个月是否有征求意见稿尚未定稿（尤其在制度快速变动期）
- 未访问到的数据源

## 硬性输出要求

1. **五级分层**：输出**必须**按法律→行政法规→部门规章→规范性文件→交易所规则的顺序组织
2. **时效性必注**：每条法规标注 `[现行有效 / 已修订 / 已废止 / 事实失效 / 部分修订]`
3. **被替代关系**：若引用的法规已有新版，必须注明新法名称 + 替代日期
4. **不得引用征求意见稿作为正式依据**：若使用征求意见稿，必须在引证处注明"尚在征求意见，
   不具备正式效力"
5. **官方 URL 优先**：链接到官方数据库；避开第三方聚合站点
6. **条款原文引用加引号 + 出处**：条款原文用 `>` 引用块，末尾给出版本日期
7. **每条方括号编号**：[L-01][L-02]……便于结论段反向引用
