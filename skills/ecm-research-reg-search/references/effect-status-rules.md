# 时效性核验规则（Effect Status Rules）

本文件规定 `ecm-research-reg-search` 在 Phase 3（时效性核验）时的判断方法和标注格式。

---

## 一、时效状态的五种分类

| 状态 | 含义 | 判断依据 |
|------|------|---------|
| **现行有效** | 当前仍在施行 | 官方数据库标注为"有效"；无废止公告；施行日期已到 |
| **已修订** | 已有新版替代 | 存在修订后版本；通常旧版本自新版施行日起停止适用 |
| **已废止** | 明文废止 | 存在废止公告（明确说明"废止 XXX"） |
| **事实失效** | 未明文废止但已不再适用 | 上位法已修改导致本法规失去依据；或主管机关在答复/指引中明确不再适用 |
| **部分修订** | 部分条款修订，其余仍有效 | 存在修订决定只修改部分条款 |

---

## 二、核验步骤

对每一部将要列入输出的法规，按以下顺序核验：

### 步骤 1：查官方数据库

首选 [国家法律法规数据库](https://flk.npc.gov.cn) 或对应发布机关的官方网站。
数据库上每部法规通常有以下标注：
- "现行有效"
- "已失效"、"已废止"
- "已修订"（并给出新版链接）
- "部分修订"

若官方数据库有明确标注，**以数据库标注为准**。

### 步骤 2：查废止公告

若官方数据库状态未明确，在以下位置查废止公告：
- 证监会官网 → 信息公开 → 废止公告
- 各部委官网相应栏目
- 交易所规则变动历史

### 步骤 3：查版本更替

若没有废止公告，但存在**同主题新规章**：
- 新旧规章的施行日期
- 新规章是否明确规定"本办法施行之日起，XXX 同时废止"
- 若新规章未明文废止旧规章，但**两者存在实质冲突**，按"新法优于旧法 + 特别法优于一般法"
  规则判断，通常得出"旧法事实失效"结论
- 此种情况标注为 **事实失效** 并说明判断依据

### 步骤 4：交叉验证

- 北大法宝 / 威科先行等商业数据库对效力状态的标注可交叉验证
- 两者有差异时，以**官方数据库 + 官方公告**为最终依据

### 步骤 5：特别警示

以下情况需特别警示：
- **2023 年注册制全面改革**（2023-02-17）前后的规章大换血——旧的"IPO 并上市管理办法"类
  规章基本都已废止或被替代
- **2024 年公司法修订**（2024-07-01 施行）后，旧公司法条款需要映射到新法条款
- 交易所规则经常用"自 YYYY-MM-DD 起施行"方式更新，**必看生效日期**

## 三、标注格式

### 现行有效

```
《XXX》（YYYY 年 M 月 D 日公布，YYYY 年 M 月 D 日起施行）[现行有效]
```

### 已修订

```
《XXX（2018）》（原文号，原施行日期）[已修订，YYYY 年 M 月 D 日被《XXX（2024 修订）》（新文号）
替代，原版自 YYYY-MM-DD 停止适用]
```

若正在引用的是**旧版**原文，应明确说明：
```
本条引用的是《XXX（2018）》第 X 条，该版本已于 YYYY-MM-DD 被《XXX（2024 修订）》替代。
对应现行版本为《XXX（2024 修订）》第 X 条。
```

### 已废止

```
《XXX》（原文号）[已废止，YYYY-MM-DD 被《废止决定 XXX 号》明文废止，后续适用《替代法规全称》]
```

### 事实失效

```
《XXX》（原文号）[事实失效，虽未明文废止，但 YYYY 年施行的《XXX 新法》已全面替代其功能，
实务中不再作为审核依据引用]
```

### 部分修订

```
《XXX》（原文号，YYYY 年部分修订）[部分修订，第 X 条、第 Y 条自 YYYY-MM-DD 起按 YYYY 年
修订版本执行，其余条款保持不变]
```

## 四、常见误判陷阱

### 陷阱 1：征求意见稿被误作正式文件

- 表现：文件名包含"（征求意见稿）"或"（公开征求意见）"
- 风险：征求意见稿不具备法律效力，**不得作为意见书依据**
- 应对：检索时**默认过滤**征求意见稿；只在用户明确要求"追踪在途立法"时才列出，且标注
  `[征求意见稿，尚未生效]`

### 陷阱 2：审议通过但未公布的法规

- 表现：人大常委会已通过但尚未主席令签署公布的法律
- 风险：尚未施行的法律不能作为现行依据，但**已批准的公司法、证券法修正案**若距施行日不远，
  应同时提示"将自 YYYY-MM-DD 起施行新版"
- 应对：现行有效的仍用旧版引用；同时在"3. 时效性与局限性"中提示即将施行的新法

### 陷阱 3：地方性法规 vs 全国性法规混淆

- 例如：地方金融监管局规定 vs 证监会规定
- 应对：**以全国性法规为准**；地方性法规只在主体或标的与该地域相关时列出

### 陷阱 4：窗口指导未正式发文

- 表现：证监会发行监管部通过**非正式渠道**（电话答复、行业协会通报）传达的口径
- 风险：律师口耳相传的"窗口指导"可能不准确，且没有书面依据
- 应对：**不作为正式引用**；如确需提及，标注为**C 级 - 公开报道/二手资料**，明确说明无官方书面依据

### 陷阱 5：实务中有效但官网未公布

- 表现：某些部委指引只在内部下发，从未公开
- 应对：不作为引用依据，标注为 D 级

### 陷阱 6：被替代关系认定错误

- 表现：新法明文只废止 A 办法，但实际上还与 B 办法冲突
- 应对：**以明文废止为准**；对 B 办法按"新法优于旧法"原则判断"事实失效"，并说明判断理由

## 五、时效性核验的 fallback

若无法访问官方数据库（WebSearch/WebFetch 不可用）：

1. **第一层**：引用 `shared/regulations/` 下本仓库已节选的法规文本（如有）
2. **第二层**：请用户提供法规原文
3. **第三层**：基于模型记忆给出初步结果，但**全部**标注为 D 级（待核实）
4. **绝不**：编造生效日期、编造文号、编造废止公告

fallback 模式下，"3. 时效性与局限性"段必须置顶声明：

> ⚠️ 本次检索未能访问官方数据库（flk.npc.gov.cn / 证监会规章库 / 各交易所官网）。
> 下列法规的时效性状态**全部基于 Claude 训练数据**，在引用前**必须**到官方源核对
> 最新版本和效力状态。

---

## 六、输出时的时效性段落

所有 `ecm-research-reg-search` 的输出都必须在"3. 时效性与局限性"段包含：

```markdown
## 3. 时效性与局限性

- **检索执行日期**：YYYY-MM-DD
- **法规最新版本截至**：YYYY-MM-DD（根据官方数据库显示日期）
- **已涵盖的数据源**：国家法律法规数据库 flk.npc.gov.cn | 证监会规章库 | 上交所规则专区 | ……
- **未能访问的数据源（如有）**：YYY
- **在途立法提示（如适用）**：《XXX（征求意见稿）》自 YYYY-MM-DD 起征求意见，尚未定稿；
  若立法通过，本研究结论可能需要更新。
- **建议复核节点**：
  - 在意见书定稿前，律师应到 flk.npc.gov.cn 核对 [L-01][L-02]……的最新版本
  - 关注 XXX 主题的征求意见稿动态（若有）
```

缺少这一段的输出**不合格**。
