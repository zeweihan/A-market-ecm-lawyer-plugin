# 会议文件内核审查 — 跨文件一致性交叉比对矩阵

本矩阵适用于 **full-set 模式**（用户提交成套会议文件）下的跨文件一致性校对。
跨文件一致性是会议文件内核的**核心价值**。

> 参考坐标：同时考察"所有会议文件之间的一致性"（互为参考源）+ "与公司章程的合规性"。

---

## 十项核心跨文件核对字段

### M1 — 议案清单一致

**核对方法**：提取以下四处的议案清单（编号 / 全称 / 议案类型标注），逐项对照：

| 来源 | 提取字段 |
|------|---------|
| 会议通知 | 第二部分"会议审议事项"下的议案编号和全称 |
| 议案文本集 | 每份议案的标题 |
| 会议决议 | "会议审议通过 / 否决议案如下"下的议案清单 |
| 表决统计表 | 每行议案编号和名称 |

**一致要求**（`meeting-docs-format §十`）：
- 编号完全一致（1、2、3…不能跳号 / 错号）
- 名称完全一致（字符级；"《关于 xxx 的议案》"而非"关于 xxx 的议案")
- 顺序完全一致
- 议案类型标注（特别决议 / 关联议案 / 中小投资者单独计票）一致

**偏差处理**：
- 编号或名称不一致：tracked change 改齐（以通知为准 or 以最近一次公告为准） + 【必改】批注
- 顺序不一致：tracked change 调整
- 议案类型标注不一致：【必改】批注（见 M5 / M6 / M7）

**批注模板**：
> 【必改】议案清单在跨文件间不一致：
> - 会议通知：议案 {N} "{通知中的议案名}"
> - 议案文本：议案 {N} "{议案文本标题}"
> - 会议决议：议案 {N} "{决议中的议案名}"
> 按 `meeting-docs-format §十`，通知中的议案清单是全套文件的**议案母版**，所有其他文件的议案列示必须完全一致。请统一。

### M2 — 股权登记日一致

**核对来源**：

| 来源 | 字段 |
|------|------|
| 会议通知 | "（二）股权登记日：YYYY 年 MM 月 DD 日" |
| 签到表 | "持股基准日" 或"股权登记日" |
| 授权委托书 | 引用的股权登记日 |

**偏差处理**：tracked change 改齐 + 【必改】批注（以通知为准）。

---

### M3 — 总股本 / 总表决权股份一致

**核对来源**：

| 来源 | 字段 |
|------|------|
| 会议通知 | "发行人总股本 XXX 股" |
| 签到表 | "占总股本 XX%" 的分母 |
| 会议决议 | "代表股份 XXX 股，占公司总股本 XX%" |

**偏差处理**：tracked change 改齐 + 【必改】批注。

---

### M4 — 会议基本情况一致

**核对字段**：
- 召开时间（YYYY-MM-DD HH:MM）
- 召开地点
- 召集人
- 主持人
- 召开方式（现场 / 通讯 / 视频 / 现场+网络投票）

**核对来源**：

| 来源 | 取字段 |
|------|-------|
| 会议通知 | 一、本次会议召开的基本情况 |
| 会议决议 | 文首段"会议于 YYYY 年 MM 月 DD 日在 XX 召开" |
| 会议记录 | 会议召开基本情况 |

**偏差处理**：tracked change 改齐 + 【必改】批注。

**高频错**：通知写"现场+网络投票"，但决议 / 记录只写"现场"（未记录网络投票时段）。

---

### M5 — 特别决议标注一致

**核对**：议案标注为"特别决议"的，应在以下三处同步出现：

| 来源 | 表现形式 |
|------|---------|
| 议案文本 | 第四段"本议案适用特别决议表决，需经 2/3 以上表决通过" |
| 会议通知 | 议案名前或议案清单后标注"（特别决议）" |
| 会议决议 | 议案表决结果后"本项议案为股东（大）会特别决议事项，已经出席本次股东（大）会的股东及股东代理人所持表决权的三分之二以上同意通过" |

**偏差处理**：
- 应标未标：【必改】批注要求补充标注（参考 `meeting-docs-format §三` 标准话术）
- 错标：【必改】批注要求删除
- 部分标注：tracked change 补齐

---

### M6 — 关联股东 / 董事回避一致

**核对**：关联议案的回避安排应在以下处同步出现：

| 来源 | 表现形式 |
|------|---------|
| 议案文本 | "关联股东 XX / YY / ZZ 需回避表决" |
| 会议通知 | 议案清单旁标注"关联股东需回避" |
| 会议决议 | 议案表决结果前"关联股东 XX / YY / ZZ 回避表决，本议案由其他股东参与表决" |
| 表决统计表 | 剔除关联股东后的表决票数 |

**偏差处理**：【必改】批注（关联回避是**效力要件**）。

---

### M7 — 中小投资者单独计票一致（上市公司）

**核对**：上市公司特定议案需中小投资者单独计票：

| 来源 | 表现形式 |
|------|---------|
| 议案文本 | "本议案为中小投资者单独计票议案" |
| 会议通知 | 议案清单旁标注"中小投资者单独计票" |
| 会议决议 | 议案表决结果应**同时列出** "整体表决结果" + "中小投资者表决结果" |
| 代理投票计票细则 | 适用中小投资者单独计票的议案清单 |

**偏差处理**：【必改】批注要求三处同步。

---

### M8 — 见证律师签字（如有见证业务）

**核对**：如股东（大）会有律师见证，以下处应体现：

| 来源 | 表现形式 |
|------|---------|
| 会议通知 | 出席对象含 "公司聘请的见证律师" |
| 会议记录 | "见证律师：XX 律师事务所 XX 律师"（签字处） |
| 见证法律意见书 | 由见证律师独立出具 |

**偏差处理**：一致性缺失 → 【核实】批注（是否实际有见证）。

---

### M9 — 公司性质与适用规则一致

**核对**：本套会议文件引用的法规依据应与公司性质匹配：

| 公司性质 | 必引 | 不得引 |
|---------|------|-------|
| 上市公司 | 《公司法》《证券法》《上市公司股东会规则》（2025 修订）《上市公司章程指引》 | - |
| 新三板挂牌 | 《公司法》《非上市公众公司监督管理办法》《全国股转系统业务规则》 | 《上市公司股东会规则》 |
| 非上市股份公司 | 《公司法》《公司章程》 | 《上市公司股东会规则》《证券交易所规则》 |
| 有限责任公司 | 《公司法》《公司章程》 | 任何"股东大会"表述（应为"股东会"） |

**偏差处理**：【必改】批注（如"有限公司会议通知中引用《上市公司股东会规则》" 是常见错误）。

---

### M10 — 公司章程依据一致

**核对**（仅 Step 1.5 拿到公司章程时）：

| 核对项 | 方法 | 偏差处理 |
|-------|------|---------|
| 通知期限是否符合章程约定（若章程更严） | 比对通知实际发出日 vs 会议日 vs 章程约定天数 | 不足：【必改】【风险提示】 |
| 表决比例（特别决议 2/3 vs 章程约定的更高比例） | 检查章程约定 | 【核实】批注要求按章程调整 |
| 召开方式（是否允许视频 / 通讯） | 检查章程 | 【核实】批注 |
| 召集人权限 | 检查章程 | 【核实】批注 |

---

## 跨届次连续性核对（post-meeting 可选）

如本次是"第 X 届第 Y 次董事会"，核对与上次（第 X 届第 Y-1 次）、下次（第 X 届第 Y+1 次）的届次编号连贯性。Level 1 自动扫描项目目录下 `04-文件输出/会议文件/` 的其他届次。
