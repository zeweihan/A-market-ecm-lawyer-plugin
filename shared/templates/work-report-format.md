# 律师工作报告 / 尽职调查报告 排版规范（Single Source of Truth）

本文件是**唯一权威**的中国 A 股证券律师"律师工作报告"及"尽职调查报告"的文档结构与排版定义。凡本仓库内 skill 输出该两类报告文书，必须引用本规范而**不得**在各自 SKILL.md 里重复定义。

本规范既是 Markdown 初稿骨架的约束，也是 Markdown → Word 套版（由 `ecm-draft:format-adjust` 或外部 `docx` skill）的样式参数源。

---

## 适用范围

- `ecm-draft:report-assembly` 输出的《律师工作报告》和《尽职调查报告》主文
- `ecm-qc:work-report-review`对上述文书的审查产出（以本规范为"参考坐标"指认项目组报告的格式偏离）

**不适用**于：
- 法律意见书 → `shared/templates/legal-opinion-format.md`
- 法律备忘录 → `shared/templates/legal-memo-format.md`
- 会议文件 → `shared/templates/meeting-docs-format.md`
- 见证意见 / QC 审查稿 → 独立格式

## 一、报告类型与差异

| 类型 | 英文 | 主要用途 | 与本规范的关系 |
|------|------|---------|--------------|
| 律师工作报告 | Work Report | 向监管机关提交的全面尽调工作过程与结论，与"法律意见书"配套 | 主用：完整沿用本规范 |
| 尽职调查报告 | Due Diligence Report | 给客户内部使用的尽调全景，不对监管披露 | 次用：可精简附件，正文沿用 |

两类报告在正文结构、章节顺序、签字页、骑缝章位上完全一致；差异仅在：
- 律师工作报告需要**与法律意见书形式配套**（同一律所、同一签字律师、同一编号逻辑）
- 尽职调查报告允许在首页加"仅供客户内部使用，不得对外披露"的脚注，律师工作报告不加

## 二、首页封面

封面元素从上到下（所有元素居中）：

```
{律所全称（中文，三号宋体加粗）}
{律所全称（英文，小四 Times New Roman，斜体）}

{律师工作报告 / 尽职调查报告}
（主标题，二号黑体加粗，段前段后各 36 磅）

（关于……的）
{项目事由，三号楷体_GB2312，单行 / 两行内）

致：{客户全称}
（小四宋体）

{律所文号：如 XX 律字[2026]第 XXX 号}
（小四宋体；草稿期可写"草稿，尚未完稿"）

{YYYY 年 MM 月 DD 日}
（小四宋体，中文格式）
```

### 封面页技术要求

- 页面设置：A4，上 3.5 cm、下 2.5 cm、左右 2.5 cm（比正文页略宽）
- **骑缝章位**：封面不盖骑缝章；正文每页右侧留 1.5 cm 空白供骑缝章（本规范层面体现为 Word 页边距设置，Markdown 层面无需特别标注）
- 封面与正文之间**插入空白分页符**，封面独占一页

## 三、正文章节结构（与《编报规则第 12 号》锁齿）

正文按《编报规则第 12 号》章节顺序组织。每个章节标题形如：

```
第一部分  本次交易的批准和授权
第二部分  发行人的主体资格
第三部分  发行人的设立
……
第十七部分  其他合规事项
```

### 章节内小节（来自 DD Memo 的五段式）

每章内部完整保留 DD Memo 的五段骨架（不改名、不改序号）：

```
第N部分  {章节标题}

一、核查要点清单
（引用 DD Memo 对应段落）

二、审阅发现
（引用 DD Memo 对应段落）

{若干 skill 专属表，如股本演变时间线表、董监高花名册——原样嵌入}

三、风险分级汇总
（引用 DD Memo 对应段落；此处的表参与末尾的全项目风险汇总）

四、结论与建议
（引用 DD Memo 对应段落）

五、参考资料
（引用 DD Memo 对应段落）
```

### 章节编号规则

- 大标题：`第一部分` / `第二部分` / ……（汉字编号，四号黑体加粗）
- 一级小节：`一、核查要点清单`（小四黑体加粗）
- 二级小节：`（一）xxx`（小四黑体，不加粗）
- 三级小节：`1、xxx`（小四宋体，不加粗）
- 四级小节：`（1）xxx`
- 五级小节：`①②③`（仅在必要时使用）

## 四、首尾固定件（所有报告通用）

### 4.1 首页之后、正文之前

**第一页：《律师工作报告》/《尽职调查报告》 的引言段**

```
一、引言

（一）本次发行 / 交易概况
（二）本所及经办律师承办概况
（三）编制依据（法律法规清单）
（四）核查范围与方法
（五）关于本报告的使用
```

引言段格式：小四宋体，段前段后各 6 磅，首行缩进 2 字符。

### 4.2 正文之后

**附件部分**（按以下顺序）：

1. 附件一：核查文件清单（按 17 个 DD 章节分组列示）
2. 附件二：律师出具人员清单（附执业证号）
3. 附件三：律所资质证明
4. 附件四：全项目风险分级汇总表（由 report-assembly 聚合生成）
5. 附件五（可选）：专项核查工作底稿索引

附件页的一级标题格式："附件一  {附件名}"（三号黑体加粗，段前 36 磅，另起一页）。

### 4.3 末页（签字盖章页）

固定格式：

```
{律所全称}（盖章）：

{经办律师姓名}（签字）：_____________________

{经办律师姓名}（签字）：_____________________

{律所负责人姓名 / 分所负责人姓名}（签字）：_____________________

{YYYY 年 MM 月 DD 日}
```

- 律所印章位于律所全称下方、签字栏上方
- 签字律师至少 2 名（其中 1 人为签字律师）
- 报告厚度较大时可加骑缝章说明（如正文附件总计 N 页）

## 六、编号、交叉引用、目录

### 6.1 章节编号的自动化

报告的"第N部分"编号**必须**用 Word 自动编号样式（Heading 1 绑定"第一部分、第二部分、……"列表），而不是硬编码。否则中途增删章节时会出现 `第一部分 / 第三部分 / 第三部分` 的断号事故。

Markdown 初稿阶段可以用固定汉字；`ecm-draft:format-adjust` 套版时转为自动编号。

### 6.2 交叉引用

- 章节内引用："详见本部分'三、风险分级汇总'"
- 跨章节引用："详见本报告第四部分'独立性'"
- 引用附件："详见附件一'核查文件清单'"

禁止使用"上文""下文""前述"等方位词（Word 排版调整后顺序会变）。

### 6.3 目录

- 深度：2 级（章 + 一级小节）
- 位于：封面之后、引言之前，单独占一页 / 多页
- 格式：第一级小四宋体加粗；第二级小四宋体不加粗；页码右对齐，中间虚线引导
- 标题："目　录"（居中，三号黑体加粗，字间插入一个全角空格）

## 七、表格样式

所有表格遵循：

- 表格宽度：100% 页面宽度
- 外框：0.5 磅黑色实线
- 内框：0.25 磅黑色实线
- 表头：浅灰底（#F2F2F2）、加粗
- 表格上方有表号和表名：`表 5-1  股权结构穿透表`（五号黑体加粗，左对齐，表格前预留 6 磅）
- 表格跨页时，表头行自动重复

### 全项目风险分级汇总表（由 report-assembly 自动生成）

这是报告**必须有**的末尾总表，列定义如下：

```markdown
| 序号 | 所属章节（第 N 部分） | 问题 | 级别 | 所涉文件 | 法规依据 | 建议措施 |
|------|---------------------|------|------|---------|---------|---------|
| 1    | 第一部分  批准和授权   | ...  | 高   | ...     | ...     | ...     |
```

列定义：
- "序号" 跨章节连续编号（不按章节重置）
- "所属章节" 格式统一："第N部分  {章节标题}"
- "级别" 限 `高/中/低`
- "建议措施" 取自各章节"四、结论与建议"段的简短汇总

表上方加导读段："本表汇总发行人在全部 17 项尽调维度下识别的问题，按风险级别提示，具体分析详见各部分。"

## 八、数字 / 金额 / 日期格式

- **金额**：大写（伍万元）+ 小写（人民币 50,000 元）并列；小写金额必须加币种和千分位
- **比率 / 百分比**：保留两位小数（如 `33.33%`）；上下游无强制时统一整数（`50%`）
- **日期**：中文全格式（`2026 年 4 月 24 日`）；附件 / 表格内可用 `2026-04-24`
- **法规引用**：`《中华人民共和国公司法》（2023 年修订）第 46 条` 首次完整出现；同一报告后续可简化为"《公司法》第 46 条"
- **法条条号**：阿拉伯数字（`第 146 条`），不用中文数字

---

## 九、语言风格

- **第一人称**：律师事务所自称"本所"；律师自称"本所经办律师"
- **避讳主观判断词**：不用"显而易见""毫无疑问""应当无疑"；改为"经核查，……"
- **结论性措辞**：对合法性发表意见用"合法有效"/"符合《xx 法》第 yy 条的规定"；存在瑕疵用"存在下列需提请注意的事项"
- **保留 / 限定意见措辞**：用"除……外"/"在……的前提下"（不用"基本符合"）

## 十、页眉页脚

### 页眉

- 奇数页：**右对齐**，"{律师工作报告 / 尽职调查报告}"
- 偶数页：**左对齐**，"{律所名}"
- 字号：小五号宋体，灰色

### 页脚

- 中央：`第 X 页，共 Y 页`（小五宋体）
- 封面、目录页不显示页码
- 从"引言"起算页码（阿拉伯数字）
- 附件部分页码**重新从附件一的第 1 页开始**，格式 `附件 X 第 Y 页`

## 十二、常见踩坑（report-assembly 注意事项）

1. **DD Memo 级别取值出现"红/黄/蓝"或"严重/一般/轻微"** → 违反 `dd-output-schema.md` 契约 → 不得静默转换，直接按 § 7 "违约处理"跳过该 Memo
2. **章节号与章节名不匹配**（Memo 标 "编报规则第 3 章" 但实际是第 4 章内容） → 以 Memo 的 `name` 字段 → dd-output-schema.md § 0 映射表 为准
3. **专属表（如股权结构图）在 Word 转换后错位** → 用图片而非 Markdown 渲染；ASCII art 图表须标注"图示仅供参考，正式版以 PDF/ 图片为准"
4. **跨章节交叉引用断链** → report-assembly 生成目录时重新计算所有交叉引用（优先使用 Word 域代码，Markdown 阶段用占位符 `{{XREF: 第N部分.小节M}}`）
5. **签字页日期晚于报告日期** → 不合规；签字页与封面日期必须一致
6. **全项目风险汇总表"序号"列跨章节未连续** → report-assembly 聚合时重新编号
