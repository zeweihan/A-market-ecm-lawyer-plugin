# ECM 法律研究类 skill 统一输出格式（Single Source of Truth）

本文件定义 `ecm-research-*` 系列所有 skill 的输出契约。其他 skill 在引用 research skill 的结果时，也按本约定消费输入。

## 核心原则

1. **每一条结论必须可溯源**——任何引用的法规或案例，都必须给出"引证"+"出处"+"访问日期"，以便律师独立核对。
2. **可靠性必须分级标注**——不同来源的案例/法规可靠性不同（官方数据库 > 商业数据库 > 公开报道 > 模型记忆），读者必须知道哪一类。
3. **时效性必须明示**——法规区分"现行有效 / 已修订 / 已废止"；案例区分"终审 / 非终审 / 审理中"。
4. **不得虚构**——当信息不可得或来源不可靠时，明确标注"待核实"或"无直接依据"，不得编造。

---

## 一、可靠性等级（Reliability Tier）

所有输出必须标注每条结论所依据材料的**可靠性等级**。四级分类：

| 等级 | 含义 | 典型来源 |
|------|------|---------|
| **A - 官方原始** | 来自立法机关 / 监管机关 / 司法机关官方网站的原始文本 | 国家法律法规数据库（flk.npc.gov.cn）、中国裁判文书网、证监会/交易所官网、最高法/最高检官网 |
| **B - 官方汇编/商业数据库** | 官方汇编出版物或主流法律数据库 | 北大法宝、威科先行、LexisNexis、万律等的有权威编辑背书的条文 |
| **C - 公开报道/二手资料** | 媒体报道、律所公众号文章、学术文章中引用 | 财新、证券时报、律所研究报告、学者论文 |
| **D - 模型记忆/未核实** | 仅由 Claude 的训练数据给出，未经独立检索 | 本 skill 的缺省状态——必须写成"待核实"，不得径行作为结论依据 |

**硬性规则**：
- 正文结论**不得**仅依据 D 级材料。凡 D 级材料形成的初步结论，必须在输出里明确用"（待核实）"标注，并给出后续核实路径（例如"请在国家法律法规数据库核对"）。
- A 级和 B 级材料可作为正式结论依据。
- C 级材料可作为补充参考，但需和 A/B 级材料相互印证后方可写入结论段。

## 二、统一引证格式

### 1. 法规引证

**格式**：`《法规全称》（文号/年份）[效力状态] 第 X 条（主题）`

示例：
- `《中华人民共和国公司法》（2023 年修订，2024 年 7 月 1 日起施行）[现行有效] 第 112 条（董事会决议）`
- `《首次公开发行股票注册管理办法》（证监会令第 205 号，2023 年 2 月 17 日公布，2023 年 2 月 17 日起施行）[现行有效] 第 13 条（发行条件）`
- `《上市公司证券发行管理办法》（证监会令第 30 号，2006 年 5 月 8 日公布）[已废止，2020 年 2 月 14 日被证监会令第 163 号废止]`

**字段清单**：
- 法规全称（不得使用简称或缩写）
- 文号（如有）
- 公布日期 / 施行日期
- 效力状态（现行有效 / 已修订 / 已废止；已修订需注明被哪部修订法替代）
- 所引条款 + 条款主题（便于读者定位）

### 2. 案例引证

**格式**：`[案件类型] 案号 / 文书号（处罚对象/当事人）[审级/终审状态]`

示例：
- `[最高法判决] （2019）最高法民终 1524 号（某某 v. 某某某，对赌协议效力纠纷）[终审]`
- `[证监会处罚决定书] [2023] 5 号（某某某上市公司虚假陈述案）[已生效]`
- `[上交所纪律处分决定书] 上证公监字〔2024〕11 号（某某公司董事长减持违规）[已生效]`
- `[上市委审议意见] 2024 年第 17 次审议会议（某某科技科创板 IPO 问询案）[审议通过/暂缓审议/终止审核]`

**字段清单**：
- 案件类型标签（用方括号包裹）：裁判文书、行政处罚决定书、纪律处分决定书、审议意见、问询回复等
- 案号或文书号（官方编号）
- 当事人/处罚对象（用"某某"脱敏或直接写实名，依案例公开程度决定）
- 审级 / 终审状态 / 是否生效

### 3. 出处（Source）

每条引证后必须给出**可核对的出处**：

```
出处：<数据库名或网址> | 访问日期：<YYYY-MM-DD> | 可靠性：A/B/C/D
```

示例：
```
出处：国家法律法规数据库 https://flk.npc.gov.cn/detail2.html?ZmY4MDgwODE4MDY5OTM5ODAxODA5ZWJiZTE1YTA3N2M%3D | 访问日期：2026-04-24 | 可靠性：A
出处：中国裁判文书网 wenshu.court.gov.cn（案号检索）| 访问日期：2026-04-24 | 可靠性：A
出处：北大法宝 pkulaw.com | 访问日期：2026-04-24 | 可靠性：B
出处：证监会官网行政处罚专栏 | 访问日期：2026-04-24 | 可靠性：A
出处：律师公众号"某某法律评论"2024 年 6 月 12 日推文 | 访问日期：2026-04-24 | 可靠性：C
出处：未核实，基于 Claude 训练数据 | 可靠性：D
```

## 三、统一输出结构

### 共用外层结构

所有 research skill 的输出都遵守以下外层四段结构：

```markdown
# <研究主题>

## 0. 检索/研究策略

- **研究主题**：<用户输入或经澄清后的主题>
- **范围**：<检索范围：哪些数据库 / 哪些层级 / 哪些年份>
- **关键词**：<使用的检索关键词和组合>
- **执行时间**：<YYYY-MM-DD HH:MM>
- **执行方式**：<WebSearch / WebFetch / 用户上传文件 / 模型记忆（标注 D 级）>

## 1. 核心结论

<用 3–8 句话直接回答用户的研究问题。每条结论后方括号内标注支撑材料的引证编号，例如 [L-01] [C-03]>

## 2. 支撑材料

### 2.1 法规清单（若适用）

<按效力层级排列的法规列表，每条含完整引证 + 出处 + 可靠性等级；用 L-01、L-02 编号>

### 2.2 案例清单（若适用）

<按时间倒序排列的案例列表，每条含完整引证 + 事实摘要 + 争点 + 裁判/处罚理由 + 结论；用 C-01、C-02 编号>

## 3. 时效性与局限性

- **时效性说明**：本次输出最新法规版本至 <YYYY-MM-DD>；<YYYY-MM-DD> 后是否有新修订**未核实**。
- **检索局限**：<哪些数据源未访问到 / 哪些关键词可能遗漏 / 哪些结论依赖 D 级材料>
- **建议复核**：<律师需独立核对的关键节点>

## 4. 下一步建议

<基于本次检索结果，建议用户下一步做什么——例如调用其他 skill、补充资料、发起访谈、向监管询问等>
```

### 子段的格式细节

**法规条目**（2.1 节内）：

```markdown
**[L-01]** 《中华人民共和国证券法》（2019 年修订，2020 年 3 月 1 日起施行）[现行有效] 第 9 条（发行条件）

> 公开发行证券，必须符合法律、行政法规规定的条件……

出处：国家法律法规数据库 flk.npc.gov.cn | 访问日期：2026-04-24 | 可靠性：A

要点：本条确立了公开发行证券的核准/注册原则，为后续《注册管理办法》《发行承销办法》提供上位法依据。
```

**案例条目**（2.2 节内）：

```markdown
**[C-01]** [最高法判决] （2019）最高法民终 1524 号（A 公司 v. B 投资，对赌条款效力纠纷）[终审]

- **案件事实**：<2–4 句>
- **争议焦点**：<核心法律问题>
- **裁判要旨**：<法院的核心说理>
- **结论**：<判决主文要点>
- **对本项目的启示**：<2–3 句，结合用户的研究主题>

出处：中国裁判文书网 wenshu.court.gov.cn | 访问日期：2026-04-24 | 可靠性：A
```

## 四、Fallback 规则（外部检索不可用时）

当 WebSearch / WebFetch 不可用、目标数据库无法访问、或用户未连接网络时：

1. **第一层 fallback**：引用本仓库 `shared/regulations/` 下已节选的法规条文（如果有）。
2. **第二层 fallback**：请用户把相关法规或案例文本作为附件上传，由 Claude 从上传文本中提取。
3. **第三层 fallback**：基于模型记忆给出**初步分析**，但必须：
   - 全篇以 **D 级（待核实）** 标注
   - 在"1. 核心结论"之前置顶声明："⚠️ 本次输出未经独立检索核对，所有结论均为待核实状态，律师**不得**直接作为意见书或尽调报告的依据。"
   - 在"4. 下一步建议"里明确告诉用户应去哪些官方数据源核实
4. **绝不编造**：宁可输出"未能检索到相关案例/法规，建议律师人工检索以下来源：……"，也不得生成虚假案号、虚假文号、虚假条款编号。

## 五、检索关键词的建议组合

所有 research skill 在执行检索时，建议先输出**检索策略**供用户确认，再执行。关键词组合的常见模式：

- **基础关键词**：主题本身，例如"对赌协议"
- **同义词扩展**："对赌" OR "估值调整协议" OR "Valuation Adjustment Mechanism" OR "VAM"
- **限定词**：年份、层级、地域（如"2023 年以来"、"上交所"、"科创板"）
- **排除词**：如研究 A 股 IPO 不想混入港股，则加 "-港股 -香港"
- **案例相关**：加"案"、"裁定"、"处罚"、"纪律处分"等关键词后缀

## 六、跨 skill 数据交接

当一个 research skill 的输出作为另一个 skill 的输入时，优先使用以下字段名（JSON 兼容）：

```json
{
  "research_topic": "研究主题文本",
  "conclusions": ["结论 1", "结论 2"],
  "regulations": [
    {
      "id": "L-01",
      "full_name": "法规全称",
      "reference_number": "文号",
      "effective_date": "YYYY-MM-DD",
      "status": "现行有效 | 已修订 | 已废止",
      "article": "第 X 条",
      "quote": "条款原文片段",
      "source_url": "出处 URL",
      "reliability": "A | B | C | D",
      "access_date": "YYYY-MM-DD"
    }
  ],
  "cases": [
    {
      "id": "C-01",
      "case_type": "裁判文书 | 行政处罚 | 纪律处分 | 审议意见",
      "case_number": "案号",
      "parties": "当事人",
      "facts": "事实摘要",
      "issue": "争议焦点",
      "holding": "裁判要旨",
      "conclusion": "结论",
      "relevance": "对本项目的启示",
      "source_url": "出处 URL",
      "reliability": "A | B | C | D",
      "access_date": "YYYY-MM-DD"
    }
  ],
  "limitations": "时效性和检索局限文本",
  "next_steps": "下一步建议文本"
}
```

`ecm-research-reg-study` 的输出除上述字段外，可再增加 `hierarchy_analysis`、`conflict_resolution`、`transition_rules` 三个子对象（具体字段见该 skill 的 SKILL.md）。

## 七、变更规则

本文件的变更会波及所有 ecm-research-* skill、以及任何将 research 输出作为输入的 DD / draft / design skill。变更前：

1. 开 issue 并标记 `research-output-schema`
2. 列出受影响 skill
3. 同步更新本文件 + 各受影响 skill 的 SKILL.md
4. `CHANGELOG.md` 记为 **Changed**（对下游是输入契约变化）
