# 法律备忘录排版规范（Single Source of Truth）

本文件是**唯一权威**的中国资本市场律师"法律备忘录"文档格式定义。凡本仓库内 skill 输出"分析备忘录 / 法律意见备忘 / 方案备忘录 / 法律问题备忘"一类文档，必须引用本规范而**不得**在各自 SKILL.md 里重复定义排版细节。

本规范既是给 Claude 输出 Markdown / 纯文本初稿时的结构约束，也是交由下游 `docx` skill（或 `ecm-draft:format-adjust`）套版时的样式参数。

---

## 适用范围

- 全部 `ecm-design:*` skill 的主输出
- 未来其他 skill 输出"备忘录"形态文档时（如 `ecm-research:reg-study` 的研究备忘、某些 DD skill 对复杂议题的单独备忘）

**不适用**于：

- **律师工作报告 / 法律意见书**——排版规范另见 `shared/templates/work-report-format.md` / `legal-opinion-format.md`（后续 batch 建立），两者有固定抬头、签字页、骑缝章位、编号规则，与备忘录不同
- **见证意见 / QC 审查稿**——结论性文书有独立格式要求
- **会议文件（通知 / 议案 / 决议 / 记录）**——各自有专属版式

---

## 一、文档元信息（首页）

备忘录首页顶部必须含以下 5 行元信息，按此顺序：

```
致    ：{{客户名称}}
自    ：{{律师事务所 / 项目组名称}}
日期  ：{{YYYY 年 MM 月 DD 日}}
事由  ：关于{{主题}}的法律备忘录
保密  ：律师工作秘密，非经许可不得对外披露
```

### 字段填写规则

| 字段 | 必填 | 填写要求 |
|------|------|----------|
| 致 | 是 | 客户公司全称；多个接收对象时并列，以顿号分隔 |
| 自 | 是 | 律所全称 + 项目组（如"XX 律师事务所 ECM 项目组"） |
| 日期 | 是 | 中文格式。草稿期可标注"（草稿，尚未完稿）" |
| 事由 | 是 | 以"关于……的法律备忘录"句式；避免空泛（如"某事项的备忘录"） |
| 保密 | 是 | 固定措辞，不得省略 |

---

## 二、主标题

- **文字**：与"事由"同文，或取"事由"中的核心短语
- **字号**：16 磅（三号）加粗
- **对齐**：居中
- **段前**：24 磅
- **段后**：24 磅
- **字体**：中文楷体_GB2312；英文 Arial

---

## 三、各级标题

| 级别 | 编号 | 字号 | 加粗 | 缩进 | 示例 |
|------|------|------|------|------|------|
| 一级 | 一、二、三、…… | 12 磅（小四） | 加粗 | 首行缩进 2 字符 | **一、背景与前提假设** |
| 二级 | （一）、（二）、…… | 12 磅 | 加粗 | 首行缩进 2 字符 | **（一）项目背景** |
| 三级 | 1、2、3、…… | 12 磅 | 加粗 | 首行缩进 2 字符 | **1、关键事实** |
| 四级 | （1）、（2）、…… | 12 磅 | 加粗 | 首行缩进 2 字符 | **（1）股权结构** |

**硬性要求**：

- 不使用"Heading 1 / Heading 2"之类西式编号
- 不使用"§1.1 / §1.1.1"这种章条号（留给法规引用）
- 各级标题**与正文同字号**（12 磅），仅靠加粗 + 中文编号区分层级
- 同一层级的编号必须**连续且不断档**（即不得跳号）

---

## 四、正文段落

### 字体

- **中文**：楷体_GB2312
- **英文 / 数字**：Arial
- **颜色**：黑色（不使用红字、蓝字，除法规引用可用深蓝外）

### 段落格式

| 参数 | 取值 |
|------|------|
| 字号 | 12 磅（小四） |
| 首行缩进 | 2 字符 |
| 对齐 | 两端对齐 |
| 段前 | 0 磅 |
| 段后 | 18 磅 |
| 行距 | 最小值 16 磅 |
| 段落间空行 | **不**使用空行（段后 18 磅已形成视觉分隔） |

### 一些易犯的错误

- 不要用两个回车代替"段后 18 磅"——会导致段距不均
- 不要在段首多敲空格代替"首行缩进 2 字符"——改动样式时无法批量调整
- 不要一段内混排过多英文——如超过 30% 字符是英文，改用 Arial 为主字体并注明

---

## 五、表格

### 边框

- **边框类型**：grid 全边框
- **粗细**：外边框 1.5 磅，内边框 0.75 磅
- **颜色**：黑色

### 单元格字体

- **字号**：10 磅（五号）
- **中文**：楷体_GB2312（与正文一致）
- **英文 / 数字**：Arial

### 单元格段落

| 参数 | 取值 |
|------|------|
| 段前 | 0.2 行 |
| 段后 | 0.2 行 |
| 行距 | 最小值 12 磅 |
| 首行缩进 | 无 |

### 加粗与对齐

| 位置 | 加粗 | 水平对齐 | 垂直对齐 |
|------|------|----------|----------|
| 表头首行 | 加粗 | 居中 | 居中 |
| 首列 | **不**加粗 | 居左 | 居中 |
| 其他单元格（文字） | 不加粗 | 居左 | 居中 |
| 其他单元格（数字） | 不加粗 | 居右 | 居中 |

### 表格前后

- **表格前**：紧挨标题或引导句，**不**加空行
- **表格后**：紧接的下一段段前设为 18 磅（而非默认 0 磅），让表格和下一段产生视觉呼吸但不空一行

---

## 六、法规与文件引用

### 法规名称

- 完整书名号括起：《中华人民共和国公司法》《首次公开发行股票注册管理办法》
- 行文中第二次出现可用简称，但简称需在首次出现时用括号注明，例："《中华人民共和国证券法》（下称'《证券法》'）"
- 条文引用格式：`《公司法》第二十五条第一款` —— 用汉字数字编号 + 不省略"第""款""项"

### 交易所 / 监管规则

- 书名号一致适用：《上海证券交易所股票上市规则（2024 年修订）》
- 规范性文件（证监会公告、窗口指导意见等）引用时注明文号：`证监会公告〔2023〕12 号`

### 关键事实与文件

- 首次出现以全称 + 括号注定义短语，例：`XX 投资有限公司（下称"目标公司"）`
- 后续引用统一用定义短语，不再用全称

### 引用不确定时的处理

- 不得虚构法规名称、条文号、窗口指导文号
- 如不确定条文精确编号，用措辞："依据《XX 法》关于……的规定"或"需结合届时适用的监管规则进一步核查"

---

## 七、数字格式

### 正文内数字

- **金额 / 万元以上数值**：加千位分隔符（如 `78,742,020 元`、`1,205.36 万元`、`USD 3,500,000`）
- **百分比**：精确到小数点后一位或两位，避免过度精确（如 `52.1%`、`3.25%`；不用 `52.123456%`）
- **比例**：列明分子与分母（如 `A 持股比例 52.1%（股数 10,420,000 股 / 总股本 20,000,000 股）`）
- **日期**：`2026 年 4 月 24 日`（正文）；`2026-04-24`（表格中节省列宽时可用）
- **起止期间**：`自 2025 年 1 月 1 日至 2025 年 12 月 31 日`，不用斜杠或短横线

### 表格内数字

- 仍按中文千位分隔，但可视列宽适度简写（如将 `78,742,020 元` 简写为 `7,874.20 万元`，在列头标注单位）
- 同一列单位统一，不混用"元 / 万元 / 亿元"

### 禁止

- 不得省略货币单位（写作"3,500,000"无单位是错误）
- 不得混用阿拉伯数字与中文数字（在纯数据语境下用阿拉伯，在行文叙述"三个维度""两点建议"这类计数词用中文）

## 八、免责声明（文末固定段）

所有备忘录末尾**必须**附免责声明。统一**引用**仓库顶层 `DISCLAIMER.md`，不在 skill 内部复制全文。

推荐措辞：

> ### 免责声明
>
> 本备忘录系基于客户提供的信息及公开可查询资料进行的初步法律分析，**不构成正式法律意见**，亦不替代签字律师在尽职调查和审慎核查基础上出具的专业法律意见。相关结论可能因事实补充、法规变化或监管口径调整而需要修正。
>
> 本备忘录的使用不改变律师事务所与客户之间的委托法律关系。完整免责声明见 [DISCLAIMER.md](../../DISCLAIMER.md)。

备忘录落款（在免责声明之上，或免责声明之下另起一页）：

```
                                   {{律师事务所全称}}
                                   {{项目组联系人 / 经办律师}}
                                   {{YYYY 年 MM 月 DD 日}}
```

## 九、Markdown 初稿与 DOCX 交付的映射

Claude 在 skill 执行中输出的通常是 Markdown。当需要交付 `.docx` 终稿时，交由 `ecm-draft:format-adjust`（或直接调用外部 `docx` skill）按本文件的样式表套版。Markdown → Word 映射约定如下：

| Markdown 元素 | Word 样式 |
|---------------|-----------|
| `# {{主标题}}`（整篇仅一处） | 主标题样式：16 磅加粗居中 |
| `## 一、XX` | 一级标题（12 磅加粗 + 首行缩进 2） |
| `### （一）XX` | 二级标题 |
| `#### 1、XX` | 三级标题 |
| 正文段 | 正文样式（12 磅 / 首行缩进 2 / 段后 18 磅 / 行距最小值 16 磅） |
| 表格 | 按"五、表格"的规范套版 |
| 代码块 | **不**直接输出到终稿——如需引用原文，改为独立段落或表格 |
| 项目符号 `- ` 列表 | 尽量转为"1、2、3"编号段落；若必须保留项目符号，段后改为 6 磅、行距 1.0 行以示紧凑 |
| 引用块 `> ` | 保留为独立段落，首行缩进 0、左右各缩进 2 字符、字号 10 磅、字形楷体_GB2312 |

**硬性要求**：skill 输出 Markdown 时应已**按上述层级使用正确的 Markdown 编号**，不要把层级混乱留给下游套版工具去猜。

## 十、常见踩坑清单

1. **把"一、"和"1、"混在同一层级**——同一层必须同一编号体系
2. **表格里给首列加粗**——本规范明确首列不加粗
3. **段落间插空行**——应依赖段后 18 磅的段距，不要用额外回车
4. **金额不写单位**——必须写"元 / 万元 / 亿元 / USD / EUR"
5. **法规条文编号凭记忆写**——拿不准时改用描述性引用，不得虚构编号
6. **免责声明被删或弱化**——法律备忘录的免责声明是合规底线，不得省略
7. **主标题使用二级以上深度的 Markdown（`##`）**——主标题永远是 `#`，第一个一级标题是 `##`
8. **引用《……》时漏掉书名号**——法规、规则、文件名必须用书名号
9. **超长标题**——标题不超过 25 字，超了就拆成"主副标题"或放到正文里
10. **中英混排未分字体**——中文楷体_GB2312、英文 Arial，混排时需显式区分

## 十一、变更规则

本规范是跨 skill 共享资源，变更需：

1. 开 issue 标记 `memo-format-schema`
2. 列出受影响 skill（至少所有 `ecm-design:*`）
3. 同步更新本文件 + 受影响 skill 的 SKILL.md（如其引用了具体样式参数）
4. 在 `CHANGELOG.md` 记为 **Changed**（因为会影响已交付备忘录的返工范围）
