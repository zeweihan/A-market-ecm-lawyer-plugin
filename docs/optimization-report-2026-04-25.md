# A 股资本市场律师 Skill 项目优化报告（2026-04-25）

## 一、审阅结论

本项目的总体架构是合理的：以 `ecm-setup`、`ecm-dd`、`ecm-design`、`ecm-draft`、`ecm-qc`、`ecm-research`、`ecm-workflow` 七个模块覆盖 A 股资本市场律师从项目启动、方案设计、尽调、文书、内核到申报推进的主要工作场景；46 个 skill 采用原子能力 + workflow 编排的方式，适合律师项目组分工使用。

> 2026-04-26 复核说明：第一轮报告把“所有 skill 新增资深律师执行标准”写得过重。该项只是通用质量底线，不能替代逐个 skill 的实质审阅。第二轮已补充每个 `SKILL.md` 的独有“本 skill 的实务加固点”，逐项写入必查事项、高风险触发器、边界或补正路径。

本次优化没有推翻原架构，而是做了五类加固：

1. **资深律师质量底线统一化**：所有 46 个 `SKILL.md` 均新增统一的资深律师执行标准引用，要求事实可追溯、法源可核验、风险可分级、建议可落地。
2. **逐个 skill 实务加固**：所有 46 个 `SKILL.md` 均新增本 skill 独有的实务加固点，不再只依赖通用标准。
3. **法规时效口径更新**：新增当前核心法源索引，修正信息披露、重大资产重组、上市公司治理、股东会规则和再融资第18号适用意见等关键口径。
4. **目录与数据契约纠偏**：明确 `02-*` 是项目目录序号，不等同于《编报规则第 12 号》章节号；在 DD→Draft 契约中新增“编报章节号”字段。
5. **工具输出路径和项目文档收敛**：修正不存在的 `04-底稿/`、`02-17-数据比对/` 等路径，统一落入 `05-底稿和附件/` 下的专门子目录。

## 二、架构评估与调整

| 维度 | 原有情况 | 评价 | 本次优化 | 修订理由 |
|------|----------|------|----------|----------|
| 模块划分 | 七大模块、46 个 skill，覆盖 setup / dd / design / draft / qc / research / workflow | 合理，贴近 A 股 ECM 项目组真实分工 | 保留模块划分，不做大重构 | 原子 skill 便于单点调用，workflow 负责项目推进，不应合并成臃肿单体 |
| 原子 skill 与 workflow | 原子 skill 做具体任务，workflow 只做阶段编排 | 合理 | 为 workflow 同步增加资深律师执行标准和阶段门禁、红旗升级、产物清单等独有加固点 | workflow 虽不直接出具法律结论，但会影响工作路径和缺项提示，仍需质量底线 |
| shared 单一事实源 | 已有 templates / schemas / regulations | 基础好，但缺统一执业标准和当前法规索引 | 新增 `shared/templates/senior-lawyer-execution-standards.md` 与 `shared/regulations/current-core-regulatory-sources.md` | 20 年经验律师最关注证据、法源、风险、交付口径，不能散落在各 skill |
| DD 章节契约 | `02-*` 目录序号曾被表述为严格对应编报章节 | 有混淆风险 | `project-folder-structure.md` 和 `dd-output-schema.md` 区分“目录序号”与“编报章节号” | `02-03` 同时承载设立与历史沿革；`02-16` 对应编报第 17 章，不能误导下游拼接 |
| 工具类 skill 输出路径 | 局部使用不存在路径 `04-底稿/`、`02-17-数据比对/` | 不合理 | 统一改为 `05-底稿和附件/文件审阅/`、`05-底稿和附件/数据比对/` | 底稿和 API 原始响应属于证据留痕，应归档到附件/底稿目录 |
| 会议文件术语 | 会议文件模板大量使用“股东大会” | 需与新《公司法》和 2025 规则衔接 | 规范输出统一使用“股东会”，保留“股东大会”作为历史/市场习惯称谓识别 | 防止新出文件继续沿用旧称，同时兼容客户历史文件和口语输入 |
| 版本与状态 | `VERSION` / plugin 版本与 README 完成状态不一致 | 不合理 | 同步为 `1.0.0` | 46 个 skill 已全部登记，版本元数据应与项目状态一致 |

## 二点五、第二轮逐项实务加固摘要（2026-04-26）

本轮补强不是再加一条通用 disclaimer，而是写入每个 skill 自己的操作约束。完整逐项索引见 [skill-hardening-index-2026-04-26.md](./skill-hardening-index-2026-04-26.md)。重点如下：

| 模块 | 第二轮新增的实质内容 | 解决的问题 |
|------|----------------------|------------|
| `ecm-setup` | 增加项目类型门禁、先索引后移动、一文多标签、低置信度待确认、缺类提示 | 防止文件分类/归档阶段制造后续 DD 证据链错误 |
| `ecm-dd` | 逐章增加必查事实、高风险触发器、补正路径和相邻 skill 分流；例如批准授权查决策链闭环，历史沿革查逐次变动台账，关联交易查全口径关联方与资金占用 | 原 DD skill 框架统一但个别章节的资深律师“红旗事项”不够外显 |
| `ecm-design` | 增加路径否决项、交易结构约束、控制权事实矩阵、重组性质先判定、跨境多线审批图 | 防止方案设计变成泛泛商业建议，缺少监管路径和不可行结构识别 |
| `ecm-draft` | 增加“只拼已核验事实”“肯定结论门槛”“披露以底稿为边界”“会议程序先于模板”“格式不改实体” | 防止起草 skill 在缺底稿时生成过度肯定或实体被格式工具改动 |
| `ecm-qc` | 增加 DD 溯源审查、三向比对、程序瑕疵优先、结论穿透审查、表决结果复算 | 强化内核独立审查，不让 QC 只做格式/语言层面修改 |
| `ecm-research` | 增加案例证明力分层、法源层级排序、时效四要素、适用性三段论和过渡安排 | 防止研究输出只摘录法规或堆案例，不能支撑项目结论 |
| `ecm-workflow` | 增加阶段门禁、红旗升级、缺章显性化、组合交易并行 workflow、strict 原则 | 防止 workflow 把未完成事项包装成可以进入下一阶段 |
| `docs/skill-authoring-guide.md` | 把“本 skill 的实务加固点”列为新增/维护 skill 的强制章节 | 防止后续新增 skill 只写通用标准，没有业务个性 |
